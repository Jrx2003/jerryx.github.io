## 协同过滤概述

协同过滤（Collaborative Filtering, CF）是推荐系统中最经典、应用最广泛的算法。其核心思想是：**相似的用户会喜欢相似的物品**。

### 基本假设

1. **用户相似性**：如果两个用户在过去有相似的兴趣，他们在未来也会有相似的兴趣
2. **物品相似性**：如果两个物品被相似的用户喜欢，它们彼此相似
3. **行为即信号**：用户的历史行为（点击、购买、评分）反映了其偏好

### 协同过滤分类

```
协同过滤
├── 基于内存的方法 (Memory-Based)
│   ├── 基于用户的协同过滤 (UserCF)
│   └── 基于物品的协同过滤 (ItemCF)
│
└── 基于模型的方法 (Model-Based)
    ├── 矩阵分解 (SVD, NMF, ALS)
    ├── 聚类方法
    └── 深度学习方法
```

---

## 基于用户的协同过滤 (UserCF)

### 核心思想

找到与目标用户兴趣相似的其他用户，推荐这些相似用户喜欢但目标用户未交互过的物品。

### 算法流程

```
1. 计算用户之间的相似度
2. 找到目标用户的K个最近邻
3. 聚合邻居用户的评分/行为
4. 生成推荐列表
```

### 相似度计算

#### 余弦相似度 (Cosine Similarity)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} r_{u,i} \cdot r_{v,i}}{\sqrt{\sum_{i \in I_u} r_{u,i}^2} \cdot \sqrt{\sum_{i \in I_v} r_{v,i}^2}}$$

其中：
- $r_{u,i}$：用户$u$对物品$i$的评分
- $I_{uv}$：用户$u$和$v$共同评分过的物品集合

#### 皮尔逊相关系数 (Pearson Correlation)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_u)(r_{v,i} - \bar{r}_v)}{\sqrt{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_u)^2} \cdot \sqrt{\sum_{i \in I_{uv}} (r_{v,i} - \bar{r}_v)^2}}$$

皮尔逊相关系数考虑了用户评分的偏差，比余弦相似度更准确。

#### 修正余弦相似度 (Adjusted Cosine)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_i)(r_{v,i} - \bar{r}_i)}{\sqrt{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_i)^2} \cdot \sqrt{\sum_{i \in I_{uv}} (r_{v,i} - \bar{r}_i)^2}}$$

### 评分预测

$$\hat{r}_{u,i} = \bar{r}_u + \frac{\sum_{v \in N(u)} \text{sim}(u, v) \cdot (r_{v,i} - \bar{r}_v)}{\sum_{v \in N(u)} |\text{sim}(u, v)|}$$

### UserCF 优缺点

| 优点 | 缺点 |
|------|------|
| 发现新颖物品能力强 | 用户数量大时计算复杂度高 |
| 适合发现兴趣社区 | 用户兴趣变化快，相似度不稳定 |
| 可解释性好（"和你相似的人喜欢"） | 新用户冷启动问题严重 |
| | 稀疏数据下效果差 |

## 基于物品的协同过滤 (ItemCF)

### 核心思想

找到与目标用户历史喜欢的物品相似的物品进行推荐。

### 算法流程

```
1. 计算物品之间的相似度
2. 获取目标用户的历史行为物品
3. 找到与历史物品相似的候选物品
4. 根据相似度加权生成推荐列表
```

### 物品相似度计算

#### 余弦相似度

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} r_{u,i} \cdot r_{u,j}}{\sqrt{\sum_{u \in U_i} r_{u,i}^2} \cdot \sqrt{\sum_{u \in U_j} r_{u,j}^2}}$$

#### 改进的余弦相似度（考虑活跃用户惩罚）

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} \frac{r_{u,i} \cdot r_{u,j}}{\log(1 + |N(u)|)}}{\sqrt{\sum_{u \in U_i} r_{u,i}^2} \cdot \sqrt{\sum_{u \in U_j} r_{u,j}^2}}$$

其中 $|N(u)|$ 是用户$u$交互过的物品数量。活跃用户（交互很多物品）对相似度的贡献被降低。

### 评分预测

$$\hat{r}_{u,i} = \frac{\sum_{j \in N(i) \cap I(u)} \text{sim}(i, j) \cdot r_{u,j}}{\sum_{j \in N(i) \cap I(u)} |\text{sim}(i, j)|}$$

### ItemCF 优缺点

| 优点 | 缺点 |
|------|------|
| 物品相似度相对稳定 | 推荐新颖性不如UserCF |
| 计算可离线完成，在线速度快 | 过度个性化导致信息茧房 |
| 适合物品数量少于用户的场景 | 新物品冷启动问题 |
| 可解释性好（"因为你喜欢X，所以推荐Y"） | |

---

## UserCF vs ItemCF 对比

| 维度 | UserCF | ItemCF |
|------|--------|--------|
| **适用场景** | 用户少、物品多、社交属性强 | 用户多、物品少、个性化要求高 |
| **相似度计算** | 用户-用户 | 物品-物品 |
| **实时性** | 用户兴趣变化快，需频繁更新 | 物品相似度稳定，可离线计算 |
| **新颖性** | 高，能发现不同领域物品 | 低，局限于相似物品 |
| **可解释性** | "和你相似的人喜欢" | "因为你喜欢X" |
| **冷启动** | 新用户问题严重 | 新物品问题严重 |
| **典型应用** | 新闻推荐、社交推荐 | 电商推荐、视频推荐 |

## 相似度计算优化

### 1. 共同评分物品数惩罚

$$\text{sim}_{final}(i, j) = \text{sim}(i, j) \cdot \frac{|U_{ij}|}{|U_{ij}| + \lambda}$$

避免只有两个用户共同评分的物品被认为高度相似。

### 2. 热门物品惩罚

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} \frac{1}{\log(1 + |I(u)|)}}{\sqrt{|U_i| \cdot |U_j|}}$$

热门物品对相似度的贡献降低。

### 3. 归一化处理

将相似度归一化到[0,1]范围：

$$\text{sim}_{norm}(i, j) = \frac{\text{sim}(i, j) + 1}{2}$$

## 工业实践优化

### 1. 倒排索引加速

```python
# 构建用户-物品倒排索引
user_item_index = {
    user_id: set(item_ids)  # 用户交互过的物品集合
}

# 构建物品-用户倒排索引
item_user_index = {
    item_id: set(user_ids)  # 交互过该物品的用户集合
}

# 快速计算物品共现
for item_i in items:
    for user in item_user_index[item_i]:
        for item_j in user_item_index[user]:
            co_occurrence[item_i][item_j] += 1
```

### 2. 局部敏感哈希 (LSH)

用于近似最近邻搜索，加速大规模相似度计算。

```
1. 将用户/物品向量哈希到多个桶
2. 相似向量大概率落入同一桶
3. 只在同桶内计算精确相似度
```

### 3. 分布式计算

使用Spark等框架分布式计算相似度矩阵：

```scala
// Spark 计算 ItemCF 相似度
val itemVectors = ratings
  .groupBy("item_id")
  .agg(collect_list("user_id").as("users"))

val similarities = itemVectors
  .cartesian(itemVectors)
  .map { case (item1, item2) =>
    val sim = cosineSimilarity(item1.users, item2.users)
    (item1.item_id, item2.item_id, sim)
  }
```

## 协同过滤的问题与改进

### 1. 数据稀疏性问题

**问题**：用户-物品矩阵极度稀疏（通常 < 1%）

**解决方案**：
- 矩阵分解（SVD、ALS）
- 隐语义模型
- 混合推荐

### 2. 冷启动问题

**问题**：新用户/新物品缺乏历史交互数据

**解决方案**：
- 基于内容的推荐
- 热门推荐
- 主动学习
- 迁移学习

### 3. 可扩展性问题

**问题**：用户和物品数量巨大时计算困难

**解决方案**：
- 近似算法（LSH、随机投影）
- 分布式计算
- 增量更新

## 代码示例

### Python 实现 ItemCF

```python
import numpy as np
from collections import defaultdict

class ItemCF:
    def __init__(self, k=20):
        self.k = k  # 最近邻数量
        self.item_similarities = {}

    def fit(self, user_item_dict):
        """
        user_item_dict: {user_id: {item_id: rating, ...}, ...}
        """
        # 构建物品-用户倒排表
        item_users = defaultdict(set)
        for user, items in user_item_dict.items():
            for item in items:
                item_users[item].add(user)

        # 计算物品共现矩阵
        co_occurrence = defaultdict(lambda: defaultdict(int))
        item_counts = defaultdict(int)

        for item, users in item_users.items():
            item_counts[item] = len(users)
            for user in users:
                for other_item in user_item_dict[user]:
                    if other_item != item:
                        co_occurrence[item][other_item] += 1

        # 计算相似度
        for item_i, related_items in co_occurrence.items():
            self.item_similarities[item_i] = {}
            for item_j, co_count in related_items.items():
                # 余弦相似度
                sim = co_count / np.sqrt(item_counts[item_i] * item_counts[item_j])
                self.item_similarities[item_i][item_j] = sim

    def recommend(self, user_id, user_item_dict, n=10):
        """为指定用户生成推荐"""
        user_items = user_item_dict[user_id]
        scores = defaultdict(float)

        for item in user_items:
            if item in self.item_similarities:
                # 取Top-K相似物品
                similar_items = sorted(
                    self.item_similarities[item].items(),
                    key=lambda x: x[1],
                    reverse=True
                )[:self.k]

                for similar_item, sim in similar_items:
                    if similar_item not in user_items:
                        scores[similar_item] += sim

        # 返回Top-N推荐
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)[:n]
```

2. 找到目标用户的K个最近邻
3. 聚合邻居用户的评分/行为
4. 生成推荐列表
```

### 相似度计算

#### 余弦相似度 (Cosine Similarity)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} r_{u,i} \cdot r_{v,i}}{\sqrt{\sum_{i \in I_u} r_{u,i}^2} \cdot \sqrt{\sum_{i \in I_v} r_{v,i}^2}}$$

其中：
- $r_{u,i}$：用户$u$对物品$i$的评分
- $I_{uv}$：用户$u$和$v$共同评分过的物品集合

#### 皮尔逊相关系数 (Pearson Correlation)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_u)(r_{v,i} - \bar{r}_v)}{\sqrt{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_u)^2} \cdot \sqrt{\sum_{i \in I_{uv}} (r_{v,i} - \bar{r}_v)^2}}$$

皮尔逊相关系数考虑了用户评分的偏差，比余弦相似度更准确。

#### 修正余弦相似度 (Adjusted Cosine)

$$\text{sim}(u, v) = \frac{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_i)(r_{v,i} - \bar{r}_i)}{\sqrt{\sum_{i \in I_{uv}} (r_{u,i} - \bar{r}_i)^2} \cdot \sqrt{\sum_{i \in I_{uv}} (r_{v,i} - \bar{r}_i)^2}}$$

### 评分预测

$$\hat{r}_{u,i} = \bar{r}_u + \frac{\sum_{v \in N(u)} \text{sim}(u, v) \cdot (r_{v,i} - \bar{r}_v)}{\sum_{v \in N(u)} |\text{sim}(u, v)|}$$

### UserCF 优缺点

| 优点 | 缺点 |
|------|------|
| 发现新颖物品能力强 | 用户数量大时计算复杂度高 |
| 适合发现兴趣社区 | 用户兴趣变化快，相似度不稳定 |
| 可解释性好（"和你相似的人喜欢"） | 新用户冷启动问题严重 |
| | 稀疏数据下效果差 |

## 基于物品的协同过滤 (ItemCF)

### 核心思想

找到与目标用户历史喜欢的物品相似的物品进行推荐。

### 算法流程

```
1. 计算物品之间的相似度
2. 获取目标用户的历史行为物品
3. 找到与历史物品相似的候选物品
4. 根据相似度加权生成推荐列表
```

### 物品相似度计算

#### 余弦相似度

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} r_{u,i} \cdot r_{u,j}}{\sqrt{\sum_{u \in U_i} r_{u,i}^2} \cdot \sqrt{\sum_{u \in U_j} r_{u,j}^2}}$$

#### 改进的余弦相似度（考虑活跃用户惩罚）

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} \frac{r_{u,i} \cdot r_{u,j}}{\log(1 + |N(u)|)}}{\sqrt{\sum_{u \in U_i} r_{u,i}^2} \cdot \sqrt{\sum_{u \in U_j} r_{u,j}^2}}$$

其中 $|N(u)|$ 是用户$u$交互过的物品数量。活跃用户（交互很多物品）对相似度的贡献被降低。

### 评分预测

$$\hat{r}_{u,i} = \frac{\sum_{j \in N(i) \cap I(u)} \text{sim}(i, j) \cdot r_{u,j}}{\sum_{j \in N(i) \cap I(u)} |\text{sim}(i, j)|}$$

### ItemCF 优缺点

| 优点 | 缺点 |
|------|------|
| 物品相似度相对稳定 | 推荐新颖性不如UserCF |
| 计算可离线完成，在线速度快 | 过度个性化导致信息茧房 |
| 适合物品数量少于用户的场景 | 新物品冷启动问题 |
| 可解释性好（"因为你喜欢X，所以推荐Y"） | |

---

## UserCF vs ItemCF 对比

| 维度 | UserCF | ItemCF |
|------|--------|--------|
| **适用场景** | 用户少、物品多、社交属性强 | 用户多、物品少、个性化要求高 |
| **相似度计算** | 用户-用户 | 物品-物品 |
| **实时性** | 用户兴趣变化快，需频繁更新 | 物品相似度稳定，可离线计算 |
| **新颖性** | 高，能发现不同领域物品 | 低，局限于相似物品 |
| **可解释性** | "和你相似的人喜欢" | "因为你喜欢X" |
| **冷启动** | 新用户问题严重 | 新物品问题严重 |
| **典型应用** | 新闻推荐、社交推荐 | 电商推荐、视频推荐 |

---

## 相似度计算优化

### 1. 共同评分物品数惩罚

$$\text{sim}_{final}(i, j) = \text{sim}(i, j) \cdot \frac{|U_{ij}|}{|U_{ij}| + \lambda}$$

避免只有两个用户共同评分的物品被认为高度相似。

### 2. 热门物品惩罚

$$\text{sim}(i, j) = \frac{\sum_{u \in U_{ij}} \frac{1}{\log(1 + |I(u)|)}}{\sqrt{|U_i| \cdot |U_j|}}$$

热门物品对相似度的贡献降低。

### 3. 归一化处理

将相似度归一化到[0,1]范围：

$$\text{sim}_{norm}(i, j) = \frac{\text{sim}(i, j) + 1}{2}$$

## 工业实践优化

### 1. 倒排索引加速

```python
# 构建用户-物品倒排索引
user_item_index = {
    user_id: set(item_ids)  # 用户交互过的物品集合
}

# 构建物品-用户倒排索引
item_user_index = {
    item_id: set(user_ids)  # 交互过该物品的用户集合
}

# 快速计算物品共现
for item_i in items:
    for user in item_user_index[item_i]:
        for item_j in user_item_index[user]:
            co_occurrence[item_i][item_j] += 1
```

### 2. 局部敏感哈希 (LSH)

用于近似最近邻搜索，加速大规模相似度计算。

```
1. 将用户/物品向量哈希到多个桶
2. 相似向量大概率落入同一桶
3. 只在同桶内计算精确相似度
```

### 3. 分布式计算

使用Spark等框架分布式计算相似度矩阵：

```scala
// Spark 计算 ItemCF 相似度
val itemVectors = ratings
  .groupBy("item_id")
  .agg(collect_list("user_id").as("users"))

val similarities = itemVectors
  .cartesian(itemVectors)
  .map { case (item1, item2) =>
    val sim = cosineSimilarity(item1.users, item2.users)
    (item1.item_id, item2.item_id, sim)
  }
```

## 协同过滤的问题与改进

### 1. 数据稀疏性问题

**问题**：用户-物品矩阵极度稀疏（通常 < 1%）

**解决方案**：
- 矩阵分解（SVD、ALS）
- 隐语义模型
- 混合推荐

### 2. 冷启动问题

**问题**：新用户/新物品缺乏历史交互数据

**解决方案**：
- 基于内容的推荐
- 热门推荐
- 主动学习
- 迁移学习

### 3. 可扩展性问题

**问题**：用户和物品数量巨大时计算困难

**解决方案**：
- 近似算法（LSH、随机投影）
- 分布式计算
- 增量更新

## 代码示例

### Python 实现 ItemCF

```python
import numpy as np
from collections import defaultdict

class ItemCF:
    def __init__(self, k=20):
        self.k = k  # 最近邻数量
        self.item_similarities = {}

    def fit(self, user_item_dict):
        """
        user_item_dict: {user_id: {item_id: rating, ...}, ...}
        """
        # 构建物品-用户倒排表
        item_users = defaultdict(set)
        for user, items in user_item_dict.items():
            for item in items:
                item_users[item].add(user)

        # 计算物品共现矩阵
        co_occurrence = defaultdict(lambda: defaultdict(int))
        item_counts = defaultdict(int)

        for item, users in item_users.items():
            item_counts[item] = len(users)
            for user in users:
                for other_item in user_item_dict[user]:
                    if other_item != item:
                        co_occurrence[item][other_item] += 1

        # 计算相似度
        for item_i, related_items in co_occurrence.items():
            self.item_similarities[item_i] = {}
            for item_j, co_count in related_items.items():
                # 余弦相似度
                sim = co_count / np.sqrt(item_counts[item_i] * item_counts[item_j])
                self.item_similarities[item_i][item_j] = sim

    def recommend(self, user_id, user_item_dict, n=10):
        """为指定用户生成推荐"""
        user_items = user_item_dict[user_id]
        scores = defaultdict(float)

        for item in user_items:
            if item in self.item_similarities:
                # 取Top-K相似物品
                similar_items = sorted(
                    self.item_similarities[item].items(),
                    key=lambda x: x[1],
                    reverse=True
                )[:self.k]

                for similar_item, sim in similar_items:
                    if similar_item not in user_items:
                        scores[similar_item] += sim

        # 返回Top-N推荐
        return sorted(scores.items(), key=lambda x: x[1], reverse=True)[:n]
```
