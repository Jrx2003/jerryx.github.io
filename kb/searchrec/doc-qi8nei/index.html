生成式推荐概述

什么是生成式推荐

生成式推荐将推荐任务建模为序列生成问题，用生成模型直接生成推荐结果。

判别式推荐:
输入: 用户特征 + 候选物品池
输出: 每个候选物品的分数
过程: 打分 + 排序

生成式推荐:
输入: 用户历史序列
输出: 下一个物品的 ID (直接生成)
过程: 序列生成

核心思想

推荐作为语言建模:

用户历史: [item_12] [item_45] [item_78]
下一个物品: [item_23]

相当于语言模型:
"12 45 78" → "23"

优势:
- 统一框架处理所有推荐场景
- 利用 LLM 的强大能力
- 更好的泛化性

生成式 vs 判别式

对比

维度判别式生成式
建模方式评分函数概率分布
候选池需要预先定义可以是开放的
输出分数物品 ID/描述
解释性较弱可生成推荐理由
训练数据正样本为主序列数据
代表性模型MF, DNNGPT4Rec, RecFormer

优缺点

生成式优点:

  • 统一框架
  • 可解释性强
  • 冷启动友好
  • 跨域迁移

生成式挑战:

  • 物品 ID 空间巨大
  • 训练不稳定
  • 推理速度慢
  • 新颖性问题

代表性方法

GPT4Rec

"""
GPT4Rec: 将推荐转化为序列生成

核心思想:
1. 将物品视为 token
2. 用 GPT 建模用户行为序列
3. 自回归生成下一个物品
"""

class GPT4Rec(nn.Module):
    """GPT4Rec 简化实现"""

    def __init__(self, num_items, embed_dim=256, num_layers=6):
        super().__init__()

        # 物品嵌入表
        self.item_embedding = nn.Embedding(num_items, embed_dim)
        self.position_embedding = nn.Embedding(512, embed_dim)

        # Transformer Decoder
        decoder_layer = nn.TransformerDecoderLayer(
            d_model=embed_dim,
            nhead=8,
            batch_first=True
        )
        self.transformer = nn.TransformerDecoder(
            decoder_layer,
            num_layers=num_layers
        )

        # 输出层
        self.output = nn.Linear(embed_dim, num_items)

    def forward(self, item_seq):
        """
        Args:
            item_seq: [batch, seq_len]
        """
        batch_size, seq_len = item_seq.shape

        # 嵌入
        item_emb = self.item_embedding(item_seq)
        pos_emb = self.position_embedding(
            torch.arange(seq_len, device=item_seq.device)
        )
        x = item_emb + pos_emb

        # 因果 mask
        mask = torch.triu(
            torch.ones(seq_len, seq_len, device=item_seq.device),
            diagonal=1
        ).bool()

        # Transformer
        out = self.transformer(x, x, tgt_mask=mask)

        # 预测下一个物品
        logits = self.output(out)

        return logits

RecFormer

"""
RecFormer: 结合语言模型和推荐

特点:
- 使用预训练语言模型 (如 BERT/GPT)
- 物品用文本描述表示
- 联合优化语言和推荐任务
"""

class RecFormer(nn.Module):
    """RecFormer 概念实现"""

    def __init__(self, lm_name='bert-base'):
        super().__init__()

        # 加载预训练语言模型
        self.lm = AutoModel.from_pretrained(lm_name)

        # 推荐头
        self.rec_head = nn.Sequential(
            nn.Linear(self.lm.config.hidden_size, 512),
            nn.ReLU(),
            nn.Linear(512, 1)
        )

    def encode_items(self, item_texts):
        """用 LM 编码物品文本描述"""
        inputs = self.tokenizer(
            item_texts,
            padding=True,
            truncation=True,
            return_tensors='pt'
        )

        outputs = self.lm(**inputs)
        # 取 [CLS] token 作为物品表示
        item_embeddings = outputs.last_hidden_state[:, 0]

        return item_embeddings

    def forward(self, user_history_texts, candidate_texts):
        """
        Args:
            user_history_texts: 用户历史物品描述列表
            candidate_texts: 候选物品描述列表
        """
        # 编码用户历史
        history_embeddings = self.encode_items(user_history_texts)
        user_repr = history_embeddings.mean(dim=0)  # 简单平均

        # 编码候选物品
        candidate_embeddings = self.encode_items(candidate_texts)

        # 计算匹配分数
        scores = torch.matmul(
            user_repr.unsqueeze(0),
            candidate_embeddings.T
        )

        return scores

生成式推荐的优化

"""
生成式推荐的关键优化:

1. 物品 ID 压缩
   - 使用分层编码
   - 向量量化 (VQ-VAE)

2. 训练稳定性
   - 标签平滑
   - 课程学习

3. 推理加速
   - 投机解码
   - 缓存机制
"""

class HierarchicalItemID(nn.Module):
    """分层物品 ID 编码"""

    def __init__(self, num_items, num_categories):
        super().__init__()

        self.category_embed = nn.Embedding(num_categories, 128)
        self.within_cat_embed = nn.Embedding(
            num_items // num_categories + 1, 128
        )

    def forward(self, item_ids, category_ids):
        """分层表示物品"""
        cat_emb = self.category_embed(category_ids)
        local_emb = self.within_cat_embed(item_ids % 1000)

        return torch.cat([cat_emb, local_emb], dim=-1)

面试要点

高频问题

Q1: 生成式推荐和传统推荐的区别？

A:

  • 建模方式: 生成式直接生成物品，判别式打分排序
  • 输出形式: 生成式输出 ID/文本，判别式输出分数
  • 候选池: 生成式可以是开放的，判别式需要预定义

Q2: 生成式推荐的优缺点？

A:

  • 优点: 统一框架、可解释性强、跨域迁移
  • 缺点: ID空间巨大、训练不稳定、推理慢

Q3: 如何处理物品 ID 空间巨大的问题？

A:

  1. 分层编码 (类别 + 局部 ID)
  2. 向量量化
  3. 采样负样本
  4. 使用文本描述而非 ID

Q4: 生成式推荐的评估指标？

A:

  • Recall@K, NDCG@K (和传统推荐相同)
  • 生成质量 (流畅度、多样性)
  • 解释质量 (人工评估)
