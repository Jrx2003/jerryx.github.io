## 什么是序列推荐

### 定义

序列推荐根据用户的历史行为序列预测下一个感兴趣的物品。

```
传统推荐:
用户画像 + 物品特征 → 预测评分

序列推荐:
用户行为序列 [i1, i2, i3, ..., in] → 预测 i(n+1)

优势:
- 捕捉用户兴趣变化
- 利用时序信息
- 适合短会话场景
```

### 应用场景

| 场景 | 序列 | 预测目标 |
|------|------|----------|
| 电商 | 浏览商品序列 | 下一个购买商品 |
| 视频 | 观看历史 | 下一个观看视频 |
| 音乐 | 播放列表 | 下一首歌曲 |
| 新闻 | 阅读历史 | 下一篇感兴趣新闻 |

---

## 核心挑战

### 1. 序列长度变化

```
用户A: [item1] → [item2] (短序列)
用户B: [item1] → ... → [item100] (长序列)

解决方案:
- Padding + Masking
- 截断
- 分层建模
```

### 2. 兴趣漂移

```
用户兴趣随时间变化:
- 早上: 看新闻
- 中午: 看美食
- 晚上: 看视频

解决方案:
- 时间衰减
- 注意力机制
- 多兴趣建模
```

### 3. 数据稀疏性

```
冷启动用户: 只有 1-2 个交互
解决方案:
- 结合用户画像
- 迁移学习
- 元学习
```

## 基础模型

### 基于马尔可夫链

```python
class MarkovChain:
    """一阶马尔可夫链"""

    def __init__(self, num_items):
        self.num_items = num_items
        # 转移概率矩阵
        self.transition = np.zeros((num_items, num_items))

    def fit(self, sequences):
        """学习转移概率"""
        for seq in sequences:
            for i in range(len(seq) - 1):
                self.transition[seq[i], seq[i+1]] += 1

        # 归一化
        row_sums = self.transition.sum(axis=1)
        self.transition = self.transition / row_sums[:, np.newaxis]

    def predict_next(self, last_item, top_k=10):
        """预测下一个物品"""
        probs = self.transition[last_item]
        top_items = np.argsort(probs)[-top_k:][::-1]
        return top_items
```

### 基于 RNN

```python
class SessionRNN(nn.Module):
    """基于 RNN 的序列模型"""

    def __init__(self, num_items, embed_dim=50, hidden_dim=100):
        super().__init__()

        self.item_embedding = nn.Embedding(num_items, embed_dim)
        self.gru = nn.GRU(embed_dim, hidden_dim, batch_first=True)
        self.output = nn.Linear(hidden_dim, num_items)

    def forward(self, item_seq):
        # item_seq: [batch, seq_len]
        embedded = self.item_embedding(item_seq)

        # GRU
        gru_out, hidden = self.gru(embedded)

        # 取最后时刻的 hidden
        logits = self.output(hidden.squeeze(0))

        return logits
```

## 面试要点

### 高频问题

**Q1: 序列推荐和传统推荐有什么区别？**

A:
- 序列推荐关注行为顺序，传统推荐关注静态偏好
- 序列推荐适合短会话，传统推荐需要长期积累
- 序列推荐能捕捉兴趣变化

**Q2: 如何处理变长序列？**

A:
- Padding 到固定长度
- 使用 Mask 忽略 padding
- RNN/Transformer 的变长处理
