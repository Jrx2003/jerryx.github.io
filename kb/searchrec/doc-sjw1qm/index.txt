## 长序列挑战

### 为什么长序列难

```
问题:
- 用户历史可能数百上千
- Transformer 复杂度 O(n²)
- 内存和计算开销大

目标:
- 有效利用长期历史
- 控制计算复杂度
- 捕捉长期兴趣
```

### 解决方案分类

| 类型 | 方法 | 代表工作 |
|------|------|----------|
| 采样 | 选择相关物品 | SIM |
| 哈希 | 局部敏感哈希 | ETA |
| 降维 | 压缩序列 | SDIM |

---

## SIM

### 概述

SIM (Search-based Interest Model) 分两阶段处理长序列。

```
阶段 1: Hard Search
- 根据类别/品牌筛选相关物品
- 从万级降到百级

阶段 2: Soft Search
- 用注意力建模筛选后的序列
- 精确捕捉兴趣
```

### 架构

```
用户长序列: [item1, item2, ..., item10000]
              ↓
         Hard Search (按类别筛选)
              ↓
候选物品: [item101, item205, ..., item500] (百级)
              ↓
         Soft Search (Attention)
              ↓
         兴趣表示
```

---

## ETA

### 概述

ETA (End-to-end Target Attention) 使用局部敏感哈希 (LSH) 加速。

```
核心思想:
- 用 LSH 找到与目标物品相似的物品
- 只对这些物品计算注意力
- 复杂度从 O(n) 降到 O(log n)
```

---

## SDIM

### 概述

SDIM (Sampling-based Deep Interest Modeling) 使用采样方法。

```
核心思想:
- 随机采样部分历史物品
- 多轮采样取平均
- 降低方差
```

---

## 面试要点

**Q1: 长序列建模的主要挑战？**

A:
- 计算复杂度高
- 内存开销大
- 噪声多

**Q2: SIM 的两阶段分别是什么？**

A:
- Hard Search: 快速筛选相关物品
- Soft Search: 精细建模兴趣
