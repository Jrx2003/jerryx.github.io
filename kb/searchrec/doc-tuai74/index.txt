## 一、推荐系统基础

### 召回-排序架构

**Q1: 推荐系统为什么需要分召回、粗排、精排、重排？**

**答案**:
```
用户请求
    │
    ▼
召回层 (百万→千)      - 速度优先，快速筛选候选
    │
粗排层 (千→百)        - 轻量模型，初步筛选
    │
精排层 (百→十)        - 复杂模型，精准打分
    │
重排层 (十→N)         - 业务规则，多样性控制
```

**分层原因**:
1. **计算效率**: 全量精排计算量太大
2. **实时性**: 用户等待时间有限
3. **效果平衡**: 不同阶段有不同的优化目标

---

### 评估指标

**Q2: AUC 和 GAUC 的区别？为什么推荐场景更看重 GAUC？**

**答案**:

| 指标 | 计算方式 | 问题 |
|------|----------|------|
| **AUC** | 全局排序能力 | 受热门用户/物品影响大 |
| **GAUC** | 分组 AUC 的平均 | 反映每个用户的体验 |

```python
# GAUC 计算
def compute_gauc(predictions, labels, user_ids):
    """
    predictions: [N] 预测分数
    labels: [N] 真实标签
    user_ids: [N] 用户ID
    """
    user_aucs = []
    for user in set(user_ids):
        mask = user_ids == user
        user_pred = predictions[mask]
        user_label = labels[mask]

        if len(set(user_label)) > 1:  # 必须有正负样本
            user_auc = compute_auc(user_pred, user_label)
            user_aucs.append(user_auc)

    return np.mean(user_aucs)
```

**推荐场景用 GAUC**:
- 推荐是千人千面的，应该看每个用户的满意度
- 全局 AUC 可能被少数活跃用户主导

**Q3: NDCG 和 AUC 的区别？**

**答案**:
- **AUC**: 评估排序相对顺序，只关心正样本在前
- **NDCG**: 评估排序质量，考虑位置信息和相关性程度

| 场景 | 推荐指标 |
|------|----------|
| CTR 预估 | AUC, GAUC, LogLoss |
| 搜索排序 | NDCG, MAP, MRR |
| 多样性 | Coverage, Intra-list Diversity |

## 二、算法原理

### 协同过滤

**Q4: UserCF 和 ItemCF 的区别？**

**答案**:

| 维度 | UserCF | ItemCF |
|------|--------|--------|
| 相似度计算 | 用户-用户 | 物品-物品 |
| 推荐解释 | "和你相似的人喜欢" | "因为你喜欢X" |
| 新颖性 | 高 | 低 |
| 冷启动 | 新用户问题 | 新物品问题 |
| 适用场景 | 用户少、社交属性 | 用户多、个性化 |

**Q5: 协同过滤的数据稀疏问题如何解决？**

**答案**:
1. **矩阵分解**: SVD, ALS
2. **隐语义模型**: 学习隐向量
3. **混合推荐**: 结合内容特征
4. **负采样**: 从未交互中采样负例

---

### 深度学习推荐

**Q6: Wide&Deep 中 Wide 和 Deep 的作用？**

**答案**:
- **Wide**: 记忆能力，学习历史数据中的频繁模式
- **Deep**: 泛化能力，学习未出现过的特征组合
- **联合训练**: 平衡记忆与泛化

**Q7: DIN 的注意力机制有什么特别之处？**

**答案**:
- **传统 Attention**: Query 是模型参数
- **DIN Attention**: Query 是候选物品，随样本变化
- **目的**: 捕捉用户对不同候选物品的不同兴趣

**Q8: 为什么需要多任务学习？**

**答案**:
场景：同时优化点击率和转化率

**优势**:
1. **数据共享**: 点击数据多，转化数据少，共享表示
2. **正则化**: 多任务互相约束，防止过拟合
3. **一致性**: 保证 CTR > CVR

**常用模型**: MMoE, PLE, ESMM

---

### 搜索算法

**Q9: Learning to Rank 的三种方法？**

**答案**:

| 方法 | 粒度 | 目标 |
|------|------|------|
| **Pointwise** | 单文档 | 预测相关性分数 |
| **Pairwise** | 文档对 | 学习相对顺序 |
| **Listwise** | 文档列表 | 直接优化排序指标 |

**代表算法**:
- Pointwise: Logistic Regression
- Pairwise: RankSVM, LambdaRank
- Listwise: LambdaMART, ListNet

**Q10: BM25 公式解释？**

**答案**:
```
BM25(q, d) = Σ IDF(q_i) × f(q_i, d) × (k_1 + 1)
             ─────────────────────────────────────
             f(q_i, d) + k_1 × (1 - b + b × |D|/avgdl)

其中:
- IDF: 逆文档频率
- f(q_i, d): 词频
- k_1: 控制词频饱和度 (通常1.2-2.0)
- b: 控制文档长度归一化 (通常0.75)
```

## 三、工程实践

### 特征工程

**Q11: 推荐系统有哪些特征？**

**答案**:
```
用户特征:
  - 画像: 年龄、性别、地域
  - 行为: 历史 CTR、活跃度、偏好类目
  - 上下文: 设备、网络、时间

物品特征:
  - 属性: 类目、品牌、价格
  - 统计: 点击率、销量、评分
  - 内容: 标题、图片、描述

交叉特征:
  - 用户-物品: 用户对类目的偏好
  - 用户-上下文: 时段偏好
  - 物品-上下文: 时段表现
```

**Q12: Embedding 维度如何选择？**

**答案**:
- **高频特征** (如 user_id): 16-64 维
- **中频特征** (如 category): 8-16 维
- **低频特征** (如 tag): 4-8 维
- **经验公式**: embedding_dim ∝ log(vocab_size)

### 模型训练

**Q13: 如何处理类别不平衡？**

**答案**:
1. **采样**: 下采样负例 或 上采样正例
2. **加权**: 正样本权重更高
3. **Focal Loss**: 降低易分样本权重

```python
# Focal Loss
class FocalLoss(nn.Module):
    def __init__(self, alpha=0.25, gamma=2):
        super().__init__()
        self.alpha = alpha
        self.gamma = gamma

    def forward(self, pred, target):
        bce = F.binary_cross_entropy(pred, target, reduction='none')
        p_t = (target * pred) + ((1 - target) * (1 - pred))
        alpha_t = target * self.alpha + (1 - target) * (1 - self.alpha)
        return (alpha_t * (1 - p_t) ** self.gamma * bce).mean()
```

**Q14: 离线 AUC 高，线上 CTR 低，可能原因？**

**答案**:
1. **数据分布变化**: 训练/测试数据与线上不一致
2. **特征穿越**: 使用了未来的信息
3. **选择偏差**: 训练数据是有偏的曝光数据
4. **评估指标**: AUC 只反映排序，不反映校准

## 四、冷启动与探索

### 冷启动

**Q15: 新用户冷启动如何解决？**

**答案**:
1. **基于内容**: 利用人口统计学推荐
2. **热门推荐**: 展示全站热门
3. **主动学习**: 让用户选择兴趣标签
4. **迁移学习**: 相似用户的偏好

**Q16: EE (Explore & Exploit) 问题是什么？**

**答案**:
- **Exploit**: 利用已知的高收益物品
- **Explore**: 探索未知但可能高收益的物品
- **平衡**: 短期收益 vs 长期收益

**常用算法**:
- ε-greedy
- Thompson Sampling
- UCB (Upper Confidence Bound)

**Q17: 如何设计一个实时推荐系统？**

**Q18: 如何实现推荐系统的 A/B 测试？**

**答案**:
```python
def ab_test_bucket(user_id, experiment_id, num_buckets=100):
    """分流"""
    hash_val = hash(f"{user_id}_{experiment_id}") % num_buckets

    if hash_val < 50:
        return "control"  # 对照组
    else:
        return "treatment"  # 实验组
```

**关键要素**:
1. **分流**: 用户级随机，保证一致性
2. **指标**: CTR, CVR, 留存, GMV
3. **时间**: 至少 1-2 周，覆盖周期效应
4. **检验**: AA 测试验证系统正常

## 八、代码题

### 快速实现

**Q23: 实现一个简单的 Top-K 推荐**

```python
def topk_recommend(user_id, user_item_matrix, k=10):
    """
    基于 ItemCF 的 Top-K 推荐
    """
    # 用户历史
    user_history = user_item_matrix[user_id]

    # 计算物品相似度
    item_similarity = cosine_similarity(user_item_matrix.T)

    # 计算推荐分数
    scores = np.zeros(user_item_matrix.shape[1])
    for item_id in user_history:
        scores += item_similarity[item_id]

    # 排除已交互
    scores[user_history] = -np.inf

    # Top-K
    topk_items = np.argsort(scores)[-k:][::-1]
    return topk_items
```

**Q24: 计算 AUC**

```python
def compute_auc(y_true, y_pred):
    """计算 AUC"""
    pos_scores = y_pred[y_true == 1]
    neg_scores = y_pred[y_true == 0]

    # 计算正负样本对的比较
    correct_pairs = 0
    total_pairs = 0

    for pos in pos_scores:
        correct_pairs += np.sum(pos > neg_scores)
        total_pairs += len(neg_scores)

    return correct_pairs / total_pairs
```

## 一、推荐系统基础

### 召回-排序架构

**Q1: 推荐系统为什么需要分召回、粗排、精排、重排？**

**答案**:
```
用户请求
    │
    ▼
召回层 (百万→千)      - 速度优先，快速筛选候选
    │
粗排层 (千→百)        - 轻量模型，初步筛选
    │
精排层 (百→十)        - 复杂模型，精准打分
    │
重排层 (十→N)         - 业务规则，多样性控制
```

**分层原因**:
1. **计算效率**: 全量精排计算量太大
2. **实时性**: 用户等待时间有限
3. **效果平衡**: 不同阶段有不同的优化目标

---

### 评估指标

**Q2: AUC 和 GAUC 的区别？为什么推荐场景更看重 GAUC？**

**答案**:

| 指标 | 计算方式 | 问题 |
|------|----------|------|
| **AUC** | 全局排序能力 | 受热门用户/物品影响大 |
| **GAUC** | 分组 AUC 的平均 | 反映每个用户的体验 |

```python
# GAUC 计算
def compute_gauc(predictions, labels, user_ids):
    """
    predictions: [N] 预测分数
    labels: [N] 真实标签
    user_ids: [N] 用户ID
    """
    user_aucs = []
    for user in set(user_ids):
        mask = user_ids == user
        user_pred = predictions[mask]
        user_label = labels[mask]

        if len(set(user_label)) > 1:  # 必须有正负样本
            user_auc = compute_auc(user_pred, user_label)
            user_aucs.append(user_auc)

    return np.mean(user_aucs)
```

**推荐场景用 GAUC**:
- 推荐是千人千面的，应该看每个用户的满意度
- 全局 AUC 可能被少数活跃用户主导

**Q3: NDCG 和 AUC 的区别？**

**答案**:
- **AUC**: 评估排序相对顺序，只关心正样本在前
- **NDCG**: 评估排序质量，考虑位置信息和相关性程度

| 场景 | 推荐指标 |
|------|----------|
| CTR 预估 | AUC, GAUC, LogLoss |
| 搜索排序 | NDCG, MAP, MRR |
| 多样性 | Coverage, Intra-list Diversity |

## 二、算法原理

### 协同过滤

**Q4: UserCF 和 ItemCF 的区别？**

**答案**:

| 维度 | UserCF | ItemCF |
|------|--------|--------|
| 相似度计算 | 用户-用户 | 物品-物品 |
| 推荐解释 | "和你相似的人喜欢" | "因为你喜欢X" |
| 新颖性 | 高 | 低 |
| 冷启动 | 新用户问题 | 新物品问题 |
| 适用场景 | 用户少、社交属性 | 用户多、个性化 |

**Q5: 协同过滤的数据稀疏问题如何解决？**

**答案**:
1. **矩阵分解**: SVD, ALS
2. **隐语义模型**: 学习隐向量
3. **混合推荐**: 结合内容特征
4. **负采样**: 从未交互中采样负例

---

### 深度学习推荐

**Q6: Wide&Deep 中 Wide 和 Deep 的作用？**

**答案**:
- **Wide**: 记忆能力，学习历史数据中的频繁模式
- **Deep**: 泛化能力，学习未出现过的特征组合
- **联合训练**: 平衡记忆与泛化

**Q7: DIN 的注意力机制有什么特别之处？**

**答案**:
- **传统 Attention**: Query 是模型参数
- **DIN Attention**: Query 是候选物品，随样本变化
- **目的**: 捕捉用户对不同候选物品的不同兴趣

**Q8: 为什么需要多任务学习？**

**答案**:
场景：同时优化点击率和转化率

**优势**:
1. **数据共享**: 点击数据多，转化数据少，共享表示
2. **正则化**: 多任务互相约束，防止过拟合
3. **一致性**: 保证 CTR > CVR

**常用模型**: MMoE, PLE, ESMM

---

### 搜索算法

**Q9: Learning to Rank 的三种方法？**

**答案**:

| 方法 | 粒度 | 目标 |
|------|------|------|
| **Pointwise** | 单文档 | 预测相关性分数 |
| **Pairwise** | 文档对 | 学习相对顺序 |
| **Listwise** | 文档列表 | 直接优化排序指标 |

**代表算法**:
- Pointwise: Logistic Regression
- Pairwise: RankSVM, LambdaRank
- Listwise: LambdaMART, ListNet

**Q10: BM25 公式解释？**

**答案**:
```
BM25(q, d) = Σ IDF(q_i) × f(q_i, d) × (k_1 + 1)
             ─────────────────────────────────────
             f(q_i, d) + k_1 × (1 - b + b × |D|/avgdl)

其中:
- IDF: 逆文档频率
- f(q_i, d): 词频
- k_1: 控制词频饱和度 (通常1.2-2.0)
- b: 控制文档长度归一化 (通常0.75)
```

## 三、工程实践

### 特征工程

**Q11: 推荐系统有哪些特征？**

**答案**:
```
用户特征:
  - 画像: 年龄、性别、地域
  - 行为: 历史 CTR、活跃度、偏好类目
  - 上下文: 设备、网络、时间

物品特征:
  - 属性: 类目、品牌、价格
  - 统计: 点击率、销量、评分
  - 内容: 标题、图片、描述

交叉特征:
  - 用户-物品: 用户对类目的偏好
  - 用户-上下文: 时段偏好
  - 物品-上下文: 时段表现
```

**Q12: Embedding 维度如何选择？**

**答案**:
- **高频特征** (如 user_id): 16-64 维
- **中频特征** (如 category): 8-16 维
- **低频特征** (如 tag): 4-8 维
- **经验公式**: embedding_dim ∝ log(vocab_size)

### 模型训练

**Q13: 如何处理类别不平衡？**

**答案**:
1. **采样**: 下采样负例 或 上采样正例
2. **加权**: 正样本权重更高
3. **Focal Loss**: 降低易分样本权重

```python
# Focal Loss
class FocalLoss(nn.Module):
    def __init__(self, alpha=0.25, gamma=2):
        super().__init__()
        self.alpha = alpha
        self.gamma = gamma

    def forward(self, pred, target):
        bce = F.binary_cross_entropy(pred, target, reduction='none')
        p_t = (target * pred) + ((1 - target) * (1 - pred))
        alpha_t = target * self.alpha + (1 - target) * (1 - self.alpha)
        return (alpha_t * (1 - p_t) ** self.gamma * bce).mean()
```

**Q14: 离线 AUC 高，线上 CTR 低，可能原因？**

**答案**:
1. **数据分布变化**: 训练/测试数据与线上不一致
2. **特征穿越**: 使用了未来的信息
3. **选择偏差**: 训练数据是有偏的曝光数据
4. **评估指标**: AUC 只反映排序，不反映校准

## 四、冷启动与探索

### 冷启动

**Q15: 新用户冷启动如何解决？**

**答案**:
1. **基于内容**: 利用人口统计学推荐
2. **热门推荐**: 展示全站热门
3. **主动学习**: 让用户选择兴趣标签
4. **迁移学习**: 相似用户的偏好

**Q16: EE (Explore & Exploit) 问题是什么？**

**答案**:
- **Exploit**: 利用已知的高收益物品
- **Explore**: 探索未知但可能高收益的物品
- **平衡**: 短期收益 vs 长期收益

**常用算法**:
- ε-greedy
- Thompson Sampling
- UCB (Upper Confidence Bound)

## 五、系统设计

### 架构设计

**Q17: 如何设计一个实时推荐系统？**

**答案要点**:
```
数据流:
用户行为 → Kafka → Flink (特征计算) → Redis → 推荐服务

推荐服务:
1. 读取用户画像 (Redis)
2. 召回 (多路并行)
3. 粗排 (轻量模型)
4. 精排 (深度模型)
5. 重排 (业务规则)
6. 返回结果

关键技术:
- 特征实时更新 (Flink)
- 模型在线服务 (TensorFlow Serving/Triton)
- 缓存策略 (Redis)
```

**Q18: 如何实现推荐系统的 A/B 测试？**

**答案**:
```python
def ab_test_bucket(user_id, experiment_id, num_buckets=100):
    """分流"""
    hash_val = hash(f"{user_id}_{experiment_id}") % num_buckets

    if hash_val < 50:
        return "control"  # 对照组
    else:
        return "treatment"  # 实验组
```

**关键要素**:
1. **分流**: 用户级随机，保证一致性
2. **指标**: CTR, CVR, 留存, GMV
3. **时间**: 至少 1-2 周，覆盖周期效应
4. **检验**: AA 测试验证系统正常

## 八、代码题

### 快速实现

**Q23: 实现一个简单的 Top-K 推荐**

```python
def topk_recommend(user_id, user_item_matrix, k=10):
    """
    基于 ItemCF 的 Top-K 推荐
    """
    # 用户历史
    user_history = user_item_matrix[user_id]

    # 计算物品相似度
    item_similarity = cosine_similarity(user_item_matrix.T)

    # 计算推荐分数
    scores = np.zeros(user_item_matrix.shape[1])
    for item_id in user_history:
        scores += item_similarity[item_id]

    # 排除已交互
    scores[user_history] = -np.inf

    # Top-K
    topk_items = np.argsort(scores)[-k:][::-1]
    return topk_items
```

**Q24: 计算 AUC**

```python
def compute_auc(y_true, y_pred):
    """计算 AUC"""
    pos_scores = y_pred[y_true == 1]
    neg_scores = y_pred[y_true == 0]

    # 计算正负样本对的比较
    correct_pairs = 0
    total_pairs = 0

    for pos in pos_scores:
        correct_pairs += np.sum(pos > neg_scores)
        total_pairs += len(neg_scores)

    return correct_pairs / total_pairs
```
