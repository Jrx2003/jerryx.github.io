什么是推荐系统

推荐系统（Recommender System）是一种信息过滤系统，旨在预测用户对物品的偏好或评分，从而向用户推荐其可能感兴趣的物品。

推荐系统的核心价值

价值维度说明典型场景
信息过载缓解从海量内容中筛选用户感兴趣的新闻推荐、视频推荐
个性化体验根据用户偏好提供定制化内容电商推荐、音乐推荐
长尾发现帮助用户发现小众但相关的内容图书推荐、电影推荐
商业变现提升点击率、转化率、用户留存广告推荐、商品推荐

推荐系统架构

经典四层架构

用户请求
    ↓
┌─────────────────────────────────────────────────────────┐
│                      召回层 (Recall)                      │
│  目标：从百万/亿级物品中快速筛选千级候选                    │
│  方法：协同过滤、向量召回、热门召回、策略召回               │
│  特点：速度快、粗粒度、高召回率                            │
└─────────────────────────────────────────────────────────┘
    ↓ 千级候选
┌─────────────────────────────────────────────────────────┐
│                     粗排层 (Coarse Rank)                  │
│  目标：快速筛选百级候选，减轻精排压力                       │
│  方法：轻量级模型、双塔模型、LR                             │
│  特点：速度快、中等精度                                    │
└─────────────────────────────────────────────────────────┘
    ↓ 百级候选
┌─────────────────────────────────────────────────────────┐
│                     精排层 (Fine Rank)                    │
│  目标：精准预测点击率/转化率，输出排序分数                  │
│  方法：深度神经网络、Wide&Deep、DeepFM                   │
│  特点：精度高、特征丰富、计算复杂                          │
└─────────────────────────────────────────────────────────┘
    ↓ 排序分数
┌─────────────────────────────────────────────────────────┐
│                     重排层 (Re-rank)                      │
│  目标：业务规则应用、多样性控制、结果展示                   │
│  方法：MMR、DPP、规则过滤、疲劳度控制                      │
│  特点：业务导向、策略灵活                                  │
└─────────────────────────────────────────────────────────┘
    ↓
返回推荐结果

各层详细对比

层级候选数量响应时间模型复杂度主要目标
召回百万→千<50ms低高召回率
粗排千→百<10ms中快速筛选
精排百→十<50ms高精准预测
重排十→N<5ms策略业务满足

推荐算法分类

按算法原理分类

推荐算法
├── 基于内容的推荐 (Content-Based)
│   ├── 物品特征匹配
│   ├── 文本相似度
│   └── 标签匹配
│
├── 协同过滤 (Collaborative Filtering)
│   ├── 基于用户的协同过滤 (UserCF)
│   ├── 基于物品的协同过滤 (ItemCF)
│   └── 矩阵分解 (MF)
│
├── 深度学习推荐
│   ├── 神经网络模型 (DNN)
│   ├── 序列模型 (RNN/Transformer)
│   ├── 图神经网络 (GNN)
│   └── 多任务学习 (MTL)
│
└── 混合推荐 (Hybrid)
    ├── 加权混合
    ├── 切换混合
    └── 特征组合

按应用场景分类

场景类型特点典型算法

指标说明计算公式

数据与特征

特征分类

推荐系统特征
├── 用户特征 (User Features)
│   ├── 人口属性：年龄、性别、地域、职业
│   ├── 行为统计：历史点击率、购买频次、活跃度
│   ├── 兴趣标签：偏好类目、品牌偏好、价格敏感度
│   └── 上下文：设备、网络、时间
│
├── 物品特征 (Item Features)
│   ├── 基础属性：类目、品牌、价格、库存
│   ├── 统计特征：历史CTR、销量、评分
│   ├── 内容特征：标题、图片、描述文本
│   └── 质量特征：商品质量分、店铺等级
│
├── 交叉特征 (Cross Features)
│   ├── 用户-物品：用户对该类目的偏好
│   ├── 用户-上下文：用户在不同时间的行为差异
│   └── 物品-上下文：物品在不同时段的表现
│
└── 序列特征 (Sequence Features)
    ├── 点击序列：最近点击的物品ID序列
    ├── 购买序列：最近购买的物品序列
    └── 行为时间：行为发生的时间间隔

特征工程要点

要点说明示例
归一化将特征缩放到相同范围Min-Max、Z-Score
离散化连续特征分桶年龄分段、价格区间
编码类别特征数值化One-Hot、Embedding
交叉特征组合性别×类目、时段×类目

工业实践要点

推荐系统挑战

挑战问题描述解决思路
冷启动新用户/新物品缺乏历史数据热门推荐、内容特征、迁移学习
数据稀疏用户-物品交互矩阵稀疏矩阵分解、负采样、数据增强
可扩展性海量用户和物品的处理分布式计算、近似算法、向量检索
实时性用户兴趣变化快流式计算、在线学习、实时特征
多样性推荐结果同质化MMR、DPP、探索利用
公平性长尾物品曝光不足重排序策略、流量扶持

  单一目标     多任务学习       生成式推荐
