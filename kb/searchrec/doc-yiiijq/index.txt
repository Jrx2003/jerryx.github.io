## 什么是推荐系统

推荐系统（Recommender System）是一种信息过滤系统，旨在预测用户对物品的偏好或评分，从而向用户推荐其可能感兴趣的物品。

### 推荐系统的核心价值

| 价值维度 | 说明 | 典型场景 |
|----------|------|----------|
| 信息过载缓解 | 从海量内容中筛选用户感兴趣的 | 新闻推荐、视频推荐 |
| 个性化体验 | 根据用户偏好提供定制化内容 | 电商推荐、音乐推荐 |
| 长尾发现 | 帮助用户发现小众但相关的内容 | 图书推荐、电影推荐 |
| 商业变现 | 提升点击率、转化率、用户留存 | 广告推荐、商品推荐 |

---

## 推荐系统架构

### 经典四层架构

```
用户请求
    ↓
┌─────────────────────────────────────────────────────────┐
│                      召回层 (Recall)                      │
│  目标：从百万/亿级物品中快速筛选千级候选                    │
│  方法：协同过滤、向量召回、热门召回、策略召回               │
│  特点：速度快、粗粒度、高召回率                            │
└─────────────────────────────────────────────────────────┘
    ↓ 千级候选
┌─────────────────────────────────────────────────────────┐
│                     粗排层 (Coarse Rank)                  │
│  目标：快速筛选百级候选，减轻精排压力                       │
│  方法：轻量级模型、双塔模型、LR                             │
│  特点：速度快、中等精度                                    │
└─────────────────────────────────────────────────────────┘
    ↓ 百级候选
┌─────────────────────────────────────────────────────────┐
│                     精排层 (Fine Rank)                    │
│  目标：精准预测点击率/转化率，输出排序分数                  │
│  方法：深度神经网络、Wide&Deep、DeepFM                   │
│  特点：精度高、特征丰富、计算复杂                          │
└─────────────────────────────────────────────────────────┘
    ↓ 排序分数
┌─────────────────────────────────────────────────────────┐
│                     重排层 (Re-rank)                      │
│  目标：业务规则应用、多样性控制、结果展示                   │
│  方法：MMR、DPP、规则过滤、疲劳度控制                      │
│  特点：业务导向、策略灵活                                  │
└─────────────────────────────────────────────────────────┘
    ↓
返回推荐结果
```

### 各层详细对比

| 层级 | 候选数量 | 响应时间 | 模型复杂度 | 主要目标 |
|------|----------|----------|------------|----------|
| 召回 | 百万→千 | <50ms | 低 | 高召回率 |
| 粗排 | 千→百 | <10ms | 中 | 快速筛选 |
| 精排 | 百→十 | <50ms | 高 | 精准预测 |
| 重排 | 十→N | <5ms | 策略 | 业务满足 |

## 推荐算法分类

### 按算法原理分类

```
推荐算法
├── 基于内容的推荐 (Content-Based)
│   ├── 物品特征匹配
│   ├── 文本相似度
│   └── 标签匹配
│
├── 协同过滤 (Collaborative Filtering)
│   ├── 基于用户的协同过滤 (UserCF)
│   ├── 基于物品的协同过滤 (ItemCF)
│   └── 矩阵分解 (MF)
│
├── 深度学习推荐
│   ├── 神经网络模型 (DNN)
│   ├── 序列模型 (RNN/Transformer)
│   ├── 图神经网络 (GNN)
│   └── 多任务学习 (MTL)
│
└── 混合推荐 (Hybrid)
    ├── 加权混合
    ├── 切换混合
    └── 特征组合
```

### 按应用场景分类

| 场景类型 | 特点 | 典型算法 |
|----------|------|----------|
| 评分预测 | 预测用户对物品的评分 | SVD、NMF |
| Top-N推荐 | 推荐最可能喜欢的N个物品 | 各种排序模型 |
| 序列推荐 | 基于行为序列预测下一个 | GRU4Rec、SASRec |
| 会话推荐 | 匿名用户的短期行为 | NARM、SR-GNN |

## 评估指标体系

### 离线评估指标

#### 准确率指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **RMSE** | $\sqrt{\frac{1}{N}\sum_{i=1}^N (y_i - \hat{y}_i)^2}$ | 均方根误差，衡量评分预测准确度 |
| **MAE** | $\frac{1}{N}\sum_{i=1}^N \|y_i - \hat{y}_i\|$ | 平均绝对误差，更鲁棒 |
| **AUC** | $\frac{\sum_{i \in P} \text{rank}_i - \frac{|P|(|P|+1)}{2}}{|P| \times |N|}$ | ROC曲线下面积，衡量排序能力 |
| **LogLoss** | $-\frac{1}{N}\sum_{i=1}^N [y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$ | 对数损失，CTR预估常用 |

#### 排序指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **Precision@K** | $\frac{\text{相关物品在前K个中的数量}}{K}$ | 前K个推荐中相关物品比例 |
| **Recall@K** | $\frac{\text{相关物品在前K个中的数量}}{\text{用户相关的总物品数}}$ | 相关物品被召回的比例 |
| **F1@K** | $\frac{2 \times \text{Precision@K} \times \text{Recall@K}}{\text{Precision@K} + \text{Recall@K}}$ | 精确率和召回率的调和平均 |
| **MAP** | $\frac{1}{|U|}\sum_{u=1}^{|U|} \frac{1}{m_u} \sum_{k=1}^{m_u} \text{Precision@k}$ | 平均精度均值 |
| **NDCG@K** | $\frac{DCG@K}{IDCG@K}$，其中$DCG = \sum_{i=1}^K \frac{2^{rel_i}-1}{\log_2(i+1)}$ | 归一化折损累计增益 |
| **MRR** | $\frac{1}{|Q|}\sum_{i=1}^{|Q|} \frac{1}{\text{rank}_i}$ | 平均倒数排名 |

#### 多样性指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **Intra-List Diversity** | $\frac{1}{|R|(|R|-1)} \sum_{i \in R} \sum_{j \in R, j \neq i} (1 - \text{sim}(i,j))$ | 推荐列表内物品间不相似度 |
| **Coverage** | $\frac{|\text{被推荐过的物品}|}{|\text{总物品数}|}$ | 长尾物品覆盖度 |

### 在线评估指标

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| **CTR** | 点击率 | $\frac{\text{点击次数}}{\text{曝光次数}} \times 100\%$ |
| **CVR** | 转化率 | $\frac{\text{转化次数}}{\text{点击次数}} \times 100\%$ |
| **GMV** | 成交总额 | $\sum \text{单价} \times \text{销量}$ |
| **人均点击数** | 用户平均点击推荐物品数 | $\frac{\text{总点击数}}{\text{用户数}}$ |
| **人均阅读时长** | 内容类推荐指标 | $\frac{\text{总阅读时长}}{\text{用户数}}$ |
| **留存率** | 次日/7日/30日留存 | $\frac{\text{留存用户数}}{\text{总用户数}}$ |

## 数据与特征

### 特征分类

```
推荐系统特征
├── 用户特征 (User Features)
│   ├── 人口属性：年龄、性别、地域、职业
│   ├── 行为统计：历史点击率、购买频次、活跃度
│   ├── 兴趣标签：偏好类目、品牌偏好、价格敏感度
│   └── 上下文：设备、网络、时间
│
├── 物品特征 (Item Features)
│   ├── 基础属性：类目、品牌、价格、库存
│   ├── 统计特征：历史CTR、销量、评分
│   ├── 内容特征：标题、图片、描述文本
│   └── 质量特征：商品质量分、店铺等级
│
├── 交叉特征 (Cross Features)
│   ├── 用户-物品：用户对该类目的偏好
│   ├── 用户-上下文：用户在不同时间的行为差异
│   └── 物品-上下文：物品在不同时段的表现
│
└── 序列特征 (Sequence Features)
    ├── 点击序列：最近点击的物品ID序列
    ├── 购买序列：最近购买的物品序列
    └── 行为时间：行为发生的时间间隔
```

### 特征工程要点

| 要点 | 说明 | 示例 |
|------|------|------|
| 归一化 | 将特征缩放到相同范围 | Min-Max、Z-Score |
| 离散化 | 连续特征分桶 | 年龄分段、价格区间 |
| 编码 | 类别特征数值化 | One-Hot、Embedding |
| 交叉 | 特征组合 | 性别×类目、时段×类目 |

## 工业实践要点

### 推荐系统挑战

| 挑战 | 问题描述 | 解决思路 |
|------|----------|----------|
| **冷启动** | 新用户/新物品缺乏历史数据 | 热门推荐、内容特征、迁移学习 |
| **数据稀疏** | 用户-物品交互矩阵稀疏 | 矩阵分解、负采样、数据增强 |
| **可扩展性** | 海量用户和物品的处理 | 分布式计算、近似算法、向量检索 |
| **实时性** | 用户兴趣变化快 | 流式计算、在线学习、实时特征 |
| **多样性** | 推荐结果同质化 | MMR、DPP、探索利用 |
| **公平性** | 长尾物品曝光不足 | 重排序策略、流量扶持 |

### 推荐系统演进趋势

```
传统推荐 → 深度学习推荐 → 大模型推荐
   ↓            ↓              ↓
 规则/统计    神经网络        预训练+提示
  特征工程     自动特征学习     语义理解
  单一目标     多任务学习       生成式推荐
```

│  特点：速度快、粗粒度、高召回率                            │
└─────────────────────────────────────────────────────────┘
    ↓ 千级候选
┌─────────────────────────────────────────────────────────┐
│                     粗排层 (Coarse Rank)                  │
│  目标：快速筛选百级候选，减轻精排压力                       │
│  方法：轻量级模型、双塔模型、LR                             │
│  特点：速度快、中等精度                                    │
└─────────────────────────────────────────────────────────┘
    ↓ 百级候选
┌─────────────────────────────────────────────────────────┐
│                     精排层 (Fine Rank)                    │
│  目标：精准预测点击率/转化率，输出排序分数                  │
│  方法：深度神经网络、Wide&Deep、DeepFM                   │
│  特点：精度高、特征丰富、计算复杂                          │
└─────────────────────────────────────────────────────────┘
    ↓ 排序分数
┌─────────────────────────────────────────────────────────┐
│                     重排层 (Re-rank)                      │
│  目标：业务规则应用、多样性控制、结果展示                   │
│  方法：MMR、DPP、规则过滤、疲劳度控制                      │
│  特点：业务导向、策略灵活                                  │
└─────────────────────────────────────────────────────────┘
    ↓
返回推荐结果
```

### 各层详细对比

| 层级 | 候选数量 | 响应时间 | 模型复杂度 | 主要目标 |
|------|----------|----------|------------|----------|
| 召回 | 百万→千 | <50ms | 低 | 高召回率 |
| 粗排 | 千→百 | <10ms | 中 | 快速筛选 |
| 精排 | 百→十 | <50ms | 高 | 精准预测 |
| 重排 | 十→N | <5ms | 策略 | 业务满足 |

## 推荐算法分类

### 按算法原理分类

```
推荐算法
├── 基于内容的推荐 (Content-Based)
│   ├── 物品特征匹配
│   ├── 文本相似度
│   └── 标签匹配
│
├── 协同过滤 (Collaborative Filtering)
│   ├── 基于用户的协同过滤 (UserCF)
│   ├── 基于物品的协同过滤 (ItemCF)
│   └── 矩阵分解 (MF)
│
├── 深度学习推荐
│   ├── 神经网络模型 (DNN)
│   ├── 序列模型 (RNN/Transformer)
│   ├── 图神经网络 (GNN)
│   └── 多任务学习 (MTL)
│
└── 混合推荐 (Hybrid)
    ├── 加权混合
    ├── 切换混合
    └── 特征组合
```

### 按应用场景分类

| 场景类型 | 特点 | 典型算法 |
|----------|------|----------|
| 评分预测 | 预测用户对物品的评分 | SVD、NMF |
| Top-N推荐 | 推荐最可能喜欢的N个物品 | 各种排序模型 |
| 序列推荐 | 基于行为序列预测下一个 | GRU4Rec、SASRec |
| 会话推荐 | 匿名用户的短期行为 | NARM、SR-GNN |

## 评估指标体系

### 离线评估指标

#### 准确率指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **RMSE** | $\sqrt{\frac{1}{N}\sum_{i=1}^N (y_i - \hat{y}_i)^2}$ | 均方根误差，衡量评分预测准确度 |
| **MAE** | $\frac{1}{N}\sum_{i=1}^N \|y_i - \hat{y}_i\|$ | 平均绝对误差，更鲁棒 |
| **AUC** | $\frac{\sum_{i \in P} \text{rank}_i - \frac{|P|(|P|+1)}{2}}{|P| \times |N|}$ | ROC曲线下面积，衡量排序能力 |
| **LogLoss** | $-\frac{1}{N}\sum_{i=1}^N [y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]$ | 对数损失，CTR预估常用 |

#### 排序指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **Precision@K** | $\frac{\text{相关物品在前K个中的数量}}{K}$ | 前K个推荐中相关物品比例 |
| **Recall@K** | $\frac{\text{相关物品在前K个中的数量}}{\text{用户相关的总物品数}}$ | 相关物品被召回的比例 |
| **F1@K** | $\frac{2 \times \text{Precision@K} \times \text{Recall@K}}{\text{Precision@K} + \text{Recall@K}}$ | 精确率和召回率的调和平均 |
| **MAP** | $\frac{1}{|U|}\sum_{u=1}^{|U|} \frac{1}{m_u} \sum_{k=1}^{m_u} \text{Precision@k}$ | 平均精度均值 |
| **NDCG@K** | $\frac{DCG@K}{IDCG@K}$，其中$DCG = \sum_{i=1}^K \frac{2^{rel_i}-1}{\log_2(i+1)}$ | 归一化折损累计增益 |
| **MRR** | $\frac{1}{|Q|}\sum_{i=1}^{|Q|} \frac{1}{\text{rank}_i}$ | 平均倒数排名 |

#### 多样性指标

| 指标 | 公式 | 说明 |
|------|------|------|
| **Intra-List Diversity** | $\frac{1}{|R|(|R|-1)} \sum_{i \in R} \sum_{j \in R, j \neq i} (1 - \text{sim}(i,j))$ | 推荐列表内物品间不相似度 |
| **Coverage** | $\frac{|\text{被推荐过的物品}|}{|\text{总物品数}|}$ | 长尾物品覆盖度 |

### 在线评估指标

| 指标 | 说明 | 计算公式 |
|------|------|----------|
| **CTR** | 点击率 | $\frac{\text{点击次数}}{\text{曝光次数}} \times 100\%$ |
| **CVR** | 转化率 | $\frac{\text{转化次数}}{\text{点击次数}} \times 100\%$ |
| **GMV** | 成交总额 | $\sum \text{单价} \times \text{销量}$ |
| **人均点击数** | 用户平均点击推荐物品数 | $\frac{\text{总点击数}}{\text{用户数}}$ |
| **人均阅读时长** | 内容类推荐指标 | $\frac{\text{总阅读时长}}{\text{用户数}}$ |
| **留存率** | 次日/7日/30日留存 | $\frac{\text{留存用户数}}{\text{总用户数}}$ |

## 数据与特征

### 特征分类

```
推荐系统特征
├── 用户特征 (User Features)
│   ├── 人口属性：年龄、性别、地域、职业
│   ├── 行为统计：历史点击率、购买频次、活跃度
│   ├── 兴趣标签：偏好类目、品牌偏好、价格敏感度
│   └── 上下文：设备、网络、时间
│
├── 物品特征 (Item Features)
│   ├── 基础属性：类目、品牌、价格、库存
│   ├── 统计特征：历史CTR、销量、评分
│   ├── 内容特征：标题、图片、描述文本
│   └── 质量特征：商品质量分、店铺等级
│
├── 交叉特征 (Cross Features)
│   ├── 用户-物品：用户对该类目的偏好
│   ├── 用户-上下文：用户在不同时间的行为差异
│   └── 物品-上下文：物品在不同时段的表现
│
└── 序列特征 (Sequence Features)
    ├── 点击序列：最近点击的物品ID序列
    ├── 购买序列：最近购买的物品序列
    └── 行为时间：行为发生的时间间隔
```

### 特征工程要点

| 要点 | 说明 | 示例 |
|------|------|------|
| 归一化 | 将特征缩放到相同范围 | Min-Max、Z-Score |
| 离散化 | 连续特征分桶 | 年龄分段、价格区间 |
| 编码 | 类别特征数值化 | One-Hot、Embedding |
| 交叉 | 特征组合 | 性别×类目、时段×类目 |

## 工业实践要点

### 推荐系统挑战

| 挑战 | 问题描述 | 解决思路 |
|------|----------|----------|
| **冷启动** | 新用户/新物品缺乏历史数据 | 热门推荐、内容特征、迁移学习 |
| **数据稀疏** | 用户-物品交互矩阵稀疏 | 矩阵分解、负采样、数据增强 |
| **可扩展性** | 海量用户和物品的处理 | 分布式计算、近似算法、向量检索 |
| **实时性** | 用户兴趣变化快 | 流式计算、在线学习、实时特征 |
| **多样性** | 推荐结果同质化 | MMR、DPP、探索利用 |
| **公平性** | 长尾物品曝光不足 | 重排序策略、流量扶持 |

### 推荐系统演进趋势

```
传统推荐 → 深度学习推荐 → 大模型推荐
   ↓            ↓              ↓
 规则/统计    神经网络        预训练+提示
  特征工程     自动特征学习     语义理解
  单一目标     多任务学习       生成式推荐
```
