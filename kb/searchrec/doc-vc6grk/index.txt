## 一、大模型推荐 (LLM4Rec)

### 核心思想

利用大语言模型的语义理解和生成能力，革新传统推荐系统。

```
传统推荐:
用户ID → Embedding → MLP → 物品ID → 评分

大模型推荐:
用户历史行为(文本) → LLM → 语义理解 → 生成推荐解释/物品
```

### 三种范式

| 范式 | 说明 | 代表工作 |
|------|------|----------|
| **LLM-as-Embedding** | 用LLM生成用户/物品Embedding | U-BERT, I-BERT |
| **LLM-as-Ranker** | 用LLM做排序打分 | GPT4Rec, RankGPT |
| **LLM-as-Generator** | 用LLM直接生成推荐 | RecGPT, Chat-Rec |

### 应用场景

| 场景 | 传统方法 | LLM方法 | 优势 |
|------|----------|---------|------|
| 推荐理由生成 | 模板填充 | LLM生成 | 更自然、个性化 |
| 跨域推荐 | 迁移学习 | 语义对齐 | 零样本能力 |
| 对话式推荐 | 规则匹配 | 多轮对话 | 交互体验好 |
| 解释性推荐 | 特征归因 | 自然语言解释 | 可解释性强 |

---

## 二、生成式推荐 (Generative Recommendation)

### 核心思想

将推荐视为序列生成任务，预测下一个物品。

```
序列推荐传统方法:
用户历史: [A, B, C] → 模型 → 预测 D

生成式推荐:
用户历史: "用户购买了A，浏览了B，收藏了C"
→ LLM → "推荐给该用户D，因为..."
```

### 技术路线

| 路线 | 方法 | 特点 |
|------|------|------|
| **Seq2Seq** | T5, BART | 编码历史，解码推荐 |
| **自回归** | GPT | 直接生成物品ID序列 |
| **扩散模型** | Diffusion | 生成Embedding，再检索 |

### 代表模型

| 模型 | 机构 | 核心创新 |
|------|------|----------|
| **P5** | UCSD | 统一范式：提示学习 |
| **M6-Rec** | 阿里 | 多任务预训练 |
| **VIP5** | 中科大 | 视觉增强提示 |
| **Chat-Rec** | 微软 | 对话式推荐 |

---

## 三、多模态推荐

### 核心思想

融合文本、图像、视频、音频等多模态信息进行推荐。

```
传统推荐特征:
- 用户ID、物品ID
- 类别、标签
- 数值特征

多模态推荐特征:
- 商品图片 (CNN/ViT编码)
- 商品描述 (BERT编码)
- 视频内容 (Video Swin编码)
- 用户评论 (Text编码)
```

### 技术架构

| 组件 | 方法 | 用途 |
|------|------|------|
| **视觉编码** | CLIP, ViT | 商品图片理解 |
| **文本编码** | BERT, RoBERTa | 标题、描述、评论 |
| **融合层** | Attention, Gating | 多模态特征融合 |
| **推荐层** | MLP, Transformer | 最终排序 |

### 代表模型

| 模型 | 模态 | 场景 |
|------|------|------|
| **VBPR** | 图像+ID | 时尚推荐 |
| **MMGCN** | 图+多模态 | 社交推荐 |
| **MKGAT** | 知识图谱+多模态 | 知识增强 |

---

## 四、因果推荐 (Causal Recommendation)

### 核心思想

从因果推断角度理解推荐，解决偏差问题。

```
传统关联学习:
用户看A后看B → 推荐B
（可能存在选择偏差）

因果学习:
用户看A → 真正导致 → 想看B
（消除混杂因素）
```

### 关键问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **选择偏差** | 用户只看到部分物品 | IPS, 因果推断 |
| **流行度偏差** | 热门物品过度推荐 | 因果去偏 |
| **位置偏差** | 排名靠前物品被点击多 | 因果位置模型 |
| **曝光偏差** | 未曝光物品无法被推荐 | 反事实推理 |

### 方法分类

| 方法 | 核心思想 |
|------|----------|
| **IPW (逆概率加权)** | 对稀有样本加权 |
| **DR (双重稳健)** | 结合模型和IPW |
| **后门调整** | 控制混杂变量 |
| **反事实推理** | 预测未发生的情况 |

---

## 五、跨域推荐 (Cross-Domain Recommendation)

### 核心思想

利用源域数据帮助目标域推荐。

```
源域: 电影推荐 (数据丰富)
目标域: 图书推荐 (数据稀疏)

迁移: 电影-图书的相似性知识
```

### 技术方法

| 方法 | 说明 | 代表 |
|------|------|------|
| **基于映射** | 学习跨域映射函数 | CMF, CDCF |
| **基于共享** | 共享部分参数 | CoNet, RC-DFM |
| **基于元学习** | 快速适应新域 | MAML4CDR |
| **基于图** | 跨域图神经网络 | GA-DTCDR |

---

## 六、去偏与公平性

### 推荐系统中的偏差

| 偏差类型 | 描述 | 影响 |
|----------|------|------|
| **选择偏差** | 用户只能看到部分物品 | 长尾物品无法曝光 |
| **位置偏差** | 用户倾向点击排名靠前物品 | 评估失真 |
| **曝光偏差** | 未曝光物品无法被推荐 | 马太效应 |
| **流行度偏差** | 热门物品被过度推荐 | 多样性降低 |

### 去偏方法

| 方法 | 原理 | 应用 |
|------|------|------|
| **IPS** | 逆倾向评分 | 消除选择偏差 |
| **DR** | 双重稳健估计 | 结合预测和IPS |
| **因果推断** | 建模因果关系 | 根本性解决 |
| **对抗学习** | 学习不变表示 | 公平性保障 |

## 八、面试要点

### 高频问题

**Q1: LLM4Rec vs 传统推荐的核心区别？**

A:
1. **特征表示**: LLM用语义理解，传统用ID Embedding
2. **冷启动**: LLM利用预训练知识，传统需要数据积累
3. **可解释性**: LLM可生成自然语言解释
4. **跨域能力**: LLM有零样本迁移能力

**Q2: 生成式推荐的优缺点？**

A:
**优点**:
- 统一的生成框架
- 自然语言交互
- 可解释性强

**缺点**:
- 推理成本高
- 幻觉问题
- 实时性挑战

**Q3: 因果推荐为什么重要？**

A:
1. **根本性解决偏差**: 不是缓解症状，而是消除病因
2. **更好泛化**: 因果模型对分布变化更鲁棒
3. **可解释性**: 理解真正的因果关系

**Q4: 多模态推荐的关键技术？**

A:
1. **对齐**: 不同模态的特征对齐
2. **融合**: 多模态信息有效融合
3. **缺失处理**: 部分模态缺失时的处理

**Q5: 跨域推荐的主要挑战？**

A:
1. **域间差异**: 不同域的分布差异大
2. **负迁移**: 源域知识可能损害目标域
3. **可迁移性**: 哪些知识可以迁移

生成式推荐:
用户历史: "用户购买了A，浏览了B，收藏了C"
→ LLM → "推荐给该用户D，因为..."
```

### 技术路线

| 路线 | 方法 | 特点 |
|------|------|------|
| **Seq2Seq** | T5, BART | 编码历史，解码推荐 |
| **自回归** | GPT | 直接生成物品ID序列 |
| **扩散模型** | Diffusion | 生成Embedding，再检索 |

### 代表模型

| 模型 | 机构 | 核心创新 |
|------|------|----------|
| **P5** | UCSD | 统一范式：提示学习 |
| **M6-Rec** | 阿里 | 多任务预训练 |
| **VIP5** | 中科大 | 视觉增强提示 |
| **Chat-Rec** | 微软 | 对话式推荐 |

---

## 三、多模态推荐

### 核心思想

融合文本、图像、视频、音频等多模态信息进行推荐。

```
传统推荐特征:
- 用户ID、物品ID
- 类别、标签
- 数值特征

多模态推荐特征:
- 商品图片 (CNN/ViT编码)
- 商品描述 (BERT编码)
- 视频内容 (Video Swin编码)
- 用户评论 (Text编码)
```

### 技术架构

| 组件 | 方法 | 用途 |
|------|------|------|
| **视觉编码** | CLIP, ViT | 商品图片理解 |
| **文本编码** | BERT, RoBERTa | 标题、描述、评论 |
| **融合层** | Attention, Gating | 多模态特征融合 |
| **推荐层** | MLP, Transformer | 最终排序 |

### 代表模型

| 模型 | 模态 | 场景 |
|------|------|------|
| **VBPR** | 图像+ID | 时尚推荐 |
| **MMGCN** | 图+多模态 | 社交推荐 |
| **MKGAT** | 知识图谱+多模态 | 知识增强 |

## 四、因果推荐 (Causal Recommendation)

### 核心思想

从因果推断角度理解推荐，解决偏差问题。

```
传统关联学习:
用户看A后看B → 推荐B
（可能存在选择偏差）

因果学习:
用户看A → 真正导致 → 想看B
（消除混杂因素）
```

### 关键问题

| 问题 | 说明 | 解决方案 |
|------|------|----------|
| **选择偏差** | 用户只看到部分物品 | IPS, 因果推断 |
| **流行度偏差** | 热门物品过度推荐 | 因果去偏 |
| **位置偏差** | 排名靠前物品被点击多 | 因果位置模型 |
| **曝光偏差** | 未曝光物品无法被推荐 | 反事实推理 |

### 方法分类

| 方法 | 核心思想 |
|------|----------|
| **IPW (逆概率加权)** | 对稀有样本加权 |
| **DR (双重稳健)** | 结合模型和IPW |
| **后门调整** | 控制混杂变量 |
| **反事实推理** | 预测未发生的情况 |

---

## 五、跨域推荐 (Cross-Domain Recommendation)

### 核心思想

利用源域数据帮助目标域推荐。

```
源域: 电影推荐 (数据丰富)
目标域: 图书推荐 (数据稀疏)

迁移: 电影-图书的相似性知识
```

### 技术方法

| 方法 | 说明 | 代表 |
|------|------|------|
| **基于映射** | 学习跨域映射函数 | CMF, CDCF |
| **基于共享** | 共享部分参数 | CoNet, RC-DFM |
| **基于元学习** | 快速适应新域 | MAML4CDR |
| **基于图** | 跨域图神经网络 | GA-DTCDR |

## 六、去偏与公平性

### 推荐系统中的偏差

| 偏差类型 | 描述 | 影响 |
|----------|------|------|
| **选择偏差** | 用户只能看到部分物品 | 长尾物品无法曝光 |
| **位置偏差** | 用户倾向点击排名靠前物品 | 评估失真 |
| **曝光偏差** | 未曝光物品无法被推荐 | 马太效应 |
| **流行度偏差** | 热门物品被过度推荐 | 多样性降低 |

### 去偏方法

| 方法 | 原理 | 应用 |
|------|------|------|
| **IPS** | 逆倾向评分 | 消除选择偏差 |
| **DR** | 双重稳健估计 | 结合预测和IPS |
| **因果推断** | 建模因果关系 | 根本性解决 |
| **对抗学习** | 学习不变表示 | 公平性保障 |

## 八、面试要点

### 高频问题

**Q1: LLM4Rec vs 传统推荐的核心区别？**

A:
1. **特征表示**: LLM用语义理解，传统用ID Embedding
2. **冷启动**: LLM利用预训练知识，传统需要数据积累
3. **可解释性**: LLM可生成自然语言解释
4. **跨域能力**: LLM有零样本迁移能力

**Q2: 生成式推荐的优缺点？**

A:
**优点**:
- 统一的生成框架
- 自然语言交互
- 可解释性强

**缺点**:
- 推理成本高
- 幻觉问题
- 实时性挑战

**Q3: 因果推荐为什么重要？**

A:
1. **根本性解决偏差**: 不是缓解症状，而是消除病因
2. **更好泛化**: 因果模型对分布变化更鲁棒
3. **可解释性**: 理解真正的因果关系

**Q4: 多模态推荐的关键技术？**

A:
1. **对齐**: 不同模态的特征对齐
2. **融合**: 多模态信息有效融合
3. **缺失处理**: 部分模态缺失时的处理

**Q5: 跨域推荐的主要挑战？**

A:
1. **域间差异**: 不同域的分布差异大
2. **负迁移**: 源域知识可能损害目标域
3. **可迁移性**: 哪些知识可以迁移
