2024-2025搜推前沿趋势

2024-2025 搜推前沿趋势

更新日期：2026-03-13 相关文档：LLM4Rec大模型推荐 | 生成式推荐 | 因果推荐

一、大模型推荐 (LLM4Rec)

核心思想

利用大语言模型的语义理解和生成能力，革新传统推荐系统。

传统推荐:
用户ID → Embedding → MLP → 物品ID → 评分

大模型推荐:
用户历史行为(文本) → LLM → 语义理解 → 生成推荐解释/物品

三种范式

范式说明代表工作
LLM-as-Embedding用LLM生成用户/物品EmbeddingU-BERT, I-BERT
LLM-as-Ranker用LLM做排序打分GPT4Rec, RankGPT
LLM-as-Generator用LLM直接生成推荐RecGPT, Chat-Rec

应用场景

场景传统方法LLM方法优势
推荐理由生成模板填充LLM生成更自然、个性化
跨域推荐迁移学习语义对齐零样本能力
对话式推荐规则匹配多轮对话交互体验好
解释性推荐特征归因自然语言解释可解释性强

二、生成式推荐 (Generative Recommendation)

核心思想

将推荐视为序列生成任务，预测下一个物品。

序列推荐传统方法:
用户历史: [A, B, C] → 模型 → 预测 D

生成式推荐:
用户历史: "用户购买了A，浏览了B，收藏了C"
→ LLM → "推荐给该用户D，因为..."

技术路线

路线方法特点
Seq2SeqT5, BART编码历史，解码推荐
自回归GPT直接生成物品ID序列
扩散模型Diffusion生成Embedding，再检索

代表模型

模型机构核心创新
P5UCSD统一范式：提示学习
M6-Rec阿里多任务预训练
VIP5中科大视觉增强提示
Chat-Rec微软对话式推荐

三、多模态推荐

核心思想

融合文本、图像、视频、音频等多模态信息进行推荐。

传统推荐特征:
- 用户ID、物品ID
- 类别、标签
- 数值特征

多模态推荐特征:
- 商品图片 (CNN/ViT编码)
- 商品描述 (BERT编码)
- 视频内容 (Video Swin编码)
- 用户评论 (Text编码)

技术架构

组件方法用途
视觉编码CLIP, ViT商品图片理解
文本编码BERT, RoBERTa标题、描述、评论
融合层Attention, Gating多模态特征融合
推荐层MLP, Transformer最终排序

代表模型

模型模态场景
VBPR图像+ID时尚推荐
MMGCN图+多模态社交推荐
MKGAT知识图谱+多模态知识增强

四、因果推荐 (Causal Recommendation)

核心思想

从因果推断角度理解推荐，解决偏差问题。

传统关联学习:
用户看A后看B → 推荐B
（可能存在选择偏差）

因果学习:
用户看A → 真正导致 → 想看B
（消除混杂因素）

关键问题

问题说明解决方案
选择偏差用户只看到部分物品IPS, 因果推断
流行度偏差热门物品过度推荐因果去偏
位置偏差排名靠前物品被点击多因果位置模型
曝光偏差未曝光物品无法被推荐反事实推理

方法分类

方法核心思想
IPW (逆概率加权)对稀有样本加权
DR (双重稳健)结合模型和IPW
后门调整控制混杂变量
反事实推理预测未发生的情况

五、跨域推荐 (Cross-Domain Recommendation)

核心思想

利用源域数据帮助目标域推荐。

源域: 电影推荐 (数据丰富)
目标域: 图书推荐 (数据稀疏)

迁移: 电影-图书的相似性知识

技术方法

方法说明代表
基于映射学习跨域映射函数CMF, CDCF
基于共享共享部分参数CoNet, RC-DFM
基于元学习快速适应新域MAML4CDR
基于图跨域图神经网络GA-DTCDR

六、去偏与公平性

推荐系统中的偏差

偏差类型描述影响
选择偏差用户只能看到部分物品长尾物品无法曝光
位置偏差用户倾向点击排名靠前物品评估失真
曝光偏差未曝光物品无法被推荐马太效应
流行度偏差热门物品被过度推荐多样性降低

去偏方法

方法原理应用
IPS逆倾向评分消除选择偏差
DR双重稳健估计结合预测和IPS
因果推断建模因果关系根本性解决
对抗学习学习不变表示公平性保障

八、面试要点

高频问题

Q1: LLM4Rec vs 传统推荐的核心区别？

A:

  1. 特征表示: LLM用语义理解，传统用ID Embedding
  2. 冷启动: LLM利用预训练知识，传统需要数据积累
  3. 可解释性: LLM可生成自然语言解释
  4. 跨域能力: LLM有零样本迁移能力

Q2: 生成式推荐的优缺点？

A: 优点:

  • 统一的生成框架
  • 自然语言交互
  • 可解释性强

缺点:

  • 推理成本高
  • 幻觉问题
  • 实时性挑战

Q3: 因果推荐为什么重要？

A:

  1. 根本性解决偏差: 不是缓解症状，而是消除病因
  2. 更好泛化: 因果模型对分布变化更鲁棒
  3. 可解释性: 理解真正的因果关系

Q4: 多模态推荐的关键技术？

A:

  1. 对齐: 不同模态的特征对齐
  2. 融合: 多模态信息有效融合
  3. 缺失处理: 部分模态缺失时的处理

Q5: 跨域推荐的主要挑战？

A:

  1. 域间差异: 不同域的分布差异大
  2. 负迁移: 源域知识可能损害目标域
  3. 可迁移性: 哪些知识可以迁移
