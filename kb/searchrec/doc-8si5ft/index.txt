## 召回层概述

召回层（Recall Layer）是推荐系统的第一阶段，目标是从海量物品库中快速筛选出一小部分候选物品。

### 召回层特点

| 特点 | 说明 |
|------|------|
| **速度快** | 通常要求在50ms以内返回结果 |
| **粗粒度** | 使用简单模型或规则快速筛选 |
| **高召回率** | 确保相关物品不被漏掉 |
| **候选量大** | 通常召回数百到数千个物品 |

### 召回层架构

```
用户请求
    ↓
┌────────────────────────────────────────────────────────────┐
│                      多路召回                               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐          │
│  │ 协同过滤 │ │ 向量召回 │ │ 热门召回 │ │ 策略召回 │          │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘          │
│       └───────────┴───────────┴───────────┘                │
│                         ↓                                  │
│                    候选物品池 (千级)                         │
└────────────────────────────────────────────────────────────┘
```

## 多路召回策略

### 1. 协同过滤召回

基于用户历史行为，通过 UserCF 或 ItemCF 召回相似物品。

```python
def cf_recall(user_id, top_k=100):
    """基于ItemCF的召回"""
    user_history = get_user_history(user_id)
    candidate_items = set()

    for item in user_history:
        # 获取相似物品
        similar_items = item_sim_matrix[item][:50]
        candidate_items.update([i[0] for i in similar_items])

    # 过滤已交互物品
    candidate_items -= user_history

    return list(candidate_items)[:top_k]
```

### 2. 向量召回（Embedding Recall）

将用户和物品表示为向量，通过向量相似度召回。

#### 双塔模型

```
用户特征                    物品特征
    ↓                          ↓
┌─────────┐                ┌─────────┐
│  User   │                │  Item   │
│  Tower  │                │  Tower  │
│ (DNN)   │                │ (DNN)   │
└────┬────┘                └────┬────┘
     ↓                          ↓
  User Vector (64d)       Item Vector (64d)
     ↓                          ↓
     └──────────┬───────────────┘
                ↓
          Cosine Similarity
```

#### 向量检索

```python
import faiss

# 构建 Faiss 索引
index = faiss.IndexFlatIP(embedding_dim)  # 内积索引
index.add(item_embeddings)  # 添加物品向量

# 向量召回
def vector_recall(user_vector, top_k=100):
    scores, item_ids = index.search(user_vector.reshape(1, -1), top_k)
    return item_ids[0]
```

### 3. 热门召回

基于全局热度召回，解决冷启动问题。

```python
def hot_recall(user_id, top_k=50):
    """基于物品热度召回"""
    # 考虑用户已交互的类目
    user_categories = get_user_preferred_categories(user_id)

    # 在偏好类目中按热度排序
    candidates = []
    for category in user_categories:
        hot_items = get_hot_items_by_category(category, top_k=20)
        candidates.extend(hot_items)

    return candidates
```

### 4. 策略召回

基于业务规则的召回策略。

| 策略类型 | 说明 | 示例 |
|----------|------|------|
| 新品召回 | 新上架物品曝光 | 新品加权 |
| 运营召回 | 运营活动物品 | 促销商品 |
| 地域召回 | 基于地理位置 | 附近商家 |
| 时间召回 | 基于时间特征 | 早餐推荐 |

## 向量召回详解

### 双塔模型训练

```python
import torch
import torch.nn as nn

class TwoTowerModel(nn.Module):
    def __init__(self, user_feature_dim, item_feature_dim, embedding_dim=64):
        super().__init__()

        # User Tower
        self.user_tower = nn.Sequential(
            nn.Linear(user_feature_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, embedding_dim)
        )

        # Item Tower
        self.item_tower = nn.Sequential(
            nn.Linear(item_feature_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, embedding_dim)
        )

    def forward(self, user_features, item_features):
        user_embedding = self.user_tower(user_features)
        item_embedding = self.item_tower(item_features)

        # 归一化
        user_embedding = F.normalize(user_embedding, p=2, dim=1)
        item_embedding = F.normalize(item_embedding, p=2, dim=1)

        # 计算相似度
        similarity = torch.sum(user_embedding * item_embedding, dim=1)
        return similarity
```

### 采样策略

| 采样方法 | 说明 | 适用场景 |
|----------|------|----------|
| **Batch内负采样** | 同batch其他样本作为负样本 | 简单高效 |
| **全局负采样** | 从全库随机采样 | 需要全库信息 |
| **困难负采样** | 采样与正样本相似的负样本 | 提升模型区分能力 |

### 向量索引技术

| 索引类型 | 特点 | 适用规模 |
|----------|------|----------|
| **Flat (暴力搜索)** | 精确、慢 | 百万级以下 |
| **IVF (倒排文件)** | 近似、快 | 千万级 |
| **HNSW (图索引)** | 高召回、较快 | 亿级 |
| **PQ (乘积量化)** | 内存省、快 | 十亿级 |

## 排序层概述

排序层（Rank Layer）对召回的候选物品进行精准排序，通常使用复杂的深度学习模型。

### 排序层特点

| 特点 | 说明 |
|------|------|
| **精度高** | 使用复杂模型进行精准预测 |
| **特征丰富** | 使用用户、物品、上下文等多维特征 |
| **候选量小** | 通常对数百个候选进行排序 |
| **时延要求** | 通常要求在50-100ms内完成 |

### 排序模型演进

```
传统模型 → 机器学习 → 深度学习
   ↓          ↓           ↓
  规则排序    LR/GBDT    Wide&Deep
              FM/FFM     DeepFM
                         DIN/DIEN
```

## 粗排与精排

### 粗排层 (Coarse Ranking)

在召回和精排之间增加轻量级排序层，减轻精排压力。

```
召回 (千级) → 粗排 (百级) → 精排 (十级) → 重排 (展示)
```

#### 粗排模型特点

| 特点 | 说明 |
|------|------|
| 轻量级 | 通常使用双塔模型或简单DNN |
| 速度快 | 要求在10ms以内完成 |
| 精度适中 | 快速筛选明显不相关的物品 |

#### 粗排双塔模型

```python
class CoarseRanker(nn.Module):
    """轻量级双塔粗排模型"""
    def __init__(self, embedding_dim=32):
        super().__init__()
        self.user_tower = nn.Sequential(
            nn.Linear(user_feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, embedding_dim)
        )
        self.item_tower = nn.Sequential(
            nn.Linear(item_feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, embedding_dim)
        )

    def forward(self, user_vec, item_vec):
        # 预计算物品向量，在线只计算用户向量
        score = torch.sum(user_vec * item_vec, dim=1)
        return score
```

### 精排层 (Fine Ranking)

使用复杂的深度学习模型进行精准预测。

## 排序模型详解

### 1. 逻辑回归 (LR)

$$P(y=1|x) = \frac{1}{1 + e^{-(w^T x + b)}}$$

**优点**：简单、可解释性强、训练速度快
**缺点**：无法自动学习特征交叉

### 2. 因子分解机 (FM)

$$\hat{y} = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^n \sum_{j=i+1}^n v_i^T v_j \cdot x_i x_j$$

**优点**：自动二阶特征交叉
**缺点**：只能学习低阶交叉

### 3. 梯度提升树 (GBDT)

```python
from sklearn.ensemble import GradientBoostingClassifier

model = GradientBoostingClassifier(
    n_estimators=100,
    max_depth=5,
    learning_rate=0.1
)
model.fit(X_train, y_train)
```

**优点**：自动特征组合、处理非线性关系
**缺点**：难以处理高维稀疏特征

### 4. 深度学习模型

见 [Wide & Deep 模型](/kb/searchrec/knowledge-6yaqh9)、[DeepFM 与 xDeepFM](/kb/searchrec/knowledge-fvgf4o)

## 多目标排序

### 问题背景

推荐系统通常需要同时优化多个目标：
- **CTR** (点击率)
- **CVR** (转化率)
- **停留时长**
- **收藏/分享率**

### 多目标建模方法

| 方法 | 说明 |
|------|------|
| **样本加权** | 对不同目标样本赋予不同权重 |
| **多任务学习** | 共享表示，多目标联合训练 |
| **序列建模** | 先预测CTR，再预测CVR |

详见 [多目标优化](/kb/searchrec/knowledge-svqsyo)

## 重排序层

### 作用

在精排之后，应用业务规则和策略进行最终调整。

### 常见策略

| 策略 | 说明 |
|------|------|
| **多样性控制** | 避免同类物品过度集中 |
| **新鲜度加权** | 新物品给予曝光机会 |
| **疲劳度控制** | 避免重复推荐同一物品 |
| **业务规则** | 强制插入运营物品 |
| **流量调控** | 扶持特定类目或商家 |

### 多样性算法

#### MMR (Maximal Marginal Relevance)

$$\text{MMR} = \lambda \cdot \text{Relevance}(d) - (1-\lambda) \cdot \max_{d_i \in S} \text{Sim}(d, d_i)$$

#### DPP (Determinantal Point Process)

通过行列式点过程建模物品间的多样性和相关性。

详见 [多样性与新颖性](/kb/searchrec/knowledge-9a858n)

### 特征服务

```python
class FeatureService:
    """特征服务：统一管理特征获取"""

    def __init__(self):
        self.redis = RedisClient()
        self.feature_store = FeatureStore()

    def get_user_features(self, user_id):
        """获取用户特征"""
        # 优先从缓存获取
        features = self.redis.get(f"user:{user_id}")
        if features:
            return features

        # 从特征仓库获取
        features = self.feature_store.query_user(user_id)
        self.redis.set(f"user:{user_id}", features, ttl=300)
        return features

    def get_item_features(self, item_ids):
        """批量获取物品特征"""
        return self.feature_store.query_items(item_ids)
```

## 召回层概述

召回层（Recall Layer）是推荐系统的第一阶段，目标是从海量物品库中快速筛选出一小部分候选物品。

### 召回层特点

| 特点 | 说明 |
|------|------|
| **速度快** | 通常要求在50ms以内返回结果 |
| **粗粒度** | 使用简单模型或规则快速筛选 |
| **高召回率** | 确保相关物品不被漏掉 |
| **候选量大** | 通常召回数百到数千个物品 |

### 召回层架构

```
用户请求
    ↓
┌────────────────────────────────────────────────────────────┐
│                      多路召回                               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐          │
│  │ 协同过滤 │ │ 向量召回 │ │ 热门召回 │ │ 策略召回 │          │
│  └────┬────┘ └────┬────┘ └────┬────┘ └────┬────┘          │
│       └───────────┴───────────┴───────────┘                │
│                         ↓                                  │
│                    候选物品池 (千级)                         │
└────────────────────────────────────────────────────────────┘
```

## 多路召回策略

### 1. 协同过滤召回

基于用户历史行为，通过 UserCF 或 ItemCF 召回相似物品。

```python
def cf_recall(user_id, top_k=100):
    """基于ItemCF的召回"""
    user_history = get_user_history(user_id)
    candidate_items = set()

    for item in user_history:
        # 获取相似物品
        similar_items = item_sim_matrix[item][:50]
        candidate_items.update([i[0] for i in similar_items])

    # 过滤已交互物品
    candidate_items -= user_history

    return list(candidate_items)[:top_k]
```

### 2. 向量召回（Embedding Recall）

将用户和物品表示为向量，通过向量相似度召回。

#### 双塔模型

```
用户特征                    物品特征
    ↓                          ↓
┌─────────┐                ┌─────────┐
│  User   │                │  Item   │
│  Tower  │                │  Tower  │
│ (DNN)   │                │ (DNN)   │
└────┬────┘                └────┬────┘
     ↓                          ↓
  User Vector (64d)       Item Vector (64d)
     ↓                          ↓
     └──────────┬───────────────┘
                ↓
          Cosine Similarity
```

#### 向量检索

```python
import faiss

# 构建 Faiss 索引
index = faiss.IndexFlatIP(embedding_dim)  # 内积索引
index.add(item_embeddings)  # 添加物品向量

# 向量召回
def vector_recall(user_vector, top_k=100):
    scores, item_ids = index.search(user_vector.reshape(1, -1), top_k)
    return item_ids[0]
```

### 3. 热门召回

基于全局热度召回，解决冷启动问题。

```python
def hot_recall(user_id, top_k=50):
    """基于物品热度召回"""
    # 考虑用户已交互的类目
    user_categories = get_user_preferred_categories(user_id)

    # 在偏好类目中按热度排序
    candidates = []
    for category in user_categories:
        hot_items = get_hot_items_by_category(category, top_k=20)
        candidates.extend(hot_items)

    return candidates
```

### 4. 策略召回

基于业务规则的召回策略。

| 策略类型 | 说明 | 示例 |
|----------|------|------|
| 新品召回 | 新上架物品曝光 | 新品加权 |
| 运营召回 | 运营活动物品 | 促销商品 |
| 地域召回 | 基于地理位置 | 附近商家 |
| 时间召回 | 基于时间特征 | 早餐推荐 |

## 向量召回详解

### 双塔模型训练

```python
import torch
import torch.nn as nn

class TwoTowerModel(nn.Module):
    def __init__(self, user_feature_dim, item_feature_dim, embedding_dim=64):
        super().__init__()

        # User Tower
        self.user_tower = nn.Sequential(
            nn.Linear(user_feature_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, embedding_dim)
        )

        # Item Tower
        self.item_tower = nn.Sequential(
            nn.Linear(item_feature_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 128),
            nn.ReLU(),
            nn.Linear(128, embedding_dim)
        )

    def forward(self, user_features, item_features):
        user_embedding = self.user_tower(user_features)
        item_embedding = self.item_tower(item_features)

        # 归一化
        user_embedding = F.normalize(user_embedding, p=2, dim=1)
        item_embedding = F.normalize(item_embedding, p=2, dim=1)

        # 计算相似度
        similarity = torch.sum(user_embedding * item_embedding, dim=1)
        return similarity
```

### 采样策略

| 采样方法 | 说明 | 适用场景 |
|----------|------|----------|
| **Batch内负采样** | 同batch其他样本作为负样本 | 简单高效 |
| **全局负采样** | 从全库随机采样 | 需要全库信息 |
| **困难负采样** | 采样与正样本相似的负样本 | 提升模型区分能力 |

### 向量索引技术

| 索引类型 | 特点 | 适用规模 |
|----------|------|----------|
| **Flat (暴力搜索)** | 精确、慢 | 百万级以下 |
| **IVF (倒排文件)** | 近似、快 | 千万级 |
| **HNSW (图索引)** | 高召回、较快 | 亿级 |
| **PQ (乘积量化)** | 内存省、快 | 十亿级 |

## 排序层概述

排序层（Rank Layer）对召回的候选物品进行精准排序，通常使用复杂的深度学习模型。

### 排序层特点

| 特点 | 说明 |
|------|------|
| **精度高** | 使用复杂模型进行精准预测 |
| **特征丰富** | 使用用户、物品、上下文等多维特征 |
| **候选量小** | 通常对数百个候选进行排序 |
| **时延要求** | 通常要求在50-100ms内完成 |

### 排序模型演进

```
传统模型 → 机器学习 → 深度学习
   ↓          ↓           ↓
  规则排序    LR/GBDT    Wide&Deep
              FM/FFM     DeepFM
                         DIN/DIEN
```

## 粗排与精排

### 粗排层 (Coarse Ranking)

在召回和精排之间增加轻量级排序层，减轻精排压力。

```
召回 (千级) → 粗排 (百级) → 精排 (十级) → 重排 (展示)
```

#### 粗排模型特点

| 特点 | 说明 |
|------|------|
| 轻量级 | 通常使用双塔模型或简单DNN |
| 速度快 | 要求在10ms以内完成 |
| 精度适中 | 快速筛选明显不相关的物品 |

#### 粗排双塔模型

```python
class CoarseRanker(nn.Module):
    """轻量级双塔粗排模型"""
    def __init__(self, embedding_dim=32):
        super().__init__()
        self.user_tower = nn.Sequential(
            nn.Linear(user_feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, embedding_dim)
        )
        self.item_tower = nn.Sequential(
            nn.Linear(item_feature_dim, 64),
            nn.ReLU(),
            nn.Linear(64, embedding_dim)
        )

    def forward(self, user_vec, item_vec):
        # 预计算物品向量，在线只计算用户向量
        score = torch.sum(user_vec * item_vec, dim=1)
        return score
```

## 排序模型详解

### 1. 逻辑回归 (LR)

$$P(y=1|x) = \frac{1}{1 + e^{-(w^T x + b)}}$$

**优点**：简单、可解释性强、训练速度快
**缺点**：无法自动学习特征交叉

### 2. 因子分解机 (FM)

$$\hat{y} = w_0 + \sum_{i=1}^n w_i x_i + \sum_{i=1}^n \sum_{j=i+1}^n v_i^T v_j \cdot x_i x_j$$

**优点**：自动二阶特征交叉
**缺点**：只能学习低阶交叉

### 3. 梯度提升树 (GBDT)

```python
from sklearn.ensemble import GradientBoostingClassifier

model = GradientBoostingClassifier(
    n_estimators=100,
    max_depth=5,
    learning_rate=0.1
)
model.fit(X_train, y_train)
```

**优点**：自动特征组合、处理非线性关系
**缺点**：难以处理高维稀疏特征

## 重排序层

### 作用

在精排之后，应用业务规则和策略进行最终调整。

### 常见策略

| 策略 | 说明 |
|------|------|
| **多样性控制** | 避免同类物品过度集中 |
| **新鲜度加权** | 新物品给予曝光机会 |
| **疲劳度控制** | 避免重复推荐同一物品 |
| **业务规则** | 强制插入运营物品 |
| **流量调控** | 扶持特定类目或商家 |

### 多样性算法

#### MMR (Maximal Marginal Relevance)

$$\text{MMR} = \lambda \cdot \text{Relevance}(d) - (1-\lambda) \cdot \max_{d_i \in S} \text{Sim}(d, d_i)$$

#### DPP (Determinantal Point Process)

通过行列式点过程建模物品间的多样性和相关性。

详见 [多样性与新颖性](/kb/searchrec/knowledge-9a858n)

class FeatureService:
    """特征服务：统一管理特征获取"""

    def __init__(self):
        self.redis = RedisClient()
