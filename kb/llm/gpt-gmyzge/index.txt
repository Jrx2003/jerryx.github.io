## 引言

GPT (Generative Pre-trained Transformer) 系列由 OpenAI 开发，代表了大规模语言模型发展的主线。从 GPT-1 到 GPT-4/o1，每一代都带来了突破性的进展。

### 发展时间线

```
2018.06  GPT-1    117M  参数，预训练+微调范式
2019.02  GPT-2    1.5B  参数，零样本能力涌现
2020.05  GPT-3    175B  参数，上下文学习(ICL)
2022.03  GPT-3.5  指令微调，RLHF
2023.03  GPT-4    多模态，推理能力飞跃
2024.05  GPT-4o   原生多模态，实时交互
2024.09  o1       推理时计算，System-2思考
2025.01  o3       更强推理，ARC-AGI突破
```

## GPT-1: 预训练+微调的范式

### 核心创新

**半监督学习框架**: 先在无标注数据上预训练，再在特定任务上微调。

### 架构细节

| 参数 | 配置 |
|------|------|
| 层数 | 12 |
| 隐藏维度 | 768 |
| 注意力头数 | 12 |
| 参数量 | 117M |
| 训练数据 | BooksCorpus (约 800M 词) |

### 预训练目标

**自回归语言建模**:

$$
L_{LM} = -\sum_{i} \log P(x_i | x_{i-1}, ..., x_1)
$$

### 微调方法

对于分类任务，添加线性层：

```
Input: [START] 文本 [EXTRACT]
Output: Transformer → Linear → 类别概率
```

### 任务适配

| 任务类型 | 输入改造 |
|----------|----------|
| 分类 | [START] 文本 [EXTRACT] |
| 蕴含 | [START] 前提 [DELIM] 假设 [EXTRACT] |
| 相似度 | [START] 文本1 [DELIM] 文本2 [EXTRACT] |
| 问答 | [START] 上下文 [DELIM] 问题 [EXTRACT] |

## GPT-2: 零样本能力的涌现

### 核心发现

**规模带来质变**: 当模型达到 1.5B 参数时，展现出零样本 (zero-shot) 能力。

### 架构升级

| 模型 | 层数 | 维度 | 头数 | 参数量 |
|------|------|------|------|--------|
| Small | 12 | 768 | 12 | 124M |
| Medium | 24 | 1024 | 16 | 355M |
| Large | 36 | 1280 | 20 | 774M |
| XL | 48 | 1600 | 25 | 1.5B |

### 关键改进

1. **LayerNorm 位置调整**: 移动到残差路径之前 (Pre-LN)
   ```
   GPT-1: x + Sublayer(LayerNorm(x))
   GPT-2: x + Sublayer(x) → LayerNorm  # 更稳定
   ```

2. **扩展词表**: 50257 tokens (BPE)

3. **更大训练数据**: WebText (40GB 文本)

### 零样本能力示例

```
提示: "翻译为法语: The cat sat on the mat."
输出: "Le chat s'est assis sur le tapis."

提示: "问题: 法国的首都是哪里? 答案:"
输出: "巴黎"
```

## GPT-3: 上下文学习革命

### 核心突破

**上下文学习 (In-Context Learning, ICL)**: 无需微调，仅通过提示中的示例即可学习新任务。

### 规模跃升

| 模型 | 层数 | 维度 | 头数 | 参数量 | 批次大小 |
|------|------|------|------|--------|----------|
| Small | 24 | 2048 | 16 | 125M | 0.5M |
| Medium | 24 | 4096 | 32 | 350M | 0.5M |
| Large | 24 | 5120 | 40 | 760M | 0.5M |
| XL | 24 | 6400 | 64 | 1.3B | 1M |
| 2.7B | 32 | 8192 | 32 | 2.7B | 1M |
| 6.7B | 32 | 12288 | 32 | 6.7B | 2M |
| 13B | 40 | 14336 | 40 | 13B | 2M |
| 175B (GPT-3) | 96 | 12288 | 96 | 175B | 3.2M |

### 上下文学习模式

```python
# Zero-shot
prompt = "翻译为法语: Hello world"

# One-shot
prompt = """
英文: Hello
法文: Bonjour

英文: Hello world
法文:
"""

# Few-shot
prompt = """
英文: Hello → 法文: Bonjour
英文: Good morning → 法文: Bonjour
英文: Thank you → 法文: Merci
英文: Hello world → 法文:
"""
```

### 涌现能力

| 能力 | 描述 | 临界规模 |
|------|------|----------|
| ICL | 从上下文示例学习 | ~6.7B |
| 算术 | 简单数学运算 | ~13B |
| 指代消解 | 理解代词指代 | ~6.7B |
| 常识推理 | 物理/社会常识 | ~13B |

## GPT-4: 多模态与推理

### 核心特性

1. **多模态输入**: 支持文本+图像
2. **更强推理**: 复杂逻辑和数学能力
3. **更长上下文**: 8K/32K tokens
4. **更安全**: RLHF 和安全训练

### 推测架构

| 特性 | 推测配置 |
|------|----------|
| 参数量 | ~1.8T (MoE) |
| 激活参数 | ~280B |
| 专家数 | 16 |
| 上下文长度 | 8K/32K/128K |
| 训练数据 | 多模态数据 |

### 能力评估

```
考试表现:
- BAR (律师): 90th percentile
- LSAT: 88th percentile
- SAT Math: 89th percentile
- GRE Quantitative: 80th percentile
```

## GPT-4o / o1 / o3: 最新演进

### GPT-4o (2024.05)

**原生多模态**: 端到端训练，非拼接模态。

| 特性 | 说明 |
|------|------|
| 音频输入 | 平均 320ms 响应 |
| 视觉理解 | 增强图像推理 |
| 成本降低 | 比 GPT-4 Turbo 便宜 50% |
| 速度提升 | 2x 文本生成速度 |

### o1 系列 (2024.09)

**推理时计算扩展**: 让模型"思考"更久。

```
传统 LLM: 输入 → 立即输出
o1:       输入 → 内部思维链 → 输出

特点:
- 在回答前进行内部推理
- 适合数学、编程、科学问题
- 有 o1-preview 和 o1-mini 版本
```

**性能表现**:
- IMO 资格赛: 83% (GPT-4o: 13%)
- Codeforces 竞赛: 89th percentile
- GPQA Diamond (博士级科学): 78.3%

### o3 系列 (2025.01)

**更强推理能力**:
- ARC-AGI 基准: 87.5% (人类水平约 85%)
- SWE-bench Verified: 71.7%

## 涌现能力分析

### 什么是涌现能力

**定义**: 模型在小规模时不具备，但在某个规模阈值后突然出现的全新能力。

### 涌现能力示例

```
参数规模 →
           1B    10B   100B   1T
           │      │      │      │
ICL        ───────┼──────┼──────┤
算术       ───────┼──────┼──────┤
代码生成   ───────┼──────┼──────┤
复杂推理   ───────┼──────┼──────┤
多步规划   ───────┼──────┼──────┤
```

### 涌现的争议

| 观点 | 说明 |
|------|------|
| 真涌现 | 新能力确实突然出现 |
| 度量假象 | 使用非线性指标导致的假象 |
| 平滑演进 | 能力实际平滑增长，只是指标未捕捉 |

## 面试要点

### 高频问题

**Q1: GPT 系列的核心演进脉络是什么？**

A:
- GPT-1: 确立预训练+微调范式
- GPT-2: 发现零样本能力，扩大规模
- GPT-3: 上下文学习，Few-shot 能力
- GPT-4: 多模态，推理能力飞跃
- o1/o3: 推理时计算，System-2 思考

**Q2: 什么是上下文学习 (ICL)？**

A: 模型无需参数更新，仅通过提示中的输入-输出示例即可学习新任务的能力。这是 GPT-3 及以后模型的核心能力。

**Q3: GPT-2 相比 GPT-1 的主要改进？**

A:
1. 规模扩大 10 倍 (117M → 1.5B)
2. LayerNorm 移到残差之前 (Pre-LN)
3. 更大词表和训练数据
4. 展现零样本能力

**Q4: GPT-3 的 175B 模型有哪些关键设计？**

A:
1. 交替使用密集和稀疏注意力
2. 大 batch size (3.2M tokens)
3. 学习率预热和衰减策略
4. 梯度裁剪

**Q5: 什么是涌现能力？**

A: 模型在小规模时不具备，但在某个规模阈值后突然出现的新能力。例如 ICL、算术能力等。

## 引言

GPT (Generative Pre-trained Transformer) 系列由 OpenAI 开发，代表了大规模语言模型发展的主线。从 GPT-1 到 GPT-4/o1，每一代都带来了突破性的进展。

### 发展时间线

```
2018.06  GPT-1    117M  参数，预训练+微调范式
2019.02  GPT-2    1.5B  参数，零样本能力涌现
2020.05  GPT-3    175B  参数，上下文学习(ICL)
2022.03  GPT-3.5  指令微调，RLHF
2023.03  GPT-4    多模态，推理能力飞跃
2024.05  GPT-4o   原生多模态，实时交互
2024.09  o1       推理时计算，System-2思考
2025.01  o3       更强推理，ARC-AGI突破
```

## GPT-1: 预训练+微调的范式

### 核心创新

**半监督学习框架**: 先在无标注数据上预训练，再在特定任务上微调。

### 架构细节

| 参数 | 配置 |
|------|------|
| 层数 | 12 |
| 隐藏维度 | 768 |
| 注意力头数 | 12 |
| 参数量 | 117M |
| 训练数据 | BooksCorpus (约 800M 词) |

### 预训练目标

**自回归语言建模**:

$$
L_{LM} = -\sum_{i} \log P(x_i | x_{i-1}, ..., x_1)
$$

### 微调方法

对于分类任务，添加线性层：

```
Input: [START] 文本 [EXTRACT]
Output: Transformer → Linear → 类别概率
```

### 任务适配

| 任务类型 | 输入改造 |
|----------|----------|
| 分类 | [START] 文本 [EXTRACT] |
| 蕴含 | [START] 前提 [DELIM] 假设 [EXTRACT] |
| 相似度 | [START] 文本1 [DELIM] 文本2 [EXTRACT] |
| 问答 | [START] 上下文 [DELIM] 问题 [EXTRACT] |

## GPT-2: 零样本能力的涌现

### 核心发现

**规模带来质变**: 当模型达到 1.5B 参数时，展现出零样本 (zero-shot) 能力。

### 架构升级

| 模型 | 层数 | 维度 | 头数 | 参数量 |
|------|------|------|------|--------|
| Small | 12 | 768 | 12 | 124M |
| Medium | 24 | 1024 | 16 | 355M |
| Large | 36 | 1280 | 20 | 774M |
| XL | 48 | 1600 | 25 | 1.5B |

### 关键改进

1. **LayerNorm 位置调整**: 移动到残差路径之前 (Pre-LN)
   ```
   GPT-1: x + Sublayer(LayerNorm(x))
   GPT-2: x + Sublayer(x) → LayerNorm  # 更稳定
   ```

2. **扩展词表**: 50257 tokens (BPE)

3. **更大训练数据**: WebText (40GB 文本)

### 零样本能力示例

```
提示: "翻译为法语: The cat sat on the mat."
输出: "Le chat s'est assis sur le tapis."

提示: "问题: 法国的首都是哪里? 答案:"
输出: "巴黎"
```

## GPT-3: 上下文学习革命

### 核心突破

**上下文学习 (In-Context Learning, ICL)**: 无需微调，仅通过提示中的示例即可学习新任务。

### 规模跃升

| 模型 | 层数 | 维度 | 头数 | 参数量 | 批次大小 |
|------|------|------|------|--------|----------|
| Small | 24 | 2048 | 16 | 125M | 0.5M |
| Medium | 24 | 4096 | 32 | 350M | 0.5M |
| Large | 24 | 5120 | 40 | 760M | 0.5M |
| XL | 24 | 6400 | 64 | 1.3B | 1M |
| 2.7B | 32 | 8192 | 32 | 2.7B | 1M |
| 6.7B | 32 | 12288 | 32 | 6.7B | 2M |
| 13B | 40 | 14336 | 40 | 13B | 2M |
| 175B (GPT-3) | 96 | 12288 | 96 | 175B | 3.2M |

### 上下文学习模式

```python
# Zero-shot
prompt = "翻译为法语: Hello world"

# One-shot
prompt = """
英文: Hello
法文: Bonjour

英文: Hello world
法文:
"""

# Few-shot
prompt = """
英文: Hello → 法文: Bonjour
英文: Good morning → 法文: Bonjour
英文: Thank you → 法文: Merci
英文: Hello world → 法文:
"""
```

### 涌现能力

| 能力 | 描述 | 临界规模 |
|------|------|----------|
| ICL | 从上下文示例学习 | ~6.7B |
| 算术 | 简单数学运算 | ~13B |
| 指代消解 | 理解代词指代 | ~6.7B |
| 常识推理 | 物理/社会常识 | ~13B |

## GPT-4o / o1 / o3: 最新演进

### GPT-4o (2024.05)

**原生多模态**: 端到端训练，非拼接模态。

| 特性 | 说明 |
|------|------|
| 音频输入 | 平均 320ms 响应 |
| 视觉理解 | 增强图像推理 |
| 成本降低 | 比 GPT-4 Turbo 便宜 50% |
| 速度提升 | 2x 文本生成速度 |

### o1 系列 (2024.09)

**推理时计算扩展**: 让模型"思考"更久。

```
传统 LLM: 输入 → 立即输出
o1:       输入 → 内部思维链 → 输出

特点:
- 在回答前进行内部推理
- 适合数学、编程、科学问题
- 有 o1-preview 和 o1-mini 版本
```

**性能表现**:
- IMO 资格赛: 83% (GPT-4o: 13%)
- Codeforces 竞赛: 89th percentile
- GPQA Diamond (博士级科学): 78.3%

### o3 系列 (2025.01)

**更强推理能力**:
- ARC-AGI 基准: 87.5% (人类水平约 85%)
- SWE-bench Verified: 71.7%

## 涌现能力分析

### 什么是涌现能力

**定义**: 模型在小规模时不具备，但在某个规模阈值后突然出现的全新能力。

### 涌现能力示例

```
参数规模 →
           1B    10B   100B   1T
           │      │      │      │
ICL        ───────┼──────┼──────┤
算术       ───────┼──────┼──────┤
代码生成   ───────┼──────┼──────┤
复杂推理   ───────┼──────┼──────┤
多步规划   ───────┼──────┼──────┤
```

### 涌现的争议

| 观点 | 说明 |
|------|------|
| 真涌现 | 新能力确实突然出现 |
| 度量假象 | 使用非线性指标导致的假象 |
| 平滑演进 | 能力实际平滑增长，只是指标未捕捉 |

## 面试要点

### 高频问题

**Q1: GPT 系列的核心演进脉络是什么？**

A:
- GPT-1: 确立预训练+微调范式
- GPT-2: 发现零样本能力，扩大规模
- GPT-3: 上下文学习，Few-shot 能力
- GPT-4: 多模态，推理能力飞跃
- o1/o3: 推理时计算，System-2 思考

**Q2: 什么是上下文学习 (ICL)？**

A: 模型无需参数更新，仅通过提示中的输入-输出示例即可学习新任务的能力。这是 GPT-3 及以后模型的核心能力。

**Q3: GPT-2 相比 GPT-1 的主要改进？**

A:
1. 规模扩大 10 倍 (117M → 1.5B)
2. LayerNorm 移到残差之前 (Pre-LN)
3. 更大词表和训练数据
4. 展现零样本能力

**Q4: GPT-3 的 175B 模型有哪些关键设计？**

A:
1. 交替使用密集和稀疏注意力
2. 大 batch size (3.2M tokens)
3. 学习率预热和衰减策略
4. 梯度裁剪

**Q5: 什么是涌现能力？**

A: 模型在小规模时不具备，但在某个规模阈值后突然出现的新能力。例如 ICL、算术能力等。
