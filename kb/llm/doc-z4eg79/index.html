长上下文挑战

外推问题

预训练: 最大长度 4K
推理时: 需要 32K

问题:
- 位置编码未见过长序列
- 注意力机制失效
- 性能急剧下降

位置插值 PI

原理

原始位置: [0, 1, 2, ..., 4095]
插值后: [0, 0.5, 1, ..., 2047.5] (扩展到 8K)

缩放因子 s = 目标长度 / 训练长度

实现

def position_interpolation(pos, scale_factor):
    return pos / scale_factor

NTK-aware 扩展

原理

高频信息外推，低频信息插值。

def ntk_scaling(base, dim, max_seq_len, scaling_factor):
    # 高频不插值，低频插值
    inv_freq = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
    inv_freq /= scaling_factor
    return inv_freq

YaRN

特点

  • 结合 PI 和 NTK
  • 温度缩放注意力
  • 更好的长上下文性能

面试要点

Q1: PI 和 NTK 的区别？

A:

  • PI: 所有频率都插值
  • NTK: 高频外推，低频插值

Q2: 长上下文的主要挑战？

A:

  • 位置编码外推
  • 注意力复杂度 O(n²)
  • 显存占用
