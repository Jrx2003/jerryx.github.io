## 长上下文挑战

### 为什么长上下文困难

| 挑战 | 说明 | 影响 |
|------|------|------|
| **计算复杂度** | Attention 是 O(n²) | 32K 上下文比 4K 慢 64 倍 |
| **显存占用** | KV Cache 随长度线性增长 | 128K 需要巨大显存 |
| **位置编码** | 超出训练长度时失效 | 性能急剧下降 |
| **注意力稀释** | 过长导致注意力分散 | 难以捕捉关键信息 |

### 上下文长度演进

| 年份 | 模型 | 上下文长度 |
|------|------|------------|
| 2019 | GPT-2 | 1K |
| 2020 | GPT-3 | 2K |
| 2022 | GPT-3.5 | 4K |
| 2023 | GPT-4 | 8K-32K |
| 2023 | Claude | 100K |
| 2024 | GPT-4 Turbo | 128K |
| 2024 | Gemini | 1M |

## 位置编码外推

### 问题描述

模型在训练长度 L 上训练，推理时超过 L 会出现：
1. 未见过的位置编码
2. 注意力分布异常
3. 性能急剧下降

### 外推方法

#### 1. NTK-Aware 外推

```python
def apply_ntk_scaling(freqs, scale=1.0):
    """
    NTK-aware 位置编码扩展

    原理: 高频信息压缩，低频信息保留
    """
    # 高频缩放更多，低频缩放更少
    scaled_freqs = freqs * scale

    return scaled_freqs

# 使用示例
base = 10000  # 原始基频
dim = 128
positions = torch.arange(max_length)

# NTK 缩放
freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
freqs = apply_ntk_scaling(freqs, scale=8)  # 扩展到 8 倍长度
```

#### 2. YaRN (Yet another RoPE extension)

```python
def apply_yarn_scaling(freqs, factor=8.0, beta_fast=32, beta_slow=1):
    """
    YaRN 位置编码扩展

    结合 NTK 和注意力温度缩放
    """
    # 计算每个维度的频率
    dims = len(freqs)

    # 高频/低频分离
    freqs_fast = freqs[:dims//beta_fast]
    freqs_slow = freqs[-dims//beta_slow:]
    freqs_mid = freqs[dims//beta_fast:-dims//beta_slow]

    # 不同缩放策略
    freqs_fast = freqs_fast * factor  # 高频完全外推
    freqs_mid = freqs_mid * (1 + (factor - 1) * ramp)  # 中频渐变
    freqs_slow = freqs_slow  # 低频不缩放

    return torch.cat([freqs_fast, freqs_mid, freqs_slow])

# 注意力温度缩放 (解决注意力稀释)
def apply_attention_temperature(attention_scores, length):
    """随长度增加温度，保持注意力分布尖锐"""
    temperature = 0.1 * math.log(length) + 1.0
    return attention_scores / temperature
```

#### 3. 位置插值 (PI)

```python
def apply_position_interpolation(positions, factor=2.0):
    """
    位置插值: 将长位置映射到训练长度内

    例如: 训练长度 4K，推理 8K，factor=2
    位置 8000 → 插值到 4000
    """
    return positions / factor
```

### 外推方法对比

| 方法 | 原理 | 效果 | 适用 |
|------|------|------|------|
| **PI** | 位置压缩 | 一般 | 短距离外推 |
| **NTK** | 频率缩放 | 好 | 中等距离 |
| **YaRN** | 组合策略 | 最好 | 长距离外推 |

## 上下文压缩

### 核心思想

将长上下文压缩为固定长度的表示。

```
原始: [token_1, token_2, ..., token_100000] (100K tokens)
压缩: [compressed_1, compressed_2, ..., compressed_1024] (1K tokens)
```

### 压缩方法

#### 1. 基于摘要的压缩

```python
class SummarizationCompressor:
    """使用摘要模型压缩上下文"""

    def __init__(self, summary_model):
        self.summary_model = summary_model
        self.chunk_size = 2048
        self.summary_length = 256

    def compress(self, long_context):
        """分段摘要压缩"""
        # 分块
        chunks = [
            long_context[i:i+self.chunk_size]
            for i in range(0, len(long_context), self.chunk_size)
        ]

        # 每块生成摘要
        summaries = [
            self.summary_model.summarize(chunk, max_length=self.summary_length)
            for chunk in chunks
        ]

        return "\n".join(summaries)
```

#### 2. 基于向量的压缩

```python
class VectorCompressor:
    """将文本压缩为向量表示"""

    def __init__(self, encoder):
        self.encoder = encoder
        self.token_budget = 1024

    def compress(self, long_context, query):
        """
        语义压缩: 只保留与查询相关的部分
        """
        # 编码查询
        query_vec = self.encoder.encode(query)

        # 分块编码
        chunks = self.chunk_text(long_context)
        chunk_vecs = [self.encoder.encode(c) for c in chunks]

        # 计算相似度
        similarities = [
            cosine_similarity(query_vec, cv)
            for cv in chunk_vecs
        ]

        # 选择最相关的块
        top_k = min(self.token_budget // len(chunks[0]), len(chunks))
        top_indices = np.argsort(similarities)[-top_k:]

        # 合并
        compressed = " ".join([chunks[i] for i in sorted(top_indices)])
        return compressed
```

#### 3. 可学习压缩器

```python
class LearnedCompressor(nn.Module):
    """可学习的上下文压缩器"""

    def __init__(self, d_model, compression_rate=8):
        super().__init__()
        self.compression_rate = compression_rate
        self.compression_layer = nn.TransformerEncoderLayer(d_model, nhead=8)
        self.projection = nn.Linear(d_model * compression_rate, d_model)

    def forward(self, context_embeddings):
        """
        context_embeddings: [batch, seq_len, d_model]
        输出: [batch, seq_len//rate, d_model]
        """
        batch, seq_len, d_model = context_embeddings.shape

        # 确保可整除
        seq_len = (seq_len // self.compression_rate) * self.compression_rate
        context_embeddings = context_embeddings[:, :seq_len]

        # reshape 为 [batch, seq_len//rate, rate, d_model]
        compressed = context_embeddings.view(
            batch, seq_len // self.compression_rate,
            self.compression_rate, d_model
        )

        # 压缩
        compressed = compressed.view(
            batch, seq_len // self.compression_rate,
            self.compression_rate * d_model
        )
        compressed = self.projection(compressed)

        return compressed
```

## 分块处理

### 滑动窗口

```python
def sliding_window_attention(query, key, value, window_size=1024):
    """
    滑动窗口注意力
    每个 token 只关注前后 window_size 个 token
    """
    batch, n_heads, seq_len, head_dim = query.shape

    outputs = []
    for i in range(seq_len):
        # 计算窗口边界
        start = max(0, i - window_size // 2)
        end = min(seq_len, i + window_size // 2)

        # 局部注意力
        local_q = query[:, :, i:i+1]
        local_k = key[:, :, start:end]
        local_v = value[:, :, start:end]

        scores = torch.matmul(local_q, local_k.transpose(-2, -1))
        scores = scores / math.sqrt(head_dim)
        attn = F.softmax(scores, dim=-1)

        output = torch.matmul(attn, local_v)
        outputs.append(output)

    return torch.cat(outputs, dim=2)
```

### 分层注意力

```python
class HierarchicalAttention(nn.Module):
    """分层注意力: 局部 + 全局"""

    def __init__(self, d_model, n_heads, chunk_size=512):
        super().__init__()
        self.chunk_size = chunk_size
        self.local_attn = nn.MultiheadAttention(d_model, n_heads)
        self.global_attn = nn.MultiheadAttention(d_model, n_heads)
        self.global_tokens = nn.Parameter(torch.randn(1, 128, d_model))

    def forward(self, x):
        batch, seq_len, d_model = x.shape

        # 1. 局部注意力 (分块)
        local_outputs = []
        for i in range(0, seq_len, self.chunk_size):
            chunk = x[:, i:i+self.chunk_size]
            local_out, _ = self.local_attn(chunk, chunk, chunk)
            local_outputs.append(local_out)

        # 2. 全局注意力 (使用摘要 token)
        global_tokens = self.global_tokens.expand(batch, -1, -1)
        global_out, _ = self.global_attn(global_tokens, x, x)

        # 3. 融合
        # 将全局信息广播到每个块
        output = torch.cat(local_outputs, dim=1)
        return output + global_out.repeat(1, seq_len // 128 + 1, 1)[:, :seq_len]
```

## 记忆机制

### 外部记忆

```python
class ExternalMemory(nn.Module):
    """可读写的外部记忆"""

    def __init__(self, memory_size=1024, d_model=768):
        super().__init__()
        self.memory = nn.Parameter(torch.randn(memory_size, d_model))
        self.write_head = nn.Linear(d_model, d_model)
        self.read_head = nn.MultiheadAttention(d_model, num_heads=8)

    def read(self, query):
        """从记忆中读取"""
        output, _ = self.read_head(query, self.memory, self.memory)
        return output

    def write(self, key, value):
        """写入记忆 (使用注意力更新)"""
        # 计算注意力权重
        scores = torch.matmul(key, self.memory.T)
        weights = F.softmax(scores, dim=-1)

        # 更新记忆
        update = torch.matmul(weights.T, value)
        self.memory = self.memory + self.write_head(update)
```

## 高效注意力

### Longformer / BigBird

```python
class SparseAttention(nn.Module):
    """稀疏注意力: 局部 + 全局 + 随机"""

    def __init__(self, d_model, n_heads, window_size=512, num_global=16):
        super().__init__()
        self.window_size = window_size
        self.num_global = num_global
        self.d_model = d_model
        self.n_heads = n_heads

    def forward(self, x):
        batch, seq_len, _ = x.shape

        # 1. 局部窗口注意力 (O(n×w))
        local_mask = self.create_local_mask(seq_len, self.window_size)

        # 2. 全局注意力 (O(n×g))
        global_indices = torch.linspace(0, seq_len-1, self.num_global).long()

        # 3. 随机注意力 (稀疏化)
        random_mask = torch.rand(seq_len, seq_len) < 0.01

        # 组合掩码
        attention_mask = local_mask | global_mask | random_mask

        # 应用稀疏注意力
        return self.sparse_attention_forward(x, attention_mask)
```

## 面试要点

### 高频问题

**Q1: 长上下文的主要技术挑战？**

A:
1. 计算复杂度 O(n²)
2. 显存占用线性增长
3. 位置编码外推
4. 注意力稀释

**Q2: RoPE 外推 vs 插值的区别？**

A:
- **外推**: 直接使用未见过的位置编码，效果差
- **插值**: 将长位置压缩到训练长度，效果好但可能损失细节

**Q3: YaRN 为什么有效？**

A:
1. 高频信息压缩少，保留细节
2. 低频信息压缩多，保留语义
3. 注意力温度缩放防止稀释

**Q4: 上下文压缩的方法？**

A:
1. 摘要压缩
2. 语义向量压缩
3. 可学习压缩器
4. RAG (检索增强)

**Q5: 滑动窗口注意力的优缺点？**

A:
**优点**: 计算复杂度 O(n×w)，可处理极长序列
**缺点**: 丢失长距离依赖，窗口边界信息损失

## 长上下文挑战

### 为什么长上下文困难

| 挑战 | 说明 | 影响 |
|------|------|------|
| **计算复杂度** | Attention 是 O(n²) | 32K 上下文比 4K 慢 64 倍 |
| **显存占用** | KV Cache 随长度线性增长 | 128K 需要巨大显存 |
| **位置编码** | 超出训练长度时失效 | 性能急剧下降 |
| **注意力稀释** | 过长导致注意力分散 | 难以捕捉关键信息 |

## 位置编码外推

### 问题描述

模型在训练长度 L 上训练，推理时超过 L 会出现：
1. 未见过的位置编码
2. 注意力分布异常
3. 性能急剧下降

### 外推方法

#### 1. NTK-Aware 外推

```python
def apply_ntk_scaling(freqs, scale=1.0):
    """
    NTK-aware 位置编码扩展

    原理: 高频信息压缩，低频信息保留
    """
    # 高频缩放更多，低频缩放更少
    scaled_freqs = freqs * scale

    return scaled_freqs

# 使用示例
base = 10000  # 原始基频
dim = 128
positions = torch.arange(max_length)

# NTK 缩放
freqs = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))
freqs = apply_ntk_scaling(freqs, scale=8)  # 扩展到 8 倍长度
```

#### 2. YaRN (Yet another RoPE extension)

```python
def apply_yarn_scaling(freqs, factor=8.0, beta_fast=32, beta_slow=1):
    """
    YaRN 位置编码扩展

    结合 NTK 和注意力温度缩放
    """
    # 计算每个维度的频率
    dims = len(freqs)

    # 高频/低频分离
    freqs_fast = freqs[:dims//beta_fast]
    freqs_slow = freqs[-dims//beta_slow:]
    freqs_mid = freqs[dims//beta_fast:-dims//beta_slow]

    # 不同缩放策略
    freqs_fast = freqs_fast * factor  # 高频完全外推
    freqs_mid = freqs_mid * (1 + (factor - 1) * ramp)  # 中频渐变
    freqs_slow = freqs_slow  # 低频不缩放

    return torch.cat([freqs_fast, freqs_mid, freqs_slow])

# 注意力温度缩放 (解决注意力稀释)
def apply_attention_temperature(attention_scores, length):
    """随长度增加温度，保持注意力分布尖锐"""
    temperature = 0.1 * math.log(length) + 1.0
    return attention_scores / temperature
```

#### 3. 位置插值 (PI)

```python
def apply_position_interpolation(positions, factor=2.0):
    """
    位置插值: 将长位置映射到训练长度内

    例如: 训练长度 4K，推理 8K，factor=2
    位置 8000 → 插值到 4000
    """
    return positions / factor
```

### 外推方法对比

| 方法 | 原理 | 效果 | 适用 |
|------|------|------|------|
| **PI** | 位置压缩 | 一般 | 短距离外推 |
| **NTK** | 频率缩放 | 好 | 中等距离 |
| **YaRN** | 组合策略 | 最好 | 长距离外推 |

## 上下文压缩

### 核心思想

将长上下文压缩为固定长度的表示。

```
原始: [token_1, token_2, ..., token_100000] (100K tokens)
压缩: [compressed_1, compressed_2, ..., compressed_1024] (1K tokens)
```

### 压缩方法

#### 1. 基于摘要的压缩

```python
class SummarizationCompressor:
    """使用摘要模型压缩上下文"""

    def __init__(self, summary_model):
        self.summary_model = summary_model
        self.chunk_size = 2048
        self.summary_length = 256

    def compress(self, long_context):
        """分段摘要压缩"""
        # 分块
        chunks = [
            long_context[i:i+self.chunk_size]
            for i in range(0, len(long_context), self.chunk_size)
        ]

        # 每块生成摘要
        summaries = [
            self.summary_model.summarize(chunk, max_length=self.summary_length)
            for chunk in chunks
        ]

        return "\n".join(summaries)
```

#### 2. 基于向量的压缩

```python
class VectorCompressor:
    """将文本压缩为向量表示"""

    def __init__(self, encoder):
        self.encoder = encoder
        self.token_budget = 1024

    def compress(self, long_context, query):
        """
        语义压缩: 只保留与查询相关的部分
        """
        # 编码查询
        query_vec = self.encoder.encode(query)

        # 分块编码
        chunks = self.chunk_text(long_context)
        chunk_vecs = [self.encoder.encode(c) for c in chunks]

        # 计算相似度
        similarities = [
            cosine_similarity(query_vec, cv)
            for cv in chunk_vecs
        ]

        # 选择最相关的块
        top_k = min(self.token_budget // len(chunks[0]), len(chunks))
        top_indices = np.argsort(similarities)[-top_k:]

        # 合并
        compressed = " ".join([chunks[i] for i in sorted(top_indices)])
        return compressed
```

#### 3. 可学习压缩器

```python
class LearnedCompressor(nn.Module):
    """可学习的上下文压缩器"""

    def __init__(self, d_model, compression_rate=8):
        super().__init__()
        self.compression_rate = compression_rate
        self.compression_layer = nn.TransformerEncoderLayer(d_model, nhead=8)
        self.projection = nn.Linear(d_model * compression_rate, d_model)

    def forward(self, context_embeddings):
        """
        context_embeddings: [batch, seq_len, d_model]
        输出: [batch, seq_len//rate, d_model]
        """
        batch, seq_len, d_model = context_embeddings.shape

        # 确保可整除
        seq_len = (seq_len // self.compression_rate) * self.compression_rate
        context_embeddings = context_embeddings[:, :seq_len]

        # reshape 为 [batch, seq_len//rate, rate, d_model]
        compressed = context_embeddings.view(
            batch, seq_len // self.compression_rate,
            self.compression_rate, d_model
        )

        # 压缩
        compressed = compressed.view(
            batch, seq_len // self.compression_rate,
            self.compression_rate * d_model
        )
        compressed = self.projection(compressed)

        return compressed
```

## 分块处理

### 滑动窗口

```python
def sliding_window_attention(query, key, value, window_size=1024):
    """
    滑动窗口注意力
    每个 token 只关注前后 window_size 个 token
    """
    batch, n_heads, seq_len, head_dim = query.shape

    outputs = []
    for i in range(seq_len):
        # 计算窗口边界
        start = max(0, i - window_size // 2)
        end = min(seq_len, i + window_size // 2)

        # 局部注意力
        local_q = query[:, :, i:i+1]
        local_k = key[:, :, start:end]
        local_v = value[:, :, start:end]

        scores = torch.matmul(local_q, local_k.transpose(-2, -1))
        scores = scores / math.sqrt(head_dim)
        attn = F.softmax(scores, dim=-1)

        output = torch.matmul(attn, local_v)
        outputs.append(output)

    return torch.cat(outputs, dim=2)
```

### 分层注意力

```python
class HierarchicalAttention(nn.Module):
    """分层注意力: 局部 + 全局"""

    def __init__(self, d_model, n_heads, chunk_size=512):
        super().__init__()
        self.chunk_size = chunk_size
        self.local_attn = nn.MultiheadAttention(d_model, n_heads)
        self.global_attn = nn.MultiheadAttention(d_model, n_heads)
        self.global_tokens = nn.Parameter(torch.randn(1, 128, d_model))

    def forward(self, x):
        batch, seq_len, d_model = x.shape

        # 1. 局部注意力 (分块)
        local_outputs = []
        for i in range(0, seq_len, self.chunk_size):
            chunk = x[:, i:i+self.chunk_size]
            local_out, _ = self.local_attn(chunk, chunk, chunk)
            local_outputs.append(local_out)

        # 2. 全局注意力 (使用摘要 token)
        global_tokens = self.global_tokens.expand(batch, -1, -1)
        global_out, _ = self.global_attn(global_tokens, x, x)

        # 3. 融合
        # 将全局信息广播到每个块
        output = torch.cat(local_outputs, dim=1)
        return output + global_out.repeat(1, seq_len // 128 + 1, 1)[:, :seq_len]
```

## 记忆机制

### 外部记忆

```python
class ExternalMemory(nn.Module):
    """可读写的外部记忆"""

    def __init__(self, memory_size=1024, d_model=768):
        super().__init__()
        self.memory = nn.Parameter(torch.randn(memory_size, d_model))
        self.write_head = nn.Linear(d_model, d_model)
        self.read_head = nn.MultiheadAttention(d_model, num_heads=8)

    def read(self, query):
        """从记忆中读取"""
        output, _ = self.read_head(query, self.memory, self.memory)
        return output

    def write(self, key, value):
        """写入记忆 (使用注意力更新)"""
        # 计算注意力权重
        scores = torch.matmul(key, self.memory.T)
        weights = F.softmax(scores, dim=-1)

        # 更新记忆
        update = torch.matmul(weights.T, value)
        self.memory = self.memory + self.write_head(update)
```

## 高效注意力

### Longformer / BigBird

```python
class SparseAttention(nn.Module):
    """稀疏注意力: 局部 + 全局 + 随机"""

    def __init__(self, d_model, n_heads, window_size=512, num_global=16):
        super().__init__()
        self.window_size = window_size
        self.num_global = num_global
        self.d_model = d_model
        self.n_heads = n_heads

    def forward(self, x):
        batch, seq_len, _ = x.shape

        # 1. 局部窗口注意力 (O(n×w))
        local_mask = self.create_local_mask(seq_len, self.window_size)

        # 2. 全局注意力 (O(n×g))
        global_indices = torch.linspace(0, seq_len-1, self.num_global).long()

        # 3. 随机注意力 (稀疏化)
        random_mask = torch.rand(seq_len, seq_len) < 0.01

        # 组合掩码
        attention_mask = local_mask | global_mask | random_mask

        # 应用稀疏注意力
        return self.sparse_attention_forward(x, attention_mask)
```

## 面试要点

### 高频问题

**Q1: 长上下文的主要技术挑战？**

A:
1. 计算复杂度 O(n²)
2. 显存占用线性增长
3. 位置编码外推
4. 注意力稀释

**Q2: RoPE 外推 vs 插值的区别？**

A:
- **外推**: 直接使用未见过的位置编码，效果差
- **插值**: 将长位置压缩到训练长度，效果好但可能损失细节

**Q3: YaRN 为什么有效？**

A:
1. 高频信息压缩少，保留细节
2. 低频信息压缩多，保留语义
3. 注意力温度缩放防止稀释

**Q4: 上下文压缩的方法？**

A:
1. 摘要压缩
2. 语义向量压缩
3. 可学习压缩器
4. RAG (检索增强)

**Q5: 滑动窗口注意力的优缺点？**

A:
**优点**: 计算复杂度 O(n×w)，可处理极长序列
**缺点**: 丢失长距离依赖，窗口边界信息损失
