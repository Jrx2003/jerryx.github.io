## PEFT 概述

### 什么是 PEFT

Parameter-Efficient Fine-Tuning，参数高效微调，旨在用少量参数微调大模型。

```
全量微调:
模型: 7B 参数
训练: 更新全部 7B 参数
显存: 需要 140GB+

PEFT:
模型: 7B 参数 (冻结)
训练: 只更新 0.1-1% 参数 (7M-70M)
显存: 只需 20-30GB
```

### 核心思想

| 方法 | 核心思想 | 可训练参数 |
|------|----------|------------|
| **LoRA** | 低秩适配 | 0.1-1% |
| **Prefix** | 前缀嵌入 | 0.1% |
| **Adapter** | 插入小网络 | 0.5-5% |
| **Prompt** | 软提示 | 0.01% |

## LoRA 详解

### 核心思想

低秩适配 (Low-Rank Adaptation)：冻结原权重，添加低秩矩阵进行微调。

```
原权重: W ∈ R^(d×k)
微调: W' = W + ΔW
其中: ΔW = A × B, A ∈ R^(d×r), B ∈ R^(r×k), r << min(d,k)
```

### 公式推导

$$
W' = W + \Delta W = W + BA
$$

其中：
- $W$: 预训练权重 (冻结)
- $A, B$: 可训练的低秩矩阵
- $r$: 秩 (通常 4-64)

### 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class LoRALayer(nn.Module):
    def __init__(
        self,
        in_features: int,
        out_features: int,
        rank: int = 8,
        lora_alpha: int = 16,
        lora_dropout: float = 0.0,
    ):
        super().__init__()
        self.rank = rank
        self.lora_alpha = lora_alpha
        self.scaling = lora_alpha / rank

        # 低秩矩阵
        self.lora_A = nn.Parameter(torch.zeros(in_features, rank))
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))

        # 初始化
        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
        nn.init.zeros_(self.lora_B)

        self.dropout = nn.Dropout(lora_dropout) if lora_dropout > 0 else nn.Identity()

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """前向传播: x @ (A @ B) * scaling"""
        # 原始输出 (冻结权重)
        # original = F.linear(x, self.weight, self.bias)

        # LoRA 分支
        lora_output = self.dropout(x) @ self.lora_A @ self.lora_B * self.scaling

        return lora_output


class LinearWithLoRA(nn.Module):
    """带 LoRA 的线性层"""

    def __init__(self, base_layer: nn.Linear, rank: int = 8, lora_alpha: int = 16):
        super().__init__()
        self.base_layer = base_layer  # 冻结
        self.lora = LoRALayer(
            base_layer.in_features,
            base_layer.out_features,
            rank=rank,
            lora_alpha=lora_alpha
        )

        # 冻结原权重
        for param in self.base_layer.parameters():
            param.requires_grad = False

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.base_layer(x) + self.lora(x)
```

### 应用到 Transformer

```python
def inject_lora_to_model(model, target_modules, rank=8, alpha=16):
    """
    将 LoRA 注入到指定模块

    target_modules: ["q_proj", "v_proj", "k_proj", "o_proj"]
    """
    for name, module in model.named_modules():
        if any(target in name for target in target_modules):
            if isinstance(module, nn.Linear):
                # 替换为 LoRA 层
                parent_name = ".".join(name.split(".")[:-1])
                child_name = name.split(".")[-1]
                parent = model.get_submodule(parent_name)

                lora_layer = LinearWithLoRA(module, rank=rank, lora_alpha=alpha)
                setattr(parent, child_name, lora_layer)

    return model
```

### 超参数选择

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **rank (r)** | 低秩维度 | 4-64，通常 8-16 |
| **lora_alpha** | 缩放系数 | 通常 2×rank |
| **target_modules** | 目标层 | q_proj, v_proj |
| **dropout** | 正则化 | 0.0-0.1 |

## QLoRA 量化微调

### 核心创新

将 LoRA 与量化结合，进一步降低显存占用。

```
全量微调: 140GB
LoRA:     30GB
QLoRA:    18GB (FP16 模型 → 4-bit 量化)
```

### 4-bit 量化

```python
from transformers import BitsAndBytesConfig
import torch

# QLoRA 配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,  # 嵌套量化

    bnb_4bit_compute_dtype=torch.bfloat16,

model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b",
    quantization_config=bnb_config,
    device_map="auto",
)

# 准备模型用于训练
from peft import prepare_model_for_kbit_training

model = prepare_model_for_kbit_training(model)

# 添加 LoRA
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",

## DoRA 权重分解

### 核心思想

Weight-Decomposed Low-Rank Adaptation，将权重分解为幅度和方向分别微调。

```
权重分解: W = m × (V / ||V||)
其中:
  m: 幅度 (magnitude)
  V: 方向 (direction)

微调:
  m' = m + Δm  (全量微调幅度)
  V' = V + ΔV  (LoRA 微调方向)
```

### 优势

| 特性 | LoRA | DoRA |
|------|------|------|
| 学习能力 | 一般 | 更强 |
| 训练稳定 | 一般 | 更稳定 |
| 层间差异 | 忽略 | 尊重 |

## Prefix Tuning

### 核心思想

在每层 Transformer 前添加可训练的前缀嵌入。

```
原输入: [x1, x2, x3, ..., xn]
Prefix: [p1, p2, ..., pk]
新输入: [p1, p2, ..., pk, x1, x2, x3, ..., xn]
         └── 可训练 ──┘   └── 原输入 ──┘
```

### 实现

```python
class PrefixTuning(nn.Module):
    def __init__(self, num_layers, num_heads, prefix_length, d_model):
        super().__init__()
        self.prefix_length = prefix_length

        # 每层的前缀嵌入
        self.prefix_embeddings = nn.ParameterList([
            nn.Parameter(torch.randn(prefix_length, d_model))
            for _ in range(num_layers)
        ])

    def forward(self, hidden_states, layer_idx):
        """将前缀拼接到输入前"""
        prefix = self.prefix_embeddings[layer_idx]
        prefix = prefix.unsqueeze(0).expand(hidden_states.size(0), -1, -1)
        return torch.cat([prefix, hidden_states], dim=1)
```

## P-Tuning v2

### 核心思想

在深层 Transformer 添加连续提示，而非仅在输入层。

```
P-Tuning v1: 只在输入层添加软提示
P-Tuning v2: 每一层都添加软提示
```

### 对比

| 方法 | 提示位置 | 效果 |
|------|----------|------|
| P-Tuning | 输入层 | 一般 |
| P-Tuning v2 | 所有层 | 更好 |
| Prompt Tuning | 输入层 | 简单 |

## Adapter 适配器

### 核心思想

在 Transformer 层之间插入小型神经网络 (Adapter)。

```
Transformer 层输出 ──→ Adapter ──→ 下一层输入

Adapter 结构:
输入 ──→ Down (d→64) ──→ GELU ──→ Up (64→d) ──→ 输出 + 残差
```

### 实现

```python
class Adapter(nn.Module):
    def __init__(self, d_model, bottleneck_dim=64):
        super().__init__()
        self.down = nn.Linear(d_model, bottleneck_dim)
        self.activation = nn.GELU()
        self.up = nn.Linear(bottleneck_dim, d_model)
        self.dropout = nn.Dropout(0.1)

    def forward(self, x):
        residual = x
        x = self.down(x)
        x = self.activation(x)
        x = self.dropout(x)
        x = self.up(x)
        return x + residual  # 残差连接


class TransformerWithAdapter(nn.Module):
    def __init__(self, base_transformer):
        super().__init__()
        self.transformer = base_transformer

        # 在每个层后添加 adapter
        self.adapters = nn.ModuleList([
            Adapter(base_transformer.config.hidden_size)
            for _ in range(base_transformer.config.num_layers)
        ])

    def forward(self, x):
        for i, layer in enumerate(self.transformer.layers):
            x = layer(x)
            x = self.adapters[i](x)
        return x
```

## 方法对比

### 综合对比表

| 方法 | 参数效率 | 推理开销 | 实现难度 | 推荐场景 |
|------|----------|----------|----------|----------|
| **LoRA** | ⭐⭐⭐ | 无 | 低 | 通用首选 |
| **QLoRA** | ⭐⭐⭐⭐⭐ | 无 | 中 | 资源受限 |
| **DoRA** | ⭐⭐⭐ | 无 | 中 | 追求效果 |
| **Prefix** | ⭐⭐ | 有 | 中 | 生成任务 |
| **P-Tuning** | ⭐⭐ | 有 | 低 | NLU任务 |
| **Adapter** | ⭐⭐⭐ | 有 | 中 | 多任务 |

### 选择建议

```
显存充足 (40GB+):
  └── 追求效果: DoRA
  └── 追求稳定: LoRA (r=16)

显存有限 (24GB):
  └── QLoRA (4-bit + LoRA)

显存极少 (16GB):
  └── QLoRA + 更小的 rank (r=4-8)

多任务场景:
  └── Adapter 或 LoRA + 多个 adapter
```

## 面试要点

### 高频问题

**Q1: LoRA 为什么有效？**

A:
1. 预训练权重已包含丰富知识
2. 任务适配只需要低秩调整
3. 减少可训练参数，防止过拟合
4. 类似矩阵分解的思想

**Q2: LoRA 的 rank 如何选择？**

A:
- 简单任务: r=4-8
- 复杂任务: r=16-32
- 通常 alpha=2×r
- 可通过实验验证

**Q3: QLoRA vs LoRA 的区别？**

A:
| 维度 | LoRA | QLoRA |
|------|------|-------|
| 基座模型 | FP16 | 4-bit 量化 |
| 显存占用 | 中等 | 极低 |
| 效果 | 好 | 接近 LoRA |
| 适用 | 通用 | 资源受限 |

**Q4: 为什么需要 PEFT？**

A:
1. 降低显存需求
2. 减少训练时间
3. 便于模型共享 (只传 adapter)
4. 防止灾难性遗忘

**Q5: LoRA 与 Adapter 的对比？**

A:
- LoRA: 修改权重，无推理开销
- Adapter: 插入模块，有推理开销
- LoRA 更适合部署
- Adapter 更适合多任务

## PEFT 概述

### 什么是 PEFT

Parameter-Efficient Fine-Tuning，参数高效微调，旨在用少量参数微调大模型。

```
全量微调:
模型: 7B 参数
训练: 更新全部 7B 参数
显存: 需要 140GB+

PEFT:
模型: 7B 参数 (冻结)
训练: 只更新 0.1-1% 参数 (7M-70M)
显存: 只需 20-30GB
```

### 核心思想

| 方法 | 核心思想 | 可训练参数 |
|------|----------|------------|
| **LoRA** | 低秩适配 | 0.1-1% |
| **Prefix** | 前缀嵌入 | 0.1% |
| **Adapter** | 插入小网络 | 0.5-5% |
| **Prompt** | 软提示 | 0.01% |

## LoRA 详解

### 核心思想

低秩适配 (Low-Rank Adaptation)：冻结原权重，添加低秩矩阵进行微调。

```
原权重: W ∈ R^(d×k)
微调: W' = W + ΔW
其中: ΔW = A × B, A ∈ R^(d×r), B ∈ R^(r×k), r << min(d,k)
```

### 公式推导

$$
W' = W + \Delta W = W + BA
$$

其中：
- $W$: 预训练权重 (冻结)
- $A, B$: 可训练的低秩矩阵
- $r$: 秩 (通常 4-64)

### 代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class LoRALayer(nn.Module):
    def __init__(
        self,
        in_features: int,
        out_features: int,
        rank: int = 8,
        lora_alpha: int = 16,
        lora_dropout: float = 0.0,
    ):
        super().__init__()
        self.rank = rank
        self.lora_alpha = lora_alpha
        self.scaling = lora_alpha / rank

        # 低秩矩阵
        self.lora_A = nn.Parameter(torch.zeros(in_features, rank))
        self.lora_B = nn.Parameter(torch.zeros(rank, out_features))

        # 初始化
        nn.init.kaiming_uniform_(self.lora_A, a=math.sqrt(5))
        nn.init.zeros_(self.lora_B)

        self.dropout = nn.Dropout(lora_dropout) if lora_dropout > 0 else nn.Identity()

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        """前向传播: x @ (A @ B) * scaling"""
        # 原始输出 (冻结权重)
        # original = F.linear(x, self.weight, self.bias)

        # LoRA 分支
        lora_output = self.dropout(x) @ self.lora_A @ self.lora_B * self.scaling

        return lora_output


class LinearWithLoRA(nn.Module):
    """带 LoRA 的线性层"""

    def __init__(self, base_layer: nn.Linear, rank: int = 8, lora_alpha: int = 16):
        super().__init__()
        self.base_layer = base_layer  # 冻结
        self.lora = LoRALayer(
            base_layer.in_features,
            base_layer.out_features,
            rank=rank,
            lora_alpha=lora_alpha
        )

        # 冻结原权重
        for param in self.base_layer.parameters():
            param.requires_grad = False

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.base_layer(x) + self.lora(x)
```

### 应用到 Transformer

```python
def inject_lora_to_model(model, target_modules, rank=8, alpha=16):
    """
    将 LoRA 注入到指定模块

    target_modules: ["q_proj", "v_proj", "k_proj", "o_proj"]
    """
    for name, module in model.named_modules():
        if any(target in name for target in target_modules):
            if isinstance(module, nn.Linear):
                # 替换为 LoRA 层
                parent_name = ".".join(name.split(".")[:-1])
                child_name = name.split(".")[-1]
                parent = model.get_submodule(parent_name)

                lora_layer = LinearWithLoRA(module, rank=rank, lora_alpha=alpha)
                setattr(parent, child_name, lora_layer)

    return model
```

### 超参数选择

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| **rank (r)** | 低秩维度 | 4-64，通常 8-16 |
| **lora_alpha** | 缩放系数 | 通常 2×rank |
| **target_modules** | 目标层 | q_proj, v_proj |
| **dropout** | 正则化 | 0.0-0.1 |

## QLoRA 量化微调

### 核心创新

将 LoRA 与量化结合，进一步降低显存占用。

```
全量微调: 140GB
LoRA:     30GB
QLoRA:    18GB (FP16 模型 → 4-bit 量化)
```

### 4-bit 量化

```python
from transformers import BitsAndBytesConfig
import torch

# QLoRA 配置
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,  # 嵌套量化
    bnb_4bit_quant_type="nf4",       # Normal Float 4
    bnb_4bit_compute_dtype=torch.bfloat16,
)

# 加载量化模型
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b",
    quantization_config=bnb_config,
    device_map="auto",
)

# 准备模型用于训练
from peft import prepare_model_for_kbit_training

model = prepare_model_for_kbit_training(model)

# 添加 LoRA
from peft import LoraConfig, get_peft_model

lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

model = get_peft_model(model, lora_config)
```

## DoRA 权重分解

### 核心思想

Weight-Decomposed Low-Rank Adaptation，将权重分解为幅度和方向分别微调。

```
权重分解: W = m × (V / ||V||)
其中:
  m: 幅度 (magnitude)
  V: 方向 (direction)

微调:
  m' = m + Δm  (全量微调幅度)
  V' = V + ΔV  (LoRA 微调方向)
```

### 优势

| 特性 | LoRA | DoRA |
|------|------|------|
| 学习能力 | 一般 | 更强 |
| 训练稳定 | 一般 | 更稳定 |
| 层间差异 | 忽略 | 尊重 |

## Prefix Tuning

### 核心思想

在每层 Transformer 前添加可训练的前缀嵌入。

```
原输入: [x1, x2, x3, ..., xn]
Prefix: [p1, p2, ..., pk]
新输入: [p1, p2, ..., pk, x1, x2, x3, ..., xn]
         └── 可训练 ──┘   └── 原输入 ──┘
```

### 实现

```python
class PrefixTuning(nn.Module):
    def __init__(self, num_layers, num_heads, prefix_length, d_model):
        super().__init__()
        self.prefix_length = prefix_length

        # 每层的前缀嵌入
        self.prefix_embeddings = nn.ParameterList([
            nn.Parameter(torch.randn(prefix_length, d_model))
            for _ in range(num_layers)
        ])

    def forward(self, hidden_states, layer_idx):
        """将前缀拼接到输入前"""
        prefix = self.prefix_embeddings[layer_idx]
        prefix = prefix.unsqueeze(0).expand(hidden_states.size(0), -1, -1)
        return torch.cat([prefix, hidden_states], dim=1)
```

## P-Tuning v2

### 核心思想

在深层 Transformer 添加连续提示，而非仅在输入层。

```
P-Tuning v1: 只在输入层添加软提示
P-Tuning v2: 每一层都添加软提示
```

### 对比

| 方法 | 提示位置 | 效果 |
|------|----------|------|
| P-Tuning | 输入层 | 一般 |
| P-Tuning v2 | 所有层 | 更好 |
| Prompt Tuning | 输入层 | 简单 |

## Adapter 适配器

### 核心思想

在 Transformer 层之间插入小型神经网络 (Adapter)。

```
Transformer 层输出 ──→ Adapter ──→ 下一层输入

Adapter 结构:
输入 ──→ Down (d→64) ──→ GELU ──→ Up (64→d) ──→ 输出 + 残差
```

### 实现

```python
class Adapter(nn.Module):
    def __init__(self, d_model, bottleneck_dim=64):
        super().__init__()
        self.down = nn.Linear(d_model, bottleneck_dim)
        self.activation = nn.GELU()
        self.up = nn.Linear(bottleneck_dim, d_model)
        self.dropout = nn.Dropout(0.1)

    def forward(self, x):
        residual = x
        x = self.down(x)
        x = self.activation(x)
        x = self.dropout(x)
        x = self.up(x)
        return x + residual  # 残差连接


class TransformerWithAdapter(nn.Module):
    def __init__(self, base_transformer):
        super().__init__()
        self.transformer = base_transformer

        # 在每个层后添加 adapter
        self.adapters = nn.ModuleList([
            Adapter(base_transformer.config.hidden_size)
            for _ in range(base_transformer.config.num_layers)
        ])

    def forward(self, x):
        for i, layer in enumerate(self.transformer.layers):
            x = layer(x)
            x = self.adapters[i](x)
        return x
```

## 方法对比

### 综合对比表

| 方法 | 参数效率 | 推理开销 | 实现难度 | 推荐场景 |
|------|----------|----------|----------|----------|
| **LoRA** | ⭐⭐⭐ | 无 | 低 | 通用首选 |
| **QLoRA** | ⭐⭐⭐⭐⭐ | 无 | 中 | 资源受限 |
| **DoRA** | ⭐⭐⭐ | 无 | 中 | 追求效果 |
| **Prefix** | ⭐⭐ | 有 | 中 | 生成任务 |
| **P-Tuning** | ⭐⭐ | 有 | 低 | NLU任务 |
| **Adapter** | ⭐⭐⭐ | 有 | 中 | 多任务 |

### 选择建议

```
显存充足 (40GB+):
  └── 追求效果: DoRA
  └── 追求稳定: LoRA (r=16)

显存有限 (24GB):
  └── QLoRA (4-bit + LoRA)

显存极少 (16GB):
  └── QLoRA + 更小的 rank (r=4-8)

多任务场景:
  └── Adapter 或 LoRA + 多个 adapter
```

## 面试要点

### 高频问题

**Q1: LoRA 为什么有效？**

A:
1. 预训练权重已包含丰富知识
2. 任务适配只需要低秩调整
3. 减少可训练参数，防止过拟合
4. 类似矩阵分解的思想

**Q2: LoRA 的 rank 如何选择？**

A:
- 简单任务: r=4-8
- 复杂任务: r=16-32
- 通常 alpha=2×r
- 可通过实验验证

**Q3: QLoRA vs LoRA 的区别？**

A:
| 维度 | LoRA | QLoRA |
|------|------|-------|
| 基座模型 | FP16 | 4-bit 量化 |
| 显存占用 | 中等 | 极低 |
| 效果 | 好 | 接近 LoRA |
| 适用 | 通用 | 资源受限 |

**Q4: 为什么需要 PEFT？**

A:
1. 降低显存需求
2. 减少训练时间
3. 便于模型共享 (只传 adapter)
4. 防止灾难性遗忘

**Q5: LoRA 与 Adapter 的对比？**

A:
- LoRA: 修改权重，无推理开销
- Adapter: 插入模块，有推理开销
- LoRA 更适合部署
- Adapter 更适合多任务
