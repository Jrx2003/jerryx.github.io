## 为什么需要位置编码

### 问题背景

Self-Attention 是**位置无关 (Permutation Invariant)** 的：

```
输入: [A, B, C] → Attention 输出
输入: [B, A, C] → 相同的 Attention 输出（只是顺序变了）

这意味着模型无法区分 "A 在 B 前面" 和 "B 在 A 前面"
```

### 解决方案

为每个位置添加唯一的标识，让模型感知顺序信息。

---

## 绝对位置编码

### 1. 正弦位置编码 (Sinusoidal)

Transformer 原始论文提出：

$$
\begin{aligned}
PE_{(pos, 2i)} &= \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \\
PE_{(pos, 2i+1)} &= \cos\left(\frac{position}{10000^{2i/d_{model}}}\right)
\end{aligned}
$$

其中：
- $pos$: 位置索引
- $i$: 维度索引
- $d_{model}$: 模型维度

### 波长分析

对于维度 $i$，波长为：

$$
\lambda_i = 2\pi \cdot 10000^{2i/d_{model}}
$$

- 小 $i$ (低维): 长波长，捕获长距离模式
- 大 $i$ (高维): 短波长，捕获细粒度变化

```
维度 0:  波长 ≈ 2π      (高频，精细位置)
维度 128: 波长 ≈ 2π×100 (中频)
维度 512: 波长 ≈ 2π×10000 (低频，粗粒度位置)
```

### 2. 可学习位置编码 (Learned)

BERT 采用的方式：

```python
self.position_embeddings = nn.Embedding(max_seq_len, d_model)
```

**优点**:
- 灵活性高，数据驱动
- 可以针对特定任务优化

**缺点**:
- 外推性差，无法处理训练时未见过的长度
- 增加了参数量

### 绝对位置编码对比

| 特性 | 正弦 | 可学习 |
|------|------|--------|
| 参数量 | 0 | $max\_seq \times d_{model}$ |
| 外推性 | 好 | 差 |
| 灵活性 | 固定 | 可优化 |
| 相对位置 | 可表示 | 不可直接表示 |

## 相对位置编码

### 核心思想

关注 token 之间的相对距离而非绝对位置：

```
绝对: token A 在位置 5，token B 在位置 8
相对: token B 距离 token A 有 +3 个位置
```

### 1. Transformer-XL 相对位置编码

在注意力计算中引入相对位置偏置：

$$
A_{ij} = \underbrace{E_{x_i}^T W_q^T W_k E_{x_j}}_{\text{内容}} + \underbrace{E_{x_i}^T W_q^T W_k R_{i-j}}_{\text{位置}} + \underbrace{u^T W_k E_{x_j}}_{\text{内容偏置}} + \underbrace{v^T W_k R_{i-j}}_{\text{位置偏置}}
$$

其中 $R_{i-j}$ 是相对位置编码。

### 2. T5 相对位置编码

简化版本，只在注意力中添加可学习的相对位置偏置：

```
attention_scores += relative_position_bias
```

相对位置偏置是一个可学习的查找表，通常限制最大距离（如 128）。

## 旋转位置编码 (RoPE)

### 核心思想

通过旋转矩阵将位置信息编码到 Q、K 向量中，使得内积自然包含相对位置信息。

### 数学推导

对于二维情况，位置 $m$ 的向量 $x_m = [x_m^{(1)}, x_m^{(2)}]$：

$$
f(x_m, m) = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix} \begin{pmatrix} x_m^{(1)} \\ x_m^{(2)} \end{pmatrix}
$$

即对向量进行旋转，旋转角度与位置 $m$ 成正比。

### 扩展到高维

将 $d$ 维向量分成 $d/2$ 对，每对应用旋转：

$$
f(x, m) = \underbrace{\begin{pmatrix} \cos m\theta_1 & -\sin m\theta_1 & 0 & 0 & \cdots \\ \sin m\theta_1 & \cos m\theta_1 & 0 & 0 & \cdots \\ 0 & 0 & \cos m\theta_2 & -\sin m\theta_2 & \cdots \\ 0 & 0 & \sin m\theta_2 & \cos m\theta_2 & \cdots \\ \vdots & \vdots & \vdots & \vdots & \ddots \end{pmatrix}}_{R_m} x
$$

其中 $\theta_i = 10000^{-2(i-1)/d}$

### 关键性质

对于位置 $m$ 和 $n$ 的向量：

$$
f(q, m)^T f(k, n) = q^T R_{m-n} k = g(q, k, m-n)
$$

**内积只依赖于相对位置 $m-n$！**

### RoPE 的优势

| 优势 | 说明 |
|------|------|
| 相对位置 | 内积自然体现相对位置 |
| 长距离衰减 | 远距离位置内积自然衰减 |
| 外推性 | 可处理训练时未见过的长度 |
| 灵活性 | 可配合各种外推技术 |

### 现代应用

- LLaMA
- PaLM
- Qwen
- Baichuan
- 大多数现代开源模型

## ALiBi

### 核心思想

Attention with Linear Biases，直接在注意力分数上添加与距离成正比的惩罚：

$$
\text{softmax}(q^T k + \underbrace{-m \cdot |i-j|}_{\text{位置偏置}})
$$

### 特点

1. **简单**: 不需要修改 Q、K 的计算
2. **有效**: 外推性极佳
3. **稳定**: 训练更稳定

### 斜率设置

每个头使用不同的斜率 $m$：

```
Head 1: m = 2^(-8)  (关注长距离)
Head 2: m = 2^(-7)
...
Head 8: m = 2^(-1)  (关注短距离)
```

### 外推性对比

```
训练长度: 2048
测试长度: 8192

正弦编码: 困惑度急剧上升
RoPE:     困惑度缓慢上升
ALiBi:    困惑度几乎不变
```

## 位置外推方法

### 问题定义

模型在长度 $L_{train}$ 上训练，如何泛化到 $L_{test} > L_{train}$？

### 1. 位置插值 (Position Interpolation, PI)

将长序列的位置缩放到训练时的范围：

$$
pos' = pos \times \frac{L_{train}}{L_{test}}
$$

**示例**:
```
训练长度: 2048
测试长度: 8192

原始位置: 0, 1, 2, ..., 8191
插值后:   0, 0.25, 0.5, ..., 2047.75
```

**优点**: 简单有效
**缺点**: 需要微调

### 2. NTK-aware 外推

不直接缩放位置，而是调整 RoPE 的 base 频率：

```
原始: θ_i = base^(-2i/d)
修改: θ_i' = (base × λ)^(-2i/d)
```

其中 $\lambda$ 是缩放因子。

### 3. YaRN (Yet another RoPE extensioN)

结合温度缩放和频率调整：

```
1. 对高频维度进行温度缩放 (temperature scaling)
2. 对低频维度进行 NTK-aware 调整
3. 注意力计算时乘以温度因子
```

### 4. 动态 NTK

根据当前序列长度动态调整：

```python
if seq_len <= train_len:
    scale = 1.0
else:
    scale = seq_len / train_len
    # 动态调整 RoPE base
```

### 外推方法对比

| 方法 | 需要微调 | 复杂度 | 效果 |
|------|----------|--------|------|
| PI | 是 | 低 | 良好 |
| NTK-aware | 可选 | 低 | 良好 |
| YaRN | 推荐 | 中 | 优秀 |
| 动态 NTK | 否 | 低 | 良好 |

## 方法对比与选型

### 综合对比表

| 方法 | 类型 | 外推性 | 训练稳定性 | 实现复杂度 | 代表模型 |
|------|------|--------|------------|------------|----------|
| 正弦 | 绝对 | 中 | 中 | 低 | 原始 Transformer |
| 可学习 | 绝对 | 差 | 高 | 低 | BERT |
| 相对 (XL) | 相对 | 好 | 中 | 高 | Transformer-XL |
| T5 偏置 | 相对 | 差 | 高 | 低 | T5 |
| RoPE | 旋转 | 好 | 高 | 中 | LLaMA, Qwen |
| ALiBi | 线性偏置 | 优秀 | 优秀 | 低 | BLOOM, MPT |

### 选型建议

| 场景 | 推荐方案 |
|------|----------|
| 追求简单稳定 | ALiBi |
| 追求最佳性能 | RoPE + YaRN |
| 快速原型 | 正弦位置编码 |
| 固定长度任务 | 可学习位置编码 |
| 超长上下文 (>100K) | RoPE + 动态 NTK |

## 代码实现

### 1. 正弦位置编码

```python
import torch
import torch.nn as nn
import math

class SinusoidalPositionalEncoding(nn.Module):
    def __init__(self, d_model, max_seq_len=5000, dropout=0.1):
        super().__init__()
        self.dropout = nn.Dropout(dropout)

        # 预计算位置编码
        pe = torch.zeros(max_seq_len, d_model)
        position = torch.arange(0, max_seq_len, dtype=torch.float).unsqueeze(1)

        # 计算分母项
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() *
            (-math.log(10000.0) / d_model)
        )

        # 偶数维: sin
        pe[:, 0::2] = torch.sin(position * div_term)
        # 奇数维: cos
        pe[:, 1::2] = torch.cos(position * div_term)

        # 注册为 buffer (不参与训练)
        self.register_buffer('pe', pe.unsqueeze(0))

    def forward(self, x):
        """
        x: [batch_size, seq_len, d_model]
        """
        x = x + self.pe[:, :x.size(1)]
        return self.dropout(x)
```

### 2. RoPE 实现

```python
class RotaryPositionalEmbedding(nn.Module):
    """旋转位置编码实现"""
    def __init__(self, dim, max_seq_len=2048, base=10000):
        super().__init__()

        # 计算旋转角度
        theta = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))

        # 预计算位置 × 角度
        positions = torch.arange(max_seq_len)
        angles = torch.outer(positions, theta)  # [max_seq_len, dim/2]

        # 注册为 buffer
        self.register_buffer('cos_cached', torch.cos(angles))
        self.register_buffer('sin_cached', torch.sin(angles))

    def forward(self, x, seq_len):
        """
        对输入 x 应用旋转位置编码
        x: [batch_size, num_heads, seq_len, head_dim]
        """
        # 将最后维度分成两半
        x1, x2 = x[..., ::2], x[..., 1::2]

        # 获取对应位置的 cos, sin
        cos = self.cos_cached[:seq_len]  # [seq_len, dim/2]
        sin = self.sin_cached[:seq_len]

        # 调整维度以匹配 x
        cos = cos.unsqueeze(0).unsqueeze(0)  # [1, 1, seq_len, dim/2]
        sin = sin.unsqueeze(0).unsqueeze(0)

        # 应用旋转
        # [x1, x2] 旋转 mθ 角度:
        # x1' = x1 * cos(mθ) - x2 * sin(mθ)
        # x2' = x1 * sin(mθ) + x2 * cos(mθ)
        x_rotated = torch.stack([
            x1 * cos - x2 * sin,
            x1 * sin + x2 * cos
        ], dim=-1)  # [..., dim/2, 2]

        return x_rotated.flatten(-2)  # [..., dim]


def apply_rotary_pos_emb(q, k, cos, sin):
    """对 Q, K 应用 RoPE"""
    def rotate_half(x):
        x1, x2 = x[..., ::2], x[..., 1::2]
        return torch.stack([-x2, x1], dim=-1).flatten(-2)

    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed
```

### 3. ALiBi 实现

```python
class ALiBiPositionalBias(nn.Module):
    """ALiBi 位置偏置"""
    def __init__(self, num_heads, max_seq_len=2048):
        super().__init__()
        self.num_heads = num_heads

        # 计算每个头的斜率
        # m = 2^(-8*i/num_heads) for i in range(num_heads)
        slopes = torch.pow(2, -torch.arange(0, num_heads) * 8 / num_heads)

        # 计算距离矩阵
        distance = torch.arange(max_seq_len).unsqueeze(0) - torch.arange(max_seq_len).unsqueeze(1)
        distance = distance.abs().unsqueeze(0)  # [1, seq_len, seq_len]

        # 计算偏置: -m * distance
        bias = -distance * slopes.unsqueeze(1).unsqueeze(1)  # [num_heads, seq_len, seq_len]

        self.register_buffer('bias', bias)

    def forward(self, attention_scores):
        """
        attention_scores: [batch_size, num_heads, seq_len, seq_len]
        """
        seq_len = attention_scores.size(-1)
        return attention_scores + self.bias[:, :seq_len, :seq_len]
```

## 为什么需要位置编码

### 问题背景

Self-Attention 是**位置无关 (Permutation Invariant)** 的：

```
输入: [A, B, C] → Attention 输出
输入: [B, A, C] → 相同的 Attention 输出（只是顺序变了）

这意味着模型无法区分 "A 在 B 前面" 和 "B 在 A 前面"
```

### 解决方案

为每个位置添加唯一的标识，让模型感知顺序信息。

---

## 绝对位置编码

### 1. 正弦位置编码 (Sinusoidal)

Transformer 原始论文提出：

$$
\begin{aligned}
PE_{(pos, 2i)} &= \sin\left(\frac{pos}{10000^{2i/d_{model}}}\right) \\
PE_{(pos, 2i+1)} &= \cos\left(\frac{position}{10000^{2i/d_{model}}}\right)
\end{aligned}
$$

其中：
- $pos$: 位置索引
- $i$: 维度索引
- $d_{model}$: 模型维度

### 波长分析

对于维度 $i$，波长为：

$$
\lambda_i = 2\pi \cdot 10000^{2i/d_{model}}
$$

- 小 $i$ (低维): 长波长，捕获长距离模式
- 大 $i$ (高维): 短波长，捕获细粒度变化

```
维度 0:  波长 ≈ 2π      (高频，精细位置)
维度 128: 波长 ≈ 2π×100 (中频)
维度 512: 波长 ≈ 2π×10000 (低频，粗粒度位置)
```

### 2. 可学习位置编码 (Learned)

BERT 采用的方式：

```python
self.position_embeddings = nn.Embedding(max_seq_len, d_model)
```

**优点**:
- 灵活性高，数据驱动
- 可以针对特定任务优化

**缺点**:
- 外推性差，无法处理训练时未见过的长度
- 增加了参数量

### 绝对位置编码对比

| 特性 | 正弦 | 可学习 |
|------|------|--------|
| 参数量 | 0 | $max\_seq \times d_{model}$ |
| 外推性 | 好 | 差 |
| 灵活性 | 固定 | 可优化 |
| 相对位置 | 可表示 | 不可直接表示 |

## 相对位置编码

### 核心思想

关注 token 之间的相对距离而非绝对位置：

```
绝对: token A 在位置 5，token B 在位置 8
相对: token B 距离 token A 有 +3 个位置
```

### 1. Transformer-XL 相对位置编码

在注意力计算中引入相对位置偏置：

$$
A_{ij} = \underbrace{E_{x_i}^T W_q^T W_k E_{x_j}}_{\text{内容}} + \underbrace{E_{x_i}^T W_q^T W_k R_{i-j}}_{\text{位置}} + \underbrace{u^T W_k E_{x_j}}_{\text{内容偏置}} + \underbrace{v^T W_k R_{i-j}}_{\text{位置偏置}}
$$

其中 $R_{i-j}$ 是相对位置编码。

### 2. T5 相对位置编码

简化版本，只在注意力中添加可学习的相对位置偏置：

```
attention_scores += relative_position_bias
```

相对位置偏置是一个可学习的查找表，通常限制最大距离（如 128）。

## 旋转位置编码 (RoPE)

### 核心思想

通过旋转矩阵将位置信息编码到 Q、K 向量中，使得内积自然包含相对位置信息。

### 数学推导

对于二维情况，位置 $m$ 的向量 $x_m = [x_m^{(1)}, x_m^{(2)}]$：

$$
f(x_m, m) = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix} \begin{pmatrix} x_m^{(1)} \\ x_m^{(2)} \end{pmatrix}
$$

即对向量进行旋转，旋转角度与位置 $m$ 成正比。

### 扩展到高维

将 $d$ 维向量分成 $d/2$ 对，每对应用旋转：

$$
f(x, m) = \underbrace{\begin{pmatrix} \cos m\theta_1 & -\sin m\theta_1 & 0 & 0 & \cdots \\ \sin m\theta_1 & \cos m\theta_1 & 0 & 0 & \cdots \\ 0 & 0 & \cos m\theta_2 & -\sin m\theta_2 & \cdots \\ 0 & 0 & \sin m\theta_2 & \cos m\theta_2 & \cdots \\ \vdots & \vdots & \vdots & \vdots & \ddots \end{pmatrix}}_{R_m} x
$$

其中 $\theta_i = 10000^{-2(i-1)/d}$

### 关键性质

对于位置 $m$ 和 $n$ 的向量：

$$
f(q, m)^T f(k, n) = q^T R_{m-n} k = g(q, k, m-n)
$$

**内积只依赖于相对位置 $m-n$！**

### RoPE 的优势

| 优势 | 说明 |
|------|------|
| 相对位置 | 内积自然体现相对位置 |
| 长距离衰减 | 远距离位置内积自然衰减 |
| 外推性 | 可处理训练时未见过的长度 |
| 灵活性 | 可配合各种外推技术 |

### 现代应用

- LLaMA
- PaLM
- Qwen
- Baichuan
- 大多数现代开源模型

## ALiBi

### 核心思想

Attention with Linear Biases，直接在注意力分数上添加与距离成正比的惩罚：

$$
\text{softmax}(q^T k + \underbrace{-m \cdot |i-j|}_{\text{位置偏置}})
$$

### 特点

1. **简单**: 不需要修改 Q、K 的计算
2. **有效**: 外推性极佳
3. **稳定**: 训练更稳定

### 斜率设置

每个头使用不同的斜率 $m$：

```
Head 1: m = 2^(-8)  (关注长距离)
Head 2: m = 2^(-7)
...
Head 8: m = 2^(-1)  (关注短距离)
```

### 外推性对比

```
训练长度: 2048
测试长度: 8192

正弦编码: 困惑度急剧上升
RoPE:     困惑度缓慢上升
ALiBi:    困惑度几乎不变
```

## 位置外推方法

### 问题定义

模型在长度 $L_{train}$ 上训练，如何泛化到 $L_{test} > L_{train}$？

### 1. 位置插值 (Position Interpolation, PI)

将长序列的位置缩放到训练时的范围：

$$
pos' = pos \times \frac{L_{train}}{L_{test}}
$$

**示例**:
```
训练长度: 2048
测试长度: 8192

原始位置: 0, 1, 2, ..., 8191
插值后:   0, 0.25, 0.5, ..., 2047.75
```

**优点**: 简单有效
**缺点**: 需要微调

### 2. NTK-aware 外推

不直接缩放位置，而是调整 RoPE 的 base 频率：

```
原始: θ_i = base^(-2i/d)
修改: θ_i' = (base × λ)^(-2i/d)
```

其中 $\lambda$ 是缩放因子。

### 3. YaRN (Yet another RoPE extensioN)

结合温度缩放和频率调整：

```
1. 对高频维度进行温度缩放 (temperature scaling)
2. 对低频维度进行 NTK-aware 调整
3. 注意力计算时乘以温度因子
```

### 4. 动态 NTK

根据当前序列长度动态调整：

```python
if seq_len <= train_len:
    scale = 1.0
else:
    scale = seq_len / train_len
    # 动态调整 RoPE base
```

### 外推方法对比

| 方法 | 需要微调 | 复杂度 | 效果 |
|------|----------|--------|------|
| PI | 是 | 低 | 良好 |
| NTK-aware | 可选 | 低 | 良好 |
| YaRN | 推荐 | 中 | 优秀 |
| 动态 NTK | 否 | 低 | 良好 |

## 代码实现

### 1. 正弦位置编码

```python
import torch
import torch.nn as nn
import math

class SinusoidalPositionalEncoding(nn.Module):
    def __init__(self, d_model, max_seq_len=5000, dropout=0.1):
        super().__init__()
        self.dropout = nn.Dropout(dropout)

        # 预计算位置编码
        pe = torch.zeros(max_seq_len, d_model)
        position = torch.arange(0, max_seq_len, dtype=torch.float).unsqueeze(1)

        # 计算分母项
        div_term = torch.exp(
            torch.arange(0, d_model, 2).float() *
            (-math.log(10000.0) / d_model)
        )

        # 偶数维: sin
        pe[:, 0::2] = torch.sin(position * div_term)
        # 奇数维: cos
        pe[:, 1::2] = torch.cos(position * div_term)

        # 注册为 buffer (不参与训练)
        self.register_buffer('pe', pe.unsqueeze(0))

    def forward(self, x):
        """
        x: [batch_size, seq_len, d_model]
        """
        x = x + self.pe[:, :x.size(1)]
        return self.dropout(x)
```

### 2. RoPE 实现

```python
class RotaryPositionalEmbedding(nn.Module):
    """旋转位置编码实现"""
    def __init__(self, dim, max_seq_len=2048, base=10000):
        super().__init__()

        # 计算旋转角度
        theta = 1.0 / (base ** (torch.arange(0, dim, 2).float() / dim))

        # 预计算位置 × 角度
        positions = torch.arange(max_seq_len)
        angles = torch.outer(positions, theta)  # [max_seq_len, dim/2]

        # 注册为 buffer
        self.register_buffer('cos_cached', torch.cos(angles))
        self.register_buffer('sin_cached', torch.sin(angles))

    def forward(self, x, seq_len):
        """
        对输入 x 应用旋转位置编码
        x: [batch_size, num_heads, seq_len, head_dim]
        """
        # 将最后维度分成两半
        x1, x2 = x[..., ::2], x[..., 1::2]

        # 获取对应位置的 cos, sin
        cos = self.cos_cached[:seq_len]  # [seq_len, dim/2]
        sin = self.sin_cached[:seq_len]

        # 调整维度以匹配 x
        cos = cos.unsqueeze(0).unsqueeze(0)  # [1, 1, seq_len, dim/2]
        sin = sin.unsqueeze(0).unsqueeze(0)

        # 应用旋转
        # [x1, x2] 旋转 mθ 角度:
        # x1' = x1 * cos(mθ) - x2 * sin(mθ)
        # x2' = x1 * sin(mθ) + x2 * cos(mθ)
        x_rotated = torch.stack([
            x1 * cos - x2 * sin,
            x1 * sin + x2 * cos
        ], dim=-1)  # [..., dim/2, 2]

        return x_rotated.flatten(-2)  # [..., dim]


def apply_rotary_pos_emb(q, k, cos, sin):
    """对 Q, K 应用 RoPE"""
    def rotate_half(x):
        x1, x2 = x[..., ::2], x[..., 1::2]
        return torch.stack([-x2, x1], dim=-1).flatten(-2)

    q_embed = (q * cos) + (rotate_half(q) * sin)
    k_embed = (k * cos) + (rotate_half(k) * sin)
    return q_embed, k_embed
```

### 3. ALiBi 实现

```python
class ALiBiPositionalBias(nn.Module):
    """ALiBi 位置偏置"""
    def __init__(self, num_heads, max_seq_len=2048):
        super().__init__()
        self.num_heads = num_heads

        # 计算每个头的斜率
        # m = 2^(-8*i/num_heads) for i in range(num_heads)
        slopes = torch.pow(2, -torch.arange(0, num_heads) * 8 / num_heads)

        # 计算距离矩阵
        distance = torch.arange(max_seq_len).unsqueeze(0) - torch.arange(max_seq_len).unsqueeze(1)
        distance = distance.abs().unsqueeze(0)  # [1, seq_len, seq_len]

        # 计算偏置: -m * distance
        bias = -distance * slopes.unsqueeze(1).unsqueeze(1)  # [num_heads, seq_len, seq_len]

        self.register_buffer('bias', bias)

    def forward(self, attention_scores):
        """
        attention_scores: [batch_size, num_heads, seq_len, seq_len]
        """
        seq_len = attention_scores.size(-1)
        return attention_scores + self.bias[:, :seq_len, :seq_len]
```
