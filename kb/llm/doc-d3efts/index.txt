## 量化概述

### 核心思想

将模型参数从高精度（FP32/FP16）降到低精度（INT8/INT4），减少显存和加速推理。

| 精度 | 每参数字节数 | 模型大小减少 |
|------|-------------|-------------|
| FP32 | 4 bytes | 1× (基准) |
| FP16 | 2 bytes | 2× |
| INT8 | 1 byte | 4× |
| INT4 | 0.5 byte | 8× |

### 权衡

| 维度 | 高精度 | 低精度 |
|------|--------|--------|
| 显存 | 大 | 小 |
| 推理速度 | 慢 | 快 |
| 模型质量 | 高 | 可能下降 |
| 实现复杂度 | 简单 | 较复杂 |

---

## 量化方法

### 1. 线性量化

将浮点数映射到整数：

$$
x_{\text{quant}} = \text{round}\left(\frac{x}{\text{scale}} + \text{zero\_point}\right)
$$

反量化：

$$
x_{\text{dequant}} = (x_{\text{quant}} - \text{zero\_point}) \times \text{scale}
$$

### 2. 对称量化

zero_point = 0，只存储 scale：

$$
x_{\text{quant}} = \text{round}\left(\frac{x}{\text{scale}}\right)
$$

### 3. 非对称量化

zero_point ≠ 0，需要存储 scale 和 zero_point。

| 类型 | 优点 | 缺点 |
|------|------|------|
| 对称 | 简单，存储少 | 数据偏移时精度低 |
| 非对称 | 精度更高 | 需要额外存储 zero_point |

## PTQ 量化

### 核心思想

Post-Training Quantization，无需重新训练，直接量化已训练模型。

### 量化流程

```python
def ptq_quantize(model, calibration_data):
    """
    model: FP16 模型
    calibration_data: 校准数据
    """
    # 1. 收集激活统计
    with torch.no_grad():
        for batch in calibration_data:
            _ = model(batch)
            # 收集每层的激活范围

    # 2. 计算量化参数
    for name, module in model.named_modules():
        if isinstance(module, (nn.Linear, nn.Conv2d)):
            weight = module.weight.data
            scale, zero_point = compute_quant_params(weight)

            # 3. 量化权重
            module.weight.data = quantize(weight, scale, zero_point)
            module.weight.scale = scale
            module.weight.zero_point = zero_point

    return model
```

### 量化粒度

| 粒度 | 说明 | 精度 | 灵活性 |
|------|------|------|--------|
| **Per-Tensor** | 整个张量共享 scale | 低 | 低 |
| **Per-Channel** | 输出通道独立 scale | 中 | 中 |
| **Per-Token** | 每个 token 独立 scale | 高 | 高 |

## GPTQ 量化

### 核心思想

针对 LLM 特化的量化方法，通过量化所有权重并解量化来减少误差。

### GPTQ 算法

```python
def gptq_quantize(weight, wbits=4, groupsize=128):
    """
    weight: [in_features, out_features]
    wbits: 量化位数 (4)
    groupsize: 量化分组大小
    """
    # 1. 分块权重
    blocks = weight.view(-1, groupsize)

    # 2. 计算每块的量化参数
    for i, block in enumerate(blocks):
        # 2.1 Hessian 计算
        H = compute_hessian(block)

        # 2.2 找最优量化
        quant_block, scale, zero = find_optimal_quantization(block, H, wbits)

        blocks[i] = quant_block

    # 3. 重新组合
    quant_weight = blocks.view(weight.shape)

    return quant_weight
```

### Hessian 的重要性

量化误差的权重由 Hessian 矩阵决定，Hessian 反映了权重对输出的敏感度。

$$
\text{Error} = \sum_{i} H_{ii} \cdot (w_i - w_{\text{quant}, i})^2
$$

Hessian 对角元素大的权重需要更精确量化。

## AWQ 激活感知量化

### 核心思想

Activation-aware Weight Quantization，根据激活的重要性调整量化精度。

### AWQ 流程

```python
def awq_quantize(model, calibration_data):
    """
    1. 收集激活统计
    2. 计算每层的重要性
    3. 根据重要性分配量化位数
    """
    # 1. 收集激活范围
    activation_stats = collect_activation_stats(model, calibration_data)

    # 2. 计算层重要性
    layer_importance = {}
    for name, stats in activation_stats.items():
        # 基于 Hessian 和激活范围计算重要性
        importance = compute_importance(stats)
        layer_importance[name] = importance

    # 3. 分配量化位数
    total_bits = 0
    for name in sorted(layer_importance, key=lambda x: layer_importance[x], reverse=True):
        # 重要层分配更多位
        bits = allocate_bits(layer_importance[name], remaining_bits)
        quantize_layer(model[name], bits)
        total_bits += bits
```

### 混合精度

| 重要性 | 量化位数 | 比例 |
|--------|----------|------|
| 前 10% | INT8/FP16 | 较高 |
| 中间 50% | INT4 | 中等 |
| 后 40% | INT4 | 较低 |

## FP8 量化

### FP8 格式

| 格式 | E5M2 | E4M3 |
|------|-------|-------|
| 指数位数 | 5 bits | 4 bits |
| 尾数位数 | 2 bits | 3 bits |
| 精度 | 动态范围高 | 精度高 |

### FP8 量化流程

```python
def fp8_quantize(tensor, format='E4M3'):
    """
    E4M3: 4 bits 指数, 3 bits 尾数
    E5M2: 5 bits 指数, 2 bits 尾数
    """
    if format == 'E4M3':
        # E4M3 量化
        exponent_bits = 4
        mantissa_bits = 3
    else:  # E5M2
        exponent_bits = 5
        mantissa_bits = 2

    # 提取符号、指数、尾数
    sign = torch.sign(tensor)
    abs_value = torch.abs(tensor)

    # 计算指数和尾数
    log_value = torch.log2(abs_value)
    exponent = torch.clamp(log_value.floor(),
                            -2**exponent_bits,
                            2**exponent_bits - 1)
    mantissa = (abs_value / 2**exponent - 1) * (2**mantissa_bits)
    mantissa = torch.clamp(mantissa, 0, 2**mantissa_bits - 1)

    # 组合
    fp8_value = sign * 2**exponent * (1 + mantissa / 2**mantissa_bits)

    return fp8_value
```

### FP8 优势

| 特性 | FP16 | FP8-E4M3 | FP8-E5M2 |
|------|------|-----------|-----------|
| 内存占用 | 2× | 4× | 4× |
| 动态范围 | 标准 | 更大 | 标准 |
| 精度 | 标准 | 较低 | 更高 |
| 硬件支持 | 广泛 | H100/A100+ | H100/A100+ |

## 面试要点

### 高频问题

**Q1: 量化为什么能加速推理？**

A:
1. **减少显存**: 低精度占用更少内存
2. **硬件加速**: INT8/FP8 有专用加速单元
3. **减少内存访问**: 更少数据传输

**Q2: PTQ vs GPTQ vs AWQ 的区别？**

A:
| 方法 | 是否需要训练 | 特点 |
|------|-------------|------|
| PTQ | 否 | 简单，但精度较低 |
| GPTQ | 否 | 基于 Hessian，精度较好 |
| AWQ | 否 | 激活感知，混合精度 |

**Q3: GPTQ 的 Hessian 为什么重要？**

A: Hessian 衡量权重对输出的敏感度：
- Hessian 大 → 权重变化对输出影响大 → 需要更精确量化
- Hessian 小 → 权重变化对输出影响小 → 可以粗糙量化

**Q4: FP8 E4M3 vs E5M2 如何选择？**

A:
- **E4M3**: 动态范围大，适合权重（范围不定）
- **E5M2**: 精度高，适合激活（通常范围固定）

**Q5: 混合精度量化的思想？**

A: 根据重要性分配不同精度：
1. 重要层：FP16/INT8
2. 中间层：INT4
3. 不重要层：更低精度

在总参数量限制下最大化效果。

            scale, zero_point = compute_quant_params(weight)

            # 3. 量化权重
            module.weight.data = quantize(weight, scale, zero_point)
            module.weight.scale = scale
            module.weight.zero_point = zero_point

    return model
```

### 量化粒度

| 粒度 | 说明 | 精度 | 灵活性 |
|------|------|------|--------|
| **Per-Tensor** | 整个张量共享 scale | 低 | 低 |
| **Per-Channel** | 输出通道独立 scale | 中 | 中 |
| **Per-Token** | 每个 token 独立 scale | 高 | 高 |

## GPTQ 量化

### 核心思想

针对 LLM 特化的量化方法，通过量化所有权重并解量化来减少误差。

### GPTQ 算法

```python
def gptq_quantize(weight, wbits=4, groupsize=128):
    """
    weight: [in_features, out_features]
    wbits: 量化位数 (4)
    groupsize: 量化分组大小
    """
    # 1. 分块权重
    blocks = weight.view(-1, groupsize)

    # 2. 计算每块的量化参数
    for i, block in enumerate(blocks):
        # 2.1 Hessian 计算
        H = compute_hessian(block)

        # 2.2 找最优量化
        quant_block, scale, zero = find_optimal_quantization(block, H, wbits)

        blocks[i] = quant_block

    # 3. 重新组合
    quant_weight = blocks.view(weight.shape)

    return quant_weight
```

### Hessian 的重要性

量化误差的权重由 Hessian 矩阵决定，Hessian 反映了权重对输出的敏感度。

$$
\text{Error} = \sum_{i} H_{ii} \cdot (w_i - w_{\text{quant}, i})^2
$$

Hessian 对角元素大的权重需要更精确量化。

## AWQ 激活感知量化

### 核心思想

Activation-aware Weight Quantization，根据激活的重要性调整量化精度。

### AWQ 流程

```python
def awq_quantize(model, calibration_data):
    """
    1. 收集激活统计
    2. 计算每层的重要性
    3. 根据重要性分配量化位数
    """
    # 1. 收集激活范围
    activation_stats = collect_activation_stats(model, calibration_data)

    # 2. 计算层重要性
    layer_importance = {}
    for name, stats in activation_stats.items():
        # 基于 Hessian 和激活范围计算重要性
        importance = compute_importance(stats)
        layer_importance[name] = importance

    # 3. 分配量化位数
    total_bits = 0
    for name in sorted(layer_importance, key=lambda x: layer_importance[x], reverse=True):
        # 重要层分配更多位
        bits = allocate_bits(layer_importance[name], remaining_bits)
        quantize_layer(model[name], bits)
        total_bits += bits
```

### 混合精度

| 重要性 | 量化位数 | 比例 |
|--------|----------|------|
| 前 10% | INT8/FP16 | 较高 |
| 中间 50% | INT4 | 中等 |
| 后 40% | INT4 | 较低 |

## FP8 量化

### FP8 格式

| 格式 | E5M2 | E4M3 |
|------|-------|-------|
| 指数位数 | 5 bits | 4 bits |
| 尾数位数 | 2 bits | 3 bits |
| 精度 | 动态范围高 | 精度高 |

### FP8 量化流程

```python
def fp8_quantize(tensor, format='E4M3'):
    """
    E4M3: 4 bits 指数, 3 bits 尾数
    E5M2: 5 bits 指数, 2 bits 尾数
    """
    if format == 'E4M3':
        # E4M3 量化
        exponent_bits = 4
        mantissa_bits = 3
    else:  # E5M2
        exponent_bits = 5
        mantissa_bits = 2

    # 提取符号、指数、尾数
    sign = torch.sign(tensor)
    abs_value = torch.abs(tensor)

    # 计算指数和尾数
    log_value = torch.log2(abs_value)
    exponent = torch.clamp(log_value.floor(),
                            -2**exponent_bits,
                            2**exponent_bits - 1)
    mantissa = (abs_value / 2**exponent - 1) * (2**mantissa_bits)
    mantissa = torch.clamp(mantissa, 0, 2**mantissa_bits - 1)

    # 组合
    fp8_value = sign * 2**exponent * (1 + mantissa / 2**mantissa_bits)

    return fp8_value
```

### FP8 优势

| 特性 | FP16 | FP8-E4M3 | FP8-E5M2 |
|------|------|-----------|-----------|
| 内存占用 | 2× | 4× | 4× |
| 动态范围 | 标准 | 更大 | 标准 |
| 精度 | 标准 | 较低 | 更高 |
| 硬件支持 | 广泛 | H100/A100+ | H100/A100+ |

## 面试要点

### 高频问题

**Q1: 量化为什么能加速推理？**

A:
1. **减少显存**: 低精度占用更少内存
2. **硬件加速**: INT8/FP8 有专用加速单元
3. **减少内存访问**: 更少数据传输

**Q2: PTQ vs GPTQ vs AWQ 的区别？**

A:
| 方法 | 是否需要训练 | 特点 |
|------|-------------|------|
| PTQ | 否 | 简单，但精度较低 |
| GPTQ | 否 | 基于 Hessian，精度较好 |
| AWQ | 否 | 激活感知，混合精度 |

**Q3: GPTQ 的 Hessian 为什么重要？**

A: Hessian 衡量权重对输出的敏感度：
- Hessian 大 → 权重变化对输出影响大 → 需要更精确量化
- Hessian 小 → 权重变化对输出影响小 → 可以粗糙量化

**Q4: FP8 E4M3 vs E5M2 如何选择？**

A:
- **E4M3**: 动态范围大，适合权重（范围不定）
- **E5M2**: 精度高，适合激活（通常范围固定）

**Q5: 混合精度量化的思想？**

A: 根据重要性分配不同精度：
1. 重要层：FP16/INT8
2. 中间层：INT4
3. 不重要层：更低精度

在总参数量限制下最大化效果。
