o1 类推理模型

核心突破

从 System-1（直觉、快速）到 System-2（思考、深度）。

特性System-1System-2
思考方式直觉、快速推理、反思
时间毫秒级秒级甚至分钟
可解释性弱强（显示思考链）
适用场景简单问答复杂推理

o1 类模型特点

问题: "房间里有 5 个苹果，拿走 2 个，还剩多少？"

System-1 (传统 LLM):
输入: "房间里有 5 个苹果，拿走 2 个，还剩多少？"
输出: "3 个"  (直接输出)

o1 类模型:
输入: "房间里有 5 个苹果，拿走 2 个，还剩多少？"
思考: "让我一步步思考：
1. 初始有 5 个苹果
2. 拿走 2 个
3. 还剩 = 5 - 2 = 3"
输出: "还剩 3 个"

模型合并与编辑

Task Arithmetic

# 模型算术
def task_arithmetic(model_a, model_b, operation='add'):
    """
    model_a: 擅长任务 A
    model_b: 擅长任务 B
    """
    # 获取模型参数
    theta_a = get_model_params(model_a)
    theta_b = get_model_params(model_b)

    # 归一化
    theta_a_norm = theta_a / ||theta_a||
    theta_b_norm = theta_b / ||theta_b||

    # 执行运算
    if operation == 'add':
        theta_merged = theta_a_norm + theta_b_norm
    elif operation == 'subtract':
        theta_merged = theta_a_norm - theta_b_norm
    elif operation == 'average':
        theta_merged = (theta_a_norm + theta_b_norm) / 2

    # 反归一化
    return scale_to_original(theta_merged, theta_a)

DARE (Drop And REscale)

def dare_merge(model_a, model_b, drop_rate=0.2):
    """
    DARE: 随机丢弃部分参数，重新缩放剩余参数
    """
    params_a = get_model_params(model_a)
    params_b = get_model_params(model_b)

    # 随机丢弃
    mask = torch.rand_like(params_a) > drop_rate

    # 丢弃部分，重新缩放
    merged = mask * (params_a + params_b) / (1 - drop_rate)

    return merged

面试要点

高频问题

Q1: o1 类模型相比传统 LLM 的核心区别？

A:

  1. 思考方式: o1 深度推理，传统 LLM 快速直觉
  2. 训练: o1 使用过程奖励，鼓励逐步推理
  3. 输出: o1 显示思考链，可解释
  4. 适用场景: o1 适合复杂推理任务

Q2: MoE 的优势和挑战？

A: 优势:

  1. 大参数量但低计算量
  2. 擅长处理不同任务
  3. 可扩展性强

挑战:

  1. 训练不稳定
  2. 负载均衡问题
  3. 推理开销（路由）

Q3: 长上下文的主要技术方向？

A:

  1. RoPE 外推: YaRN, NTK-aware
  2. 滑动窗口: 动态窗口
  3. 层次注意力: 分块处理
  4. RAG: 外部知识补充

Q4: Agent 系统的核心组件？

A:

  1. ReAct: 推理-行动循环
  2. 工具使用: Function Calling
  3. 记忆: 长期记忆管理
  4. 规划: 任务分解与执行

Q5: 模型合并的典型场景？

A:

  1. 任务迁移: A + B = A 擅长 + B
  2. 遗忘: A - B = A 擅长 - B 不需要的能力
  3. 语言切换: 平均不同语言的模型
  4. 能力整合: 合并不同能力的模型
