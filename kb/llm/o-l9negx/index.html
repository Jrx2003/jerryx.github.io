o1 模型概述

什么是 o1 模型

OpenAI 于 2024 年 9 月发布的 o1 系列模型（o1-preview、o1-mini）代表了 LLM 推理能力的重要突破。

传统 LLM: 问题 → 直接回答
o1 模型:   问题 → 思考过程 (Chain-of-Thought) → 回答

特点:
- 生成详细的思考过程
- 可以回溯和修正思路
- 在数学、代码、科学推理上显著提升

核心思想

传统 Scaling Law: 提升模型性能主要靠
- 增加模型参数
- 增加训练数据
- 增加训练计算

o1 的创新: Test-Time Scaling
- 在推理时增加"思考时间"
- 生成更长的推理链
- 通过验证和回溯提高答案质量

Test-Time Scaling

概念

Test-Time Scaling 是指在推理阶段投入更多计算资源来提升模型性能。

# 传统推理
answer = model.generate(prompt, max_tokens=100)

# Test-Time Scaling
def test_time_scaling(model, prompt, max_iterations=10):
    """通过多步推理提升答案质量"""

    thoughts = []
    for i in range(max_iterations):
        # 生成思考步骤
        thought_prompt = f"""
问题: {prompt}

当前思路: {' '.join(thoughts)}

下一步思考:"""

        thought = model.generate(thought_prompt, temperature=0.7)
        thoughts.append(thought)

        # 检查是否可以给出答案
        answer_prompt = f"""
问题: {prompt}

思考过程:
{chr(10).join(f'{i+1}. {t}' for i, t in enumerate(thoughts))}

基于以上思考，给出最终答案:"""

        answer = model.generate(answer_prompt)

        # 验证答案
        if verify_answer(answer, prompt):
            return answer

    return answer  # 返回最后一次的答案

具体技术

1. Self-Consistency

通过采样多个推理路径，选择最常见的答案。

def self_consistency_decoding(model, prompt, n_samples=10):
    """自洽性解码"""

    # 采样多个推理路径
    answers = []
    for _ in range(n_samples):
        # 使用不同温度采样
        temperature = random.uniform(0.5, 1.0)
        answer = model.generate(prompt, temperature=temperature)
        answers.append(answer)

    # 选择最常见的答案
    from collections import Counter
    answer_counts = Counter(answers)
    most_common = answer_counts.most_common(1)[0][0]

    return most_common

2. Step-by-Step Verification

逐步验证推理过程的正确性。

def step_by_step_verification(model, problem, num_steps=5):
    """逐步验证"""

    steps = []

    for step_idx in range(num_steps):
        # 生成下一步
        step_prompt = f"""
问题: {problem}
已完成的步骤:
{chr(10).join(steps)}

请继续下一步推理:"""

        step = model.generate(step_prompt)

        # 验证这一步的正确性
        verify_prompt = f"""
问题: {problem}
当前步骤: {step}

这一步是否正确？请回答 Yes 或 No 并解释原因。"""

        verification = model.generate(verify_prompt)

        if "No" in verification:
            # 重新生成这一步
            correction_prompt = f"""
问题: {problem}
之前的步骤有错误。

请重新思考这一步:"""
            step = model.generate(correction_prompt)

        steps.append(step)

    # 生成最终答案
    final_prompt = f"""
问题: {problem}
推理步骤:
{chr(10).join(steps)}

最终答案:"""

    return model.generate(final_prompt)

3. Tree of Thoughts (ToT)

使用树形搜索探索多条推理路径。

class TreeOfThoughts:
    """思维树搜索"""

    def __init__(self, model, breadth=3, depth=5):
        self.model = model
        self.breadth = breadth  # 每层的分支数
        self.depth = depth      # 最大深度

    def search(self, problem):
        """执行树搜索"""

        # 初始化根节点
        root = ThoughtNode(content=problem, score=0)

        for depth in range(self.depth):
            # 扩展当前层的所有节点
            current_nodes = self.get_nodes_at_depth(root, depth)

            for node in current_nodes:
                # 生成多个可能的下一步
                candidates = self.generate_candidates(node, k=self.breadth)

                for candidate in candidates:
                    # 评估候选节点的质量
                    score = self.evaluate(candidate, problem)

                    child = ThoughtNode(
                        content=candidate,
                        parent=node,
                        score=score
                    )
                    node.children.append(child)

        # 选择最佳路径
        best_path = self.select_best_path(root)
        return self.extract_answer(best_path)

    def generate_candidates(self, node, k=3):
        """生成 k 个候选思考"""

        candidates = []
        for i in range(k):
            prompt = f"""
问题: {node.root.content}
当前思路: {node.get_path_content()}

请提供第 {i+1} 个可能的下一步思考:"""

            candidate = self.model.generate(prompt, temperature=0.8)
            candidates.append(candidate)

        return candidates

    def evaluate(self, thought, problem):
        """评估思考步骤的质量"""

        eval_prompt = f"""
问题: {problem}
思考步骤: {thought}

请评估这个思考步骤的合理性（0-10分）："""

        score_text = self.model.generate(eval_prompt)
        # 解析分数
        score = self.extract_score(score_text)

        return score

Process Reward Model

什么是 PRM

Process Reward Model (PRM) 用于评估推理过程中每一步的正确性，而不仅仅是最终结果。

传统 Reward Model: 评估最终答案的正确性
Process Reward Model: 评估每一步推理的正确性

优势:
1. 更细粒度的反馈
2. 可以定位错误步骤
3. 更好的信用分配

PRM 训练

class ProcessRewardModel(nn.Module):
    """过程奖励模型"""

    def __init__(self, base_model_name: str):
        super().__init__()
        self.base_model = AutoModel.from_pretrained(base_model_name)
        self.value_head = nn.Linear(self.base_model.config.hidden_size, 1)

    def forward(self, input_ids, attention_mask):
        """
        为每个 token 位置预测奖励值

        输入:
        - input_ids: [batch, seq_len] 包含问题和推理步骤

        输出:
        - rewards: [batch, seq_len] 每个位置的奖励值
        """
        # 获取隐藏状态
        outputs = self.base_model(
            input_ids=input_ids,
            attention_mask=attention_mask,
            output_hidden_states=True
        )

        hidden_states = outputs.last_hidden_state  # [batch, seq_len, hidden]

        # 预测每个位置的奖励
        rewards = self.value_head(hidden_states).squeeze(-1)  # [batch, seq_len]

        return rewards

    def compute_step_rewards(self, problem: str, reasoning_steps: List[str]):
        """计算每一步的奖励"""

        full_text = problem + "\n"
        step_rewards = []

        for i, step in enumerate(reasoning_steps):
            full_text += f"Step {i+1}: {step}\n"

            # 编码当前状态
            inputs = self.tokenizer(
                full_text,
                return_tensors="pt",
                truncation=True
            )

            # 获取最后一步的奖励
            with torch.no_grad():
                rewards = self.forward(**inputs)
                step_reward = rewards[0, -1].item()

            step_rewards.append(step_reward)

        return step_rewards

PRM 训练数据构建

def create_prm_training_data(problems, num_samples_per_problem=10):
    """创建 PRM 训练数据"""

    training_data = []

    for problem in problems:
        # 使用模型生成多个推理路径
        for _ in range(num_samples_per_problem):
            reasoning_steps, final_answer = generate_reasoning_path(problem)

            # 标注每一步的正确性
            step_labels = []
            for i, step in enumerate(reasoning_steps):
                # 人工或自动标注
                is_correct = verify_step(problem, reasoning_steps[:i+1])
                step_labels.append(1.0 if is_correct else 0.0)

            training_data.append({
                'problem': problem,
                'steps': reasoning_steps,
                'step_labels': step_labels,
                'final_answer': final_answer
            })

    return training_data


def train_prm(prm_model, training_data, epochs=3):
    """训练 PRM"""

    optimizer = AdamW(prm_model.parameters(), lr=1e-5)

    for epoch in range(epochs):
        total_loss = 0

        for batch in training_data:
            # 构建输入
            text = batch['problem'] + "\n"
            for i, step in enumerate(batch['steps']):
                text += f"Step {i+1}: {step}\n"

            inputs = prm_model.tokenizer(
                text,
                return_tensors="pt",
                truncation=True
            )

            # 前向传播
            rewards = prm_model(**inputs)

            # 计算损失 - 只在推理步骤位置计算
            step_positions = find_step_positions(inputs['input_ids'])
            step_rewards = rewards[0, step_positions]

            # MSE 损失
            labels = torch.tensor(batch['step_labels'])
            loss = F.mse_loss(step_rewards, labels)

            # 反向传播
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()

            total_loss += loss.item()

        print(f"Epoch {epoch+1}, Loss: {total_loss/len(training_data):.4f}")

Self-Consistency with CoT

class SelfConsistencyCoT:
    """自洽性 CoT"""

    def __init__(self, model, num_paths=10):
        self.model = model
        self.num_paths = num_paths

    def solve(self, problem):
        """使用多条推理路径并投票"""

        # 生成多条推理路径
        reasoning_paths = []
        for i in range(self.num_paths):
            prompt = f"""问题: {problem}

请详细分析这个问题，给出逐步推理过程，最后给出答案。

思考过程:"""

            response = self.model.generate(
                prompt,
                temperature=0.7 + i * 0.05  # 不同的温度
            )

            # 提取最终答案
            answer = self.extract_answer(response)
            reasoning_paths.append({
                'reasoning': response,
                'answer': answer
            })

        # 答案投票
        from collections import Counter
        answers = [p['answer'] for p in reasoning_paths]
        vote_counts = Counter(answers)

        # 选择最常见的答案
        best_answer = vote_counts.most_common(1)[0][0]

        # 选择对应的最佳推理过程
        for path in reasoning_paths:
            if path['answer'] == best_answer:
                return path['reasoning'], best_answer

        return reasoning_paths[0]['reasoning'], best_answer

DeepSeek-R1 开源方案

DeepSeek-R1 简介

DeepSeek 于 2025 年 1 月发布的 R1 模型，首次在开源社区实现了接近 o1 的推理能力。

模型系列:
- DeepSeek-R1-Zero: 纯强化学习训练
- DeepSeek-R1: 冷启动 + 强化学习 + SFT
- DeepSeek-R1-Distill: 蒸馏版本 (1.5B-70B)

训练流程

class DeepSeekR1Trainer:
    """DeepSeek-R1 训练流程"""

    def __init__(self, base_model):
        self.model = base_model

    def stage1_cold_start(self, reasoning_data):
        """
        阶段1: 冷启动
        - 使用少量高质量 CoT 数据微调
        - 让模型学会基本的推理格式
        """

        # 构建 SFT 数据
        sft_data = []
        for item in reasoning_data:
            prompt = item['problem']
            response = f"""<think>
{item['reasoning']}
</think>

{item['answer']}"""

            sft_data.append({
                'prompt': prompt,
                'response': response
            })

        # SFT 训练
        self.model = self.supervised_finetune(self.model, sft_data)

        return self.model

    def stage2_reasoning_rl(self, math_problems, code_problems):
        """
        阶段2: 推理导向的强化学习
        - 使用 GRPO (Group Relative Policy Optimization)
        - 奖励基于答案正确性和格式
        """

        for batch in self.get_batches(math_problems + code_problems):
            # 对每个问题采样多个回答
            responses = []
            for problem in batch:
                # 采样 4 个回答
                group_responses = []
                for _ in range(4):
                    response = self.model.generate(
                        problem['question'],
                        temperature=0.7
                    )
                    group_responses.append(response)

                responses.append(group_responses)

            # 计算奖励
            rewards = []
            for group in responses:
                group_rewards = []
                for response in group:
                    # 格式奖励: 是否包含 <think> 标签
                    format_reward = 1.0 if '<think>' in response else 0.0

                    # 答案正确性奖励
                    answer = self.extract_answer(response)
                    correctness_reward = self.verify_answer(
                        answer,
                        problem['answer']
                    )

                    total_reward = format_reward + correctness_reward
                    group_rewards.append(total_reward)

                rewards.append(group_rewards)

            # GRPO 更新
            self.grpo_update(responses, rewards)

        return self.model

    def stage3_sft_all_scenarios(self, general_data):
        """
        阶段3: 全场景 SFT
        - 使用通用数据提升写作能力
        - 保持推理能力
        """

        # 混合推理数据和通用数据
        mixed_data = self.mix_data(
            reasoning_data=self.get_reasoning_data(),
            general_data=general_data,
            ratio=0.7  # 70% 推理数据
        )

        self.model = self.supervised_finetune(self.model, mixed_data)

        return self.model

    def stage4_rl_human_preference(self, human_preferences):
        """
        阶段4: 基于人类偏好的 RL
        - 使用 DPO 或 PPO
        - 对齐人类偏好
        """

        # DPO 训练
        self.model = self.dpo_train(self.model, human_preferences)

        return self.model

    def grpo_update(self, responses, rewards):
        """
        Group Relative Policy Optimization

        核心思想: 使用组内相对奖励而非绝对奖励
        """

        for group_responses, group_rewards in zip(responses, rewards):
            # 计算组内平均奖励
            mean_reward = np.mean(group_rewards)
            std_reward = np.std(group_rewards) + 1e-8

            # 归一化奖励
            advantages = [(r - mean_reward) / std_reward for r in group_rewards]

            # 策略梯度更新
            for response, advantage in zip(group_responses, advantages):
                # 计算策略损失
                log_prob = self.model.compute_log_prob(response)

                # PPO 风格的目标函数
                loss = -log_prob * advantage

                # 反向传播
                loss.backward()

        # 更新模型参数
        self.optimizer.step()
        self.optimizer.zero_grad()

面试要点

高频问题

Q1: o1 模型相比 GPT-4 的核心创新是什么？

A:

  1. Test-Time Scaling: 在推理时增加思考时间
  2. Process Reward Model: 细粒度的步骤级反馈
  3. 长推理链: 生成详细的思考过程
  4. 自我修正: 可以回溯和修正思路

Q2: Test-Time Scaling 与 Pre-Train Scaling 的区别？

A:

维度Pre-Train ScalingTest-Time Scaling
时机训练阶段推理阶段
资源训练计算推理计算
灵活性固定动态可调
适用场景通用能力复杂推理

Q3: PRM 相比 ORM (Outcome Reward Model) 的优势？

A:

  1. 细粒度反馈: 可以定位错误步骤
  2. 信用分配: 明确哪一步导致错误
  3. 学习效率: 从部分正确的样本学习
  4. 可解释性: 知道模型在哪一步出错

Q4: DeepSeek-R1 的训练流程是怎样的？

A:

  1. 冷启动: 使用高质量 CoT 数据 SFT
  2. 推理 RL: 使用 GRPO 训练推理能力
  3. 全场景 SFT: 混合推理和通用数据
  4. 偏好对齐: 使用 DPO 对齐人类偏好

Q5: Self-Consistency 的原理是什么？

A:

  • 对同一问题采样多个推理路径
  • 通过投票选择最常见的答案
  • 基于假设: 正确的推理路径会收敛到相同答案
  • 类似于集成学习中的投票机制
