## 引言

BERT (Bidirectional Encoder Representations from Transformers) 由 Google 于 2018 年提出，通过双向编码器和创新的预训练任务，在 NLP 理解任务上取得了突破性进展。

### 核心创新

| 特性 | GPT | BERT |
|------|-----|------|
| 架构 | Decoder-only | Encoder-only |
| 注意力 | 因果 (单向) | 双向 |
| 预训练 | 自回归 | MLM + NSP |
| 优势 | 生成 | 理解 |

---

## BERT 架构

### 模型配置

| 参数 | BERT-Base | BERT-Large |
|------|-----------|------------|
| 层数 | 12 | 24 |
| 隐藏维度 | 768 | 1024 |
| 注意力头数 | 12 | 16 |
| FFN 维度 | 3072 | 4096 |
| 参数量 | 110M | 340M |

### 输入表示

```
输入: [CLS] 今天 天气 很好 [SEP] 适合 外出 [SEP]
      │   └────── 句子 A ──────┘ └── 句子 B ──┘
      └── 分类标记

Token Embeddings: 词嵌入
+ Position Embeddings: 位置嵌入 (可学习)
+ Segment Embeddings: 句子分段嵌入
= 最终输入表示
```

### 位置编码

BERT 使用**可学习的位置嵌入**而非正弦编码：

```python
self.position_embeddings = nn.Embedding(512, 768)  # max_pos=512, dim=768
```

## 预训练任务

### 1. Masked Language Model (MLM)

**核心思想**: 随机 mask 部分 token，让模型预测被 mask 的词。

```
原始: 今天 天气 很好 适合 外出
Mask: 今天 天气 [MASK] 适合 外出
目标: 很好
```

**Mask 策略**:
- 80% 概率替换为 [MASK]
- 10% 概率替换为随机词
- 10% 概率保持不变

**数学表达**:

$$
L_{MLM} = -\mathbb{E}_{x \in \mathcal{M}} \log P(x | \hat{x})
$$

其中 $\mathcal{M}$ 是被 mask 的位置集合。

**为什么不是 100% mask？**

- 预训练和微调的不一致：微调时没有 [MASK]
- 混合策略缓解不一致问题

### 2. Next Sentence Prediction (NSP)

**核心思想**: 预测句子 B 是否是句子 A 的下一句。

```
正例:
[CLS] 今天天气很好 [SEP] 适合外出游玩 [SEP] → IsNext

负例:
[CLS] 今天天气很好 [SEP] 股票今天大涨 [SEP] → NotNext
```

**作用**: 学习句子级关系，对下游任务如 QA、NLI 有帮助。

**后续研究**: RoBERTa 等发现 NSP 作用有限，予以移除。

## 微调策略

### 句子分类

```
输入: [CLS] 这部电影太精彩了 [SEP]
输出: [CLS] 的隐藏状态 → Linear → softmax → 正面/负面
```

### 句子对分类

```
输入: [CLS] 前提 [SEP] 假设 [SEP]
输出: [CLS] → Linear → 蕴含/矛盾/中立
```

### Token 级别任务

**序列标注** (NER):
```
输入: [CLS] 北京 是 中国 首都 [SEP]
输出: 每个 token 的隐藏状态 → Linear → B-LOC O B-LOC O
```

**问答** (SQuAD):
```
输入: [CLS] 问题 [SEP] 段落 [SEP]
输出: Start 概率分布 + End 概率分布
```

## BERT 变体

### RoBERTa

**Robustly Optimized BERT Approach**

| 改进 | 说明 |
|------|------|
| 移除 NSP | 发现 NSP 对性能无帮助 |
| 更大 batch | 8K → 32K |
| 更多数据 | 160GB → 10 倍数据 |
| 动态 masking | 每个 epoch 重新随机 mask |
| 更长训练 | 100K → 500K steps |

**结果**: 在 GLUE 上全面超越 BERT-Large

### ALBERT

**A Lite BERT**

**核心改进**:

1. **因子化嵌入参数化**
   ```
   V × H → (V × E) + (E × H)
   30K × 4096 → (30K × 128) + (128 × 4096)
   大幅减小参数量
   ```

2. **跨层参数共享**
   - 所有层共享同一套参数
   - 参数量减少，训练更稳定

3. **句子顺序预测 (SOP)**
   - 替代 NSP
   - 预测句子顺序是否被交换
   - 比 NSP 更难，更有学习价值

| 模型 | 参数量 | 层数 | 隐藏维度 |
|------|--------|------|----------|
| BERT-Base | 110M | 12 | 768 |
| ALBERT-Base | 12M | 12 | 768 |
| ALBERT-xxlarge | 235M | 12 | 4096 |

### ELECTRA

**更高效预训练**

**核心思想**: 替换 token 检测 (Replaced Token Detection)

```
流程:
1. 小型生成器 (Generator) 预测被 mask 的词
2. 判别器 (Discriminator) 判断每个词是否被替换
3. 所有位置都参与训练，效率更高

对比:
MLM: 只训练 15% 的 mask 位置
ELECTRA: 训练 100% 的位置
```

**优势**:
- 训练效率提升 4 倍
- 小模型效果更好

### 其他变体

| 模型 | 核心改进 |
|------|----------|
| SpanBERT | mask 连续片段而非单个词 |
| DistilBERT | 知识蒸馏，轻量级 |
| TinyBERT | 更深蒸馏，6 层 |
| MobileBERT | 移动端优化 |
| DeBERTa | 解耦注意力 + 增强 mask |

## 其他编码器模型

### 1. RoBERTa (已介绍)

### 2. DeBERTa

**Decoding-enhanced BERT with Disentangled Attention**

**核心创新**:

1. **解耦注意力**
   ```
   传统: 内容和位置耦合在一起
   DeBERTa: 分别计算内容-内容、内容-位置、位置-内容的注意力
   ```

2. **增强 mask 解码器**
   - 在微调时引入绝对位置
   - 弥补预训练时的信息缺失

3. **规模扩展**
   - DeBERTa 1.5B 首次在 SuperGLUE 超越人类

## Encoder vs Decoder

### 架构对比

| 特性 | Encoder (BERT) | Decoder (GPT) |
|------|----------------|---------------|
| 注意力 | 双向 | 因果 (单向) |
| 预训练 | MLM | 自回归 |
| 优势 | 理解、表示 | 生成 |
| 典型任务 | 分类、NER、相似度 | 文本生成、对话 |
| 并行性 | 完全并行 | 训练并行，推理串行 |

### 选择指南

| 任务 | 推荐架构 |
|------|----------|
| 文本分类 | Encoder |
| 命名实体识别 | Encoder |
| 语义相似度 | Encoder |
| 抽取式问答 | Encoder |
| 文本生成 | Decoder |
| 机器翻译 | Encoder-Decoder |
| 对话系统 | Decoder |

### 为什么现代 LLM 多为 Decoder-only?

1. **统一架构**: 一个模型处理所有任务
2. **生成能力**: Decoder 天然适合生成
3. **规模效应**: 大 Decoder 在理解任务上也不差
4. **训练效率**: 自回归训练简单高效

## 代码实现

### MLM 预训练实现

```python
import torch
import torch.nn as nn
from transformers import BertTokenizer, BertConfig

class MLMPretraining(nn.Module):
    def __init__(self, vocab_size, hidden_size=768):
        super().__init__()
        self.bert = BertModel(...)  # BERT 编码器
        self.cls = nn.Linear(hidden_size, vocab_size)

    def forward(self, input_ids, attention_mask, labels=None):
        """
        input_ids: [batch_size, seq_len]
        labels: [batch_size, seq_len], -100 表示不计算 loss
        """
        outputs = self.bert(input_ids, attention_mask)
        sequence_output = outputs.last_hidden_state  # [B, L, H]

        prediction_scores = self.cls(sequence_output)  # [B, L, V]

        if labels is not None:
            loss_fct = nn.CrossEntropyLoss(ignore_index=-100)
            loss = loss_fct(
                prediction_scores.view(-1, self.vocab_size),
                labels.view(-1)
            )
            return loss

        return prediction_scores


def create_mlm_inputs(tokenizer, text, mlm_probability=0.15):
    """创建 MLM 训练数据"""
    tokens = tokenizer.tokenize(text)
    labels = [-100] * len(tokens)  # -100 表示不计算 loss

    # 随机选择 mask 位置
    n_mask = int(len(tokens) * mlm_probability)
    mask_indices = torch.randperm(len(tokens))[:n_mask]

    for idx in mask_indices:
        labels[idx] = tokenizer.convert_tokens_to_ids(tokens[idx])

        prob = torch.rand(1).item()
        if prob < 0.8:
            tokens[idx] = '[MASK]'
        elif prob < 0.9:
            tokens[idx] = tokenizer.convert_ids_to_tokens(
                torch.randint(0, tokenizer.vocab_size, (1,)).item()
            )
        # else: 10% 保持不变

    input_ids = tokenizer.convert_tokens_to_ids(tokens)
    return input_ids, labels
```

### 分类任务微调

```python
class BertForSequenceClassification(nn.Module):
    def __init__(self, bert_model, num_labels):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)
        self.classifier = nn.Linear(768, num_labels)

    def forward(self, input_ids, attention_mask, labels=None):
        outputs = self.bert(input_ids, attention_mask)
        pooled_output = outputs.pooler_output  # [CLS] 表示

        pooled_output = self.dropout(pooled_output)
        logits = self.classifier(pooled_output)

        if labels is not None:
            loss_fct = nn.CrossEntropyLoss()
            loss = loss_fct(logits, labels)
            return loss

        return logits
```

### Token 分类 (NER)

```python
class BertForTokenClassification(nn.Module):
    def __init__(self, bert_model, num_labels):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)

            return loss

## 面试要点

### 高频问题

**Q1: BERT 和 GPT 的核心区别？**

A:
1. 架构: BERT 是 Encoder (双向)，GPT 是 Decoder (单向)
2. 预训练: BERT 用 MLM，GPT 用自回归
3. 应用: BERT 适合理解任务，GPT 适合生成任务

**Q2: 为什么 BERT 使用 MLM 而不是自回归？**

A: MLM 允许模型利用双向上下文，获得更丰富的语义表示。自回归只能利用单向上下文。

**Q3: MLM 的 mask 策略是什么？为什么要这样设计？**

A:
- 80% 替换为 [MASK]
- 10% 替换为随机词
- 10% 保持不变

原因: 缓解预训练和微调的不一致。微调时没有 [MASK]，混合策略让模型不完全依赖 [MASK] 标记。

**Q4: RoBERTa 相比 BERT 的主要改进？**

A:
1. 移除 NSP 任务
2. 更大 batch size
3. 更多训练数据
4. 动态 masking

3. 自回归训练更简单高效
4. 工程实现更统一

## 引言

BERT (Bidirectional Encoder Representations from Transformers) 由 Google 于 2018 年提出，通过双向编码器和创新的预训练任务，在 NLP 理解任务上取得了突破性进展。

### 核心创新

| 特性 | GPT | BERT |
|------|-----|------|
| 架构 | Decoder-only | Encoder-only |
| 注意力 | 因果 (单向) | 双向 |
| 预训练 | 自回归 | MLM + NSP |
| 优势 | 生成 | 理解 |

---

## BERT 架构

### 模型配置

| 参数 | BERT-Base | BERT-Large |
|------|-----------|------------|
| 层数 | 12 | 24 |
| 隐藏维度 | 768 | 1024 |
| 注意力头数 | 12 | 16 |
| FFN 维度 | 3072 | 4096 |
| 参数量 | 110M | 340M |

### 输入表示

```
输入: [CLS] 今天 天气 很好 [SEP] 适合 外出 [SEP]
      │   └────── 句子 A ──────┘ └── 句子 B ──┘
      └── 分类标记

Token Embeddings: 词嵌入
+ Position Embeddings: 位置嵌入 (可学习)
+ Segment Embeddings: 句子分段嵌入
= 最终输入表示
```

### 位置编码

BERT 使用**可学习的位置嵌入**而非正弦编码：

```python
self.position_embeddings = nn.Embedding(512, 768)  # max_pos=512, dim=768
```

## 预训练任务

### 1. Masked Language Model (MLM)

**核心思想**: 随机 mask 部分 token，让模型预测被 mask 的词。

```
原始: 今天 天气 很好 适合 外出
Mask: 今天 天气 [MASK] 适合 外出
目标: 很好
```

**Mask 策略**:
- 80% 概率替换为 [MASK]
- 10% 概率替换为随机词
- 10% 概率保持不变

**数学表达**:

$$
L_{MLM} = -\mathbb{E}_{x \in \mathcal{M}} \log P(x | \hat{x})
$$

其中 $\mathcal{M}$ 是被 mask 的位置集合。

**为什么不是 100% mask？**

- 预训练和微调的不一致：微调时没有 [MASK]
- 混合策略缓解不一致问题

### 2. Next Sentence Prediction (NSP)

**核心思想**: 预测句子 B 是否是句子 A 的下一句。

```
正例:
[CLS] 今天天气很好 [SEP] 适合外出游玩 [SEP] → IsNext

负例:
[CLS] 今天天气很好 [SEP] 股票今天大涨 [SEP] → NotNext
```

**作用**: 学习句子级关系，对下游任务如 QA、NLI 有帮助。

**后续研究**: RoBERTa 等发现 NSP 作用有限，予以移除。

## 微调策略

### 句子分类

```
输入: [CLS] 这部电影太精彩了 [SEP]
输出: [CLS] 的隐藏状态 → Linear → softmax → 正面/负面
```

### 句子对分类

```
输入: [CLS] 前提 [SEP] 假设 [SEP]
输出: [CLS] → Linear → 蕴含/矛盾/中立
```

### Token 级别任务

**序列标注** (NER):
```
输入: [CLS] 北京 是 中国 首都 [SEP]
输出: 每个 token 的隐藏状态 → Linear → B-LOC O B-LOC O
```

**问答** (SQuAD):
```
输入: [CLS] 问题 [SEP] 段落 [SEP]
输出: Start 概率分布 + End 概率分布
```

## BERT 变体

### RoBERTa

**Robustly Optimized BERT Approach**

| 改进 | 说明 |
|------|------|
| 移除 NSP | 发现 NSP 对性能无帮助 |
| 更大 batch | 8K → 32K |
| 更多数据 | 160GB → 10 倍数据 |
| 动态 masking | 每个 epoch 重新随机 mask |
| 更长训练 | 100K → 500K steps |

**结果**: 在 GLUE 上全面超越 BERT-Large

### ALBERT

**A Lite BERT**

**核心改进**:

1. **因子化嵌入参数化**
   ```
   V × H → (V × E) + (E × H)
   30K × 4096 → (30K × 128) + (128 × 4096)
   大幅减小参数量
   ```

2. **跨层参数共享**
   - 所有层共享同一套参数
   - 参数量减少，训练更稳定

3. **句子顺序预测 (SOP)**
   - 替代 NSP
   - 预测句子顺序是否被交换
   - 比 NSP 更难，更有学习价值

| 模型 | 参数量 | 层数 | 隐藏维度 |
|------|--------|------|----------|
| BERT-Base | 110M | 12 | 768 |
| ALBERT-Base | 12M | 12 | 768 |
| ALBERT-xxlarge | 235M | 12 | 4096 |

### ELECTRA

**更高效预训练**

**核心思想**: 替换 token 检测 (Replaced Token Detection)

```
流程:
1. 小型生成器 (Generator) 预测被 mask 的词
2. 判别器 (Discriminator) 判断每个词是否被替换
3. 所有位置都参与训练，效率更高

对比:
MLM: 只训练 15% 的 mask 位置
ELECTRA: 训练 100% 的位置
```

**优势**:
- 训练效率提升 4 倍
- 小模型效果更好

### 其他变体

| 模型 | 核心改进 |
|------|----------|
| SpanBERT | mask 连续片段而非单个词 |
| DistilBERT | 知识蒸馏，轻量级 |
| TinyBERT | 更深蒸馏，6 层 |
| MobileBERT | 移动端优化 |
| DeBERTa | 解耦注意力 + 增强 mask |

### 2. DeBERTa

**Decoding-enhanced BERT with Disentangled Attention**

**核心创新**:

1. **解耦注意力**
   ```
   传统: 内容和位置耦合在一起

   - 在微调时引入绝对位置
   - 弥补预训练时的信息缺失

## Encoder vs Decoder

### 架构对比

| 特性 | Encoder (BERT) | Decoder (GPT) |
|------|----------------|---------------|
| 注意力 | 双向 | 因果 (单向) |
| 预训练 | MLM | 自回归 |
| 优势 | 理解、表示 | 生成 |
| 典型任务 | 分类、NER、相似度 | 文本生成、对话 |
| 并行性 | 完全并行 | 训练并行，推理串行 |

### 选择指南

| 任务 | 推荐架构 |
|------|----------|
| 文本分类 | Encoder |
| 命名实体识别 | Encoder |
| 语义相似度 | Encoder |
| 抽取式问答 | Encoder |
| 文本生成 | Decoder |
| 机器翻译 | Encoder-Decoder |
| 对话系统 | Decoder |

### 为什么现代 LLM 多为 Decoder-only?

1. **统一架构**: 一个模型处理所有任务
2. **生成能力**: Decoder 天然适合生成
3. **规模效应**: 大 Decoder 在理解任务上也不差
4. **训练效率**: 自回归训练简单高效

## 代码实现

### MLM 预训练实现

```python
import torch
import torch.nn as nn
from transformers import BertTokenizer, BertConfig

class MLMPretraining(nn.Module):
    def __init__(self, vocab_size, hidden_size=768):
        super().__init__()
        self.bert = BertModel(...)  # BERT 编码器
        self.cls = nn.Linear(hidden_size, vocab_size)

    def forward(self, input_ids, attention_mask, labels=None):
        """
        input_ids: [batch_size, seq_len]
        labels: [batch_size, seq_len], -100 表示不计算 loss
        """
        outputs = self.bert(input_ids, attention_mask)
        sequence_output = outputs.last_hidden_state  # [B, L, H]

        prediction_scores = self.cls(sequence_output)  # [B, L, V]

        if labels is not None:
            loss_fct = nn.CrossEntropyLoss(ignore_index=-100)
            loss = loss_fct(
                prediction_scores.view(-1, self.vocab_size),
                labels.view(-1)
            )
            return loss

        return prediction_scores


def create_mlm_inputs(tokenizer, text, mlm_probability=0.15):
    """创建 MLM 训练数据"""
    tokens = tokenizer.tokenize(text)
    labels = [-100] * len(tokens)  # -100 表示不计算 loss

    # 随机选择 mask 位置
    n_mask = int(len(tokens) * mlm_probability)
    mask_indices = torch.randperm(len(tokens))[:n_mask]

    for idx in mask_indices:
        labels[idx] = tokenizer.convert_tokens_to_ids(tokens[idx])

        prob = torch.rand(1).item()
        if prob < 0.8:
            tokens[idx] = '[MASK]'
        elif prob < 0.9:
            tokens[idx] = tokenizer.convert_ids_to_tokens(
                torch.randint(0, tokenizer.vocab_size, (1,)).item()
            )
        # else: 10% 保持不变

    input_ids = tokenizer.convert_tokens_to_ids(tokens)
    return input_ids, labels
```

### 分类任务微调

```python
class BertForSequenceClassification(nn.Module):
    def __init__(self, bert_model, num_labels):
        super().__init__()
        self.bert = bert_model
        self.dropout = nn.Dropout(0.1)
        self.classifier = nn.Linear(768, num_labels)

    def forward(self, input_ids, attention_mask, labels=None):
        outputs = self.bert(input_ids, attention_mask)
        pooled_output = outputs.pooler_output  # [CLS] 表示

        pooled_output = self.dropout(pooled_output)
        logits = self.classifier(pooled_output)

        if labels is not None:
            loss_fct = nn.CrossEntropyLoss()
            loss = loss_fct(logits, labels)
            return loss

        return logits
```

## 面试要点

### 高频问题

**Q1: BERT 和 GPT 的核心区别？**

A:
1. 架构: BERT 是 Encoder (双向)，GPT 是 Decoder (单向)
2. 预训练: BERT 用 MLM，GPT 用自回归
3. 应用: BERT 适合理解任务，GPT 适合生成任务

**Q2: 为什么 BERT 使用 MLM 而不是自回归？**

A: MLM 允许模型利用双向上下文，获得更丰富的语义表示。自回归只能利用单向上下文。

**Q3: MLM 的 mask 策略是什么？为什么要这样设计？**

A:
- 80% 替换为 [MASK]
- 10% 替换为随机词
- 10% 保持不变

原因: 缓解预训练和微调的不一致。微调时没有 [MASK]，混合策略让模型不完全依赖 [MASK] 标记。

**Q4: RoBERTa 相比 BERT 的主要改进？**

A:
1. 移除 NSP 任务
2. 更大 batch size
3. 更多训练数据
4. 动态 masking
5. 更长训练时间

**Q5: Encoder-only 模型为什么在现代 LLM 中较少使用？**

A:
1. Decoder-only 可以统一处理理解和生成
2. 大 Decoder 在理解任务上表现也很好
3. 自回归训练更简单高效
4. 工程实现更统一
