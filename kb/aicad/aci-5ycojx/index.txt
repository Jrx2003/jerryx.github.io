c:T1aea,# ACI 测试指南

> 更新日期：2026-03-12
> 来源：`docs/work_logs/ACI_终端运行与产物查看指南_20260309.md`

## 目标

本指南用于在本地终端运行 ACI 对比测试（迭代式 vs 一次性），并查看每一轮的中间产物、拼装提示词、动作决策与最终模型。

---

## 前置条件

1. **已安装依赖**
   ```bash
   uv sync
   ```

2. **Docker 可用**
   - 有 `cadquery-runtime:latest` 镜像

3. **.env 配置**
   - 已配置可用的 LLM Key（当前默认建议 GLM）

---

## 开关配置（.env）

已新增以下开关，脚本会自动读取：

```env
AICAD_TEST_RUNS_ROOT=.tmp/test_runs
AICAD_TEST_REASONING_PROVIDER=glm
AICAD_TEST_REASONING_MODEL=glm-4.7
AICAD_TEST_LLM_TIMEOUT_SECONDS=40
AICAD_TEST_LLM_CALL_TIMEOUT_SECONDS=40
AICAD_TEST_CASE_LIMIT=3
AICAD_TEST_MAX_ITERATIVE_ROUNDS=6
AICAD_TEST_ONE_SHOT_MAX_ATTEMPTS=2
AICAD_TEST_LLM_RETRY_ATTEMPTS=2
```

**说明：**
- 不传命令行变量时，脚本用以上默认值
- 命令行环境变量优先级高于 `.env`

---

## 运行命令

### 一键跑单元测试 + 真实 LLM 对比测试

```bash
./scripts/run_aci_tests_with_artifacts.sh
```

脚本会自动：
1. 先跑 unit tests
2. 再跑真实 LLM e2e 对比测试
3. 将产物写入分钟级目录（如 `20260309_2340`）

---

### 指定 run 目录名（可选）

```bash
./scripts/run_aci_tests_with_artifacts.sh 20260309_2355
```

建议格式保持 `YYYYMMDD_HHMM`

---

## 目录结构

### 运行根目录

```
.tmp/test_runs/<YYYYMMDD_HHMM>/
```

例如：`.tmp/test_runs/20260309_2340/`

---

### 完整目录结构

```
<run>/
  run_manifest.json          # 本次运行参数、命令、provider/model、日志路径总览
  logs/
    unit_aci.log            # 单元测试原始输出
    e2e_real_aci.log        # 真实 LLM + sandbox 执行日志，包含每轮动作执行痕迹
  cases/
    comparison_summary.json   # 每个 case 的迭代式与一次性总结果
    case_01_plate_fillet/
      input.json            # 测试用例输入
      iterative/
        prompts/
          round_XX_request.json       # 每轮拼装后的提示词（重点看 user_prompt、input_state）
        plans/
          round_XX_response.json      # 每轮 LLM 返回动作计划与 token usage
          round_XX_llm_retry_XX.json # LLM 超时/重试错误记录
        actions/
          round_XX_action_YY_request.json # 实际送入 apply_action_sequence 的动作序列
          round_XX_action_YY_result.json # 每一步 sandbox 执行结果、snapshot、suggestions、completeness
        outputs/
          step_after_action_XX.step # 每一步的 STEP 快照，可验证迭代建模过程
          final_model.step            # 迭代式最终模型
        summary.json                  # 迭代式模式总结
      oneshot/
        prompts/
          attempt_XX_request.json   # 一次性模式提示词
        attempts/
          attempt_XX_generated_code.py # 一次性模式生成的 CadQuery 代码
        outputs/
          final_model.step            # 一次性最终模型
        summary.json                  # 一次性模式总结
    case_02_...
    case_03_...
```

**另外：** `.tmp/test_runs/latest` 会始终指向最近一次 run。

---

## 每类产物含义

### 1. run_manifest.json

**内容：** 本次运行参数、命令、provider/model、日志路径总览

---

### 2. logs/unit_aci.log

**内容：** 单元测试原始输出

---

### 3. logs/e2e_real_aci.log

**内容：** 真实 LLM + sandbox 执行日志，包含每轮动作执行痕迹

---

### 4. cases/comparison_summary.json

**内容：** 每个 case 的迭代式与一次性总结果（是否有最终 step、动作数、轮数、错误）

---

### 5. iterative/prompts/round_XX_request.json

**内容：** 每轮拼装后的提示词

**重点字段：**
- `user_prompt`：发送给 LLM 的完整提示
- `input_state`：当前模型状态快照

---

### 6. iterative/plans/round_XX_response.json

**内容：** 每轮 LLM 返回动作计划与 token usage

---

### 7. iterative/plans/*_llm_retry_XX.json

**内容：** LLM 超时/重试错误记录

**用途：** 调试网络和模型稳定性

---

### 8. iterative/actions/*_request.json

**内容：** 实际送入 `apply_action_sequence` 的动作序列

---

### 9. iterative/actions/*_result.json

**内容：** 每一步 sandbox 执行结果、snapshot、suggestions、completeness

---

### 10. iterative/outputs/step_after_action_XX.step

**内容：** 每一步的 STEP 快照，可验证迭代建模过程

---

### 11. iterative/outputs/final_model.step

**内容：** 迭代式最终模型

---

### 12. oneshot/prompts/attempt_XX_request.json

**内容：** 一次性模式提示词

---

### 13. oneshot/attempts/attempt_XX_generated_code.py

**内容：** 一次性模式生成的 CadQuery 代码

---

### 14. oneshot/outputs/final_model.step

**内容：** 一次性最终模型

---

## 如何做"迭代式 vs 一次性"对比

以 `plate_fillet` 为例：

### 1. 看同一 case 的两份最终模型

```bash
# 迭代式
cases/case_01_plate_fillet/iterative/outputs/final_model.step

# 一次性
cases/case_01_plate_fillet/oneshot/outputs/final_model.step
```

---

### 2. 看迭代动作链路

```bash
# 查看所有动作
cases/case_01_plate_fillet/iterative/actions/
```

按 `round_XX_action_YY` 顺序查看。

---

### 3. 看关键动作参数是否符合需求

例如 fillet 是否用了 `edge_scope=all_outer`：

```bash
cases/case_01_plate_fillet/iterative/actions/round_02_action_04_request.json
```

---

### 4. 看提示词与几何反馈是否驱动了下一轮决策

```bash
# 查看第2轮的提示词
iterative/prompts/round_02_request.json

# 查看第2轮的响应
iterative/plans/round_02_response.json

# 对照前一轮的反馈
actions/round_XX_action_YY_result.json  # 查看 suggestions 和 completeness
```

---

## 常见问题与处理

### 1. LLM 超时

**解决：** 查看 `*_llm_retry_XX.json`；增大以下参数：
- `AICAD_TEST_LLM_CALL_TIMEOUT_SECONDS`
- `AICAD_TEST_LLM_RETRY_ATTEMPTS`

---

### 2. 想临时切换 provider/model

**解决：** 直接命令行覆盖
```bash
LLM_REASONING_PROVIDER=glm LLM_REASONING_MODEL=glm-4.7 \
  ./scripts/run_aci_tests_with_artifacts.sh
```

---

### 3. 只想少跑几个 case

**解决：** 设置
```bash
AICAD_CASE_LIMIT=1 ./scripts/run_aci_tests_with_artifacts.sh
```

---

```bash
uv run pytest tests/unit/sub_agent/test_codegen_aci.py \

  tests/unit/sandbox/test_mcp_runner.py -v
```

- `input_state`：当前模型状态快照

---

### 6. iterative/plans/round_XX_response.json

**内容：** 每轮 LLM 返回动作计划与 token usage

---

### 7. iterative/plans/*_llm_retry_XX.json

**内容：** LLM 超时/重试错误记录

**用途：** 调试网络和模型稳定性

---

### 8. iterative/actions/*_request.json

**内容：** 实际送入 `apply_action_sequence` 的动作序列

---

### 9. iterative/actions/*_result.json

**内容：** 每一步 sandbox 执行结果、snapshot、suggestions、completeness

---

### 10. iterative/outputs/step_after_action_XX.step

**内容：** 每一步的 STEP 快照，可验证迭代建模过程

---

### 11. iterative/outputs/final_model.step

**内容：** 迭代式最终模型

---

### 12. oneshot/prompts/attempt_XX_request.json

**内容：** 一次性模式提示词

---

### 13. oneshot/attempts/attempt_XX_generated_code.py

**内容：** 一次性模式生成的 CadQuery 代码

---

### 14. oneshot/outputs/final_model.step

**内容：** 一次性最终模型

## 如何做"迭代式 vs 一次性"对比

以 `plate_fillet` 为例：

### 1. 看同一 case 的两份最终模型

```bash
# 迭代式
cases/case_01_plate_fillet/iterative/outputs/final_model.step

# 一次性
cases/case_01_plate_fillet/oneshot/outputs/final_model.step
```

```bash
# 查看所有动作
cases/case_01_plate_fillet/iterative/actions/
```

按 `round_XX_action_YY` 顺序查看。

---

### 3. 看关键动作参数是否符合需求

例如 fillet 是否用了 `edge_scope=all_outer`：

```bash
cases/case_01_plate_fillet/iterative/actions/round_02_action_04_request.json
```

```bash
# 查看第2轮的提示词
iterative/prompts/round_02_request.json

# 查看第2轮的响应
iterative/plans/round_02_response.json

# 对照前一轮的反馈
actions/round_XX_action_YY_result.json  # 查看 suggestions 和 completeness
```

## 常见问题与处理

### 1. LLM 超时

**解决：** 查看 `*_llm_retry_XX.json`；增大以下参数：
- `AICAD_TEST_LLM_CALL_TIMEOUT_SECONDS`
- `AICAD_TEST_LLM_RETRY_ATTEMPTS`

---

### 2. 想临时切换 provider/model

**解决：** 直接命令行覆盖
```bash
LLM_REASONING_PROVIDER=glm LLM_REASONING_MODEL=glm-4.7 \
  ./scripts/run_aci_tests_with_artifacts.sh
```

---

### 3. 只想少跑几个 case

**解决：** 设置
```bash
AICAD_CASE_LIMIT=1 ./scripts/run_aci_tests_with_artifacts.sh
```
