c:T17f8,# Sandbox MCP 改造（Phase 1）

> 更新日期：2026-03-12
> 来源：`docs/work_logs/SANDBOX.md`

## 目标与边界

**本阶段只做一件事：** 把现有 sandbox 执行能力标准化为 MCP 工具，保证可被 Claude Code 等 MCP client 直接调用。

**本阶段不做：** 队列负载与多实例调度。

---

## MCP 工具契约

### Tool 名称

`execute_cadquery`

---

### Input Schema

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|--------|--------|------|
| `code` | string | 是 | - | CadQuery Python 代码，必须最终赋值 `result` |
| `timeout_seconds` | integer | 否 | 120 | 执行超时（秒），范围 `[1, 600]` |
| `include_artifact_content` | boolean | 否 | true | 是否在响应中附带 artifact 的 base64 内容 |

---

### Output Schema

| 字段 | 类型 | 说明 |
|------|------|------|
| `success` | boolean | 执行是否成功 |
| `stdout` | string | 标准输出 |
| `stderr` | string | 标准错误 |
| `error_code` | enum | 错误码：none, invalid_request, timeout, image_not_found, docker_api_error, execution_error |
| `error_message` | string \| null | 错误信息 |
| `output_files` | string[] | 输出文件列表 |
| `artifacts` | object[] | 产物列表（filename, uri, mime_type, size_bytes, content_base64） |

---

### 返回形态

`call_tool` 同时返回：

1. **structuredContent**：严格遵循 output schema（便于 client 程序化消费）
2. **content**：
   - 一条 `TextContent` 执行摘要
   - 若包含 artifact 内容，则附加 `EmbeddedResource`

---

## 隔离方案

新增独立模块：`services/sandbox/mcp_server`

### 架构设计

```
sandbox_mcp_server.service.SandboxMCPService
    └─ 只依赖 SandboxRunner 协议，不依赖 FastAPI/HTTP
    └─ 支持直接单元测试

sandbox_mcp_server.server
    └─ 只负责 MCP transport（list_tools/call_tool）
    └─ 通过依赖注入接入 SandboxMCPService
```

这使得后续可在不启动 API Gateway/MainAgent/SubAgent 的情况下，单独验证 sandbox MCP 行为。

---

## 本地测试方式

### 单元测试

```bash
cd services/sandbox/mcp_server
uv run pytest tests
```

---

### 本地连通性演示

```bash
cd services/sandbox/mcp_server
uv run python scripts/demo_connectivity.py
```

该脚本会做三步：
1. 通过 stdio 启动 `sandbox_mcp_server`
2. 调用 `tools/list` 确认存在 `execute_cadquery`
3. 调用 `execute_cadquery` 并验证返回 `model.step` 的 STEP 头（`ISO-10303-21`）

---

## MCP Client 配置

### Claude Code（project scope）

```bash
claude mcp add aicad-sandbox --scope project -- uv run python -m sandbox_mcp_server
```

---

### Codex

```bash
codex mcp add aicad-sandbox -- uv run python -m sandbox_mcp_server
```

对应 `~/.codex/config.toml`：

```toml
[mcp_servers.aicad-sandbox]
command = "uv"
args = ["run", "python", "-m", "sandbox_mcp_server"]
cwd = "/Users/jerryx/code/aicad.app.agent"
```

---

## 完整运行流程

### 1. 准备环境

```bash
cd /Users/jerryx/code/aicad.app.agent
uv sync
cd infra && docker compose up -d && cd ..
./scripts/run_migrations.sh
docker image inspect cadquery-runtime:latest >/dev/null
mkdir -p storage/cad_files
```

---

### 2. 启动服务（两个终端）

#### 终端 A（API Gateway）

```bash
SANDBOX_TYPE=mcp \
SANDBOX_MCP_SERVER_CWD=/Users/jerryx/code/aicad.app.agent \
uv run uvicorn api_gateway.main:app --host 0.0.0.0 --port 8100
```

---

#### 终端 B（SubAgent）

```bash
SANDBOX_TYPE=mcp \
SANDBOX_MCP_SERVER_CWD=/Users/jerryx/code/aicad.app.agent \
uv run python -m sub_agent.main
```

---

### 3. 健康检查

```bash
curl -sS http://localhost:8100/health
curl -sS http://localhost:8100/health/ready
```

---

### 4. 发起建模请求（两段式确认）

#### 第一段：请求建模（应返回 `confirmation_required=true`）

```bash
curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[{"role":"user","content":"Create a simple 3D cylinder with radius 5 mm and height 20 mm"}],
    "stream":false
  }'
```

---

#### 第二段：携带 design-summary 确认创建任务（应返回 `job_id`）

```bash
curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[
      {"role":"user","content":"Create a simple 3D cylinder with radius 5 mm and height 20 mm"},
      {"role":"assistant","content":"I'\''ll create a cylinder with radius 5mm and height 20mm. Shall I proceed?\n\n```design-summary\nCylinder, radius 5mm, height 20mm\n```"}
    ],
    "stream":false,
    "confirm_job_creation":true
  }'
```

---

### 5. 查询任务状态直到成功

```bash
curl -sS http://localhost:8100/v1/jobs/<job_id> \
  -H "Authorization: Bearer dev_token_1"
```

**期望最终状态：** `status = succeeded`，并返回 `result.cad_file.filename = model.step`

---

## 关键日志验真点（SubAgent 终端）

- 出现 `using_mcp_sandbox_runner` → 说明已切到 mcp 路径
- 出现 `sandbox_mcp_server_starting` → 说明确实拉起了 MCP server
- 出现 `sandbox_execution_starting / sandbox_execution_complete` → 说明最终在 Docker sandbox 执行
- 出现 `job_succeeded` → 说明完整链路成功

---

### 本次实测样例

- job_id: `34ea913d-2e61-47ea-8c50-df2b7ea09458`
- 最终文件：`model.step`
- 文件大小：`10627` bytes

---

## 结束清理

```bash
# 两个服务终端 Ctrl+C
cd infra && docker compose down
```

---

## 后续迭代建议（Phase 2+）

1. 增加 artifact resource 模式（内容按需拉取，避免大文件 base64 放入工具结果）
2. 引入队列负载（worker 池、并发控制、背压与重试策略）
3. 增加生产观测：每次 tool call 的耗时、错误码分布、artifact 大小统计
