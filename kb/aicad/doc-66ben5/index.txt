## 架构与系统

| 术语 | 英文 | 定义 |
|------|------|------|
| Agent | - | LLM 驱动的智能体，负责特定任务的推理和决策 |
| Sandbox | - | 隔离执行环境，运行 CadQuery 代码并回收产物 |
| MCP | Model Context Protocol | LLM 与工具间的标准化通信协议 |
| Session | - | ACI 中的会话状态，跨多个 MCP 调用保持建模状态 |
| Job | - | 建模任务的顶层概念，包含一个或多个 Step |
| Step | - | 任务的基本执行单元，对应一次 Sandbox 执行 |

---

## CAD 相关

| 术语 | 英文 | 定义 |
|------|------|------|
| CadQuery | - | Python 的 CAD 建模库，用于程序化创建 3D 模型 |
| STEP | ISO-10303-21 | CAD 文件的标准交换格式，包含完整的几何数据 |
| Sketch | - | 二维草图，用于定义三维模型的截面 |
| Extrude | - | 将二维草图拉伸成三维实体 |
| Fillet | - | 圆角，将尖锐边缘平滑过渡 |
| Chamfer | - | 倒角，将尖锐边缘切削成斜面 |
| Solid | - | 三维实体对象 |
| Face | - | 三维模型的面 |
| Edge | - | 三维模型的边 |
| Vertex | - | 三维模型的顶点 |
| Bbox | Bounding Box | 包围盒，描述模型的空间范围 |

---

## 评估相关

| 术语 | 英文 | 定义 |
|------|------|------|
| LLM Judge | - | 使用 LLM 对生成结果进行质量评估的机制 |
| Ground Truth | - | 预先准备的标准答案，用作评估基准 |
| Rubric | - | 评分标准和规则的配置文件 |
| Evidence | - | 评估所需的上下文信息（代码、产物、截图等） |
| Dimension | - | 评分维度（如 requirement_fidelity, geometric_reasonableness）|
| Anchor | - | 评分锚点，定义不同等级的判定标准（high/mid/low）|
| Hard Gate | - | 硬性门槛，如无 STEP 时封顶分 |
| Jaccard | - | Jaccard 相似度系数，用于集合相似度计算 |
| Confidence | - | 评分置信度，表示评估结果的可靠性 |

---

## ACI 相关

| 术语 | 英文 | 定义 |
|------|------|------|
| ACI | Agent-Computer Interface | LLM 与 CAD 环境的交互接口 |
| Action | - | 基础 CAD 操作单元（如 extrude, fillet, hole）|
| Snapshot | - | 动作执行后的状态快照，包含几何信息和多视图截图 |
| Iteration | - | 多轮优化的每一轮，包含观察、决策、执行 |
| Rollback | - | 回退到之前的状态 |
| Modify Action | - | 修改之前动作的参数 |
| Suggestion | - | 基于几何分析自动生成的改进建议 |
| Completeness | - | 完整性诊断，评估模型是否完成 |
| Selector | - | 目标选择器，指定操作的目标（如 all_outer_edges, top_face）|

---

## 数据与配置

| 术语 | 英文 | 定义 |
|------|------|------|
| Benchmark | - | 标准测试集，包含样本和 Ground Truth |
| Prompt | - | 发送给 LLM 的指令文本 |
| Token | - | LLM API 使用的基本计费单位 |
| Provider | - | LLM 服务提供商（如 glm, kimi, claude）|
| Model | - | 具体的 LLM 模型名称（如 glm-4.7, kimi-k2.5）|
| Environment Variable | - | 环境变量，用于配置系统参数 |

---

## 测试相关

| 术语 | 英文 | 定义 |
|------|------|------|
| Unit Test | - | 单元测试，测试单个函数或类 |
| Integration Test | - | 集成测试，测试多个组件的协作 |
| E2E Test | End-to-End Test | 端到端测试，测试完整的用户流程 |
| Mock | - | 模拟对象，用于隔离测试 |
| Artifact | - | 测试或运行过程中生成的产物文件 |
