Ground Truth 评估

Ground Truth 评估

更新日期：2026-03-12 来源：docs/work_logs/EVALUATION_LLM_JUDGE_PLAN.md 和 docs/work_logs/2026_03_02_TERMINAL_TEST_PLAYBOOK.md

概述

Ground Truth (GT) 评估使用预先准备的标准答案作为参考，通过对比生成结果与标准答案来评估质量。

评估架构

评估流程

用户生成代码
    ↓
Sandbox 执行
    ↓
STEP 文件生成
    ↓
GT Evaluator
    ├─ 读取 GT STEP 文件
    ├─ 解析生成 STEP 文件
    ├─ 计算几何差异
    └─ 输出评分结果
    ↓
Evaluation (mode=ground_truth)

Benchmark Index

配置文件： configs/eval/benchmark_index.json

结构：

{
  "benchmark_name": "sampled_10_per_L",
  "metric_name": "step_data_jaccard_v1",
  "pass_threshold": 0.85,
  "root_dir": "/path/to/benchmark",
  "samples": {
    "sample_001": "L1/steps/sample_001.step",
    "sample_002": "L2/steps/sample_002.step",
    ...
  }
}

评估指标

step_data_jaccard_v1

原理： 使用 Jaccard 相似度系数比较两个 STEP 文件的几何特征

计算方式：

J(A, B) = |A ∩ B| / |A ∪ B|

其中：

  • A = GT STEP 文件的几何特征集合
  • B = 生成 STEP 文件的几何特征集合
  • |A ∩ B| = 共同特征数量
  • |A ∪ B| = 所有特征数量

评分范围： [0, 1]

  • 1.0 = 完全一致
  • 0.0 = 完全不同

pass_threshold

含义： 评估通过的阈值

默认值： 0.85

判断逻辑：

if score >= pass_threshold:
    passed = True
else:
    passed = False

使用方式

1. 配置 Benchmark Index

export SANDBOX_MCP_BENCHMARK_INDEX_PATH=/path/to/benchmark_index.json

2. 触发 GT 评估

方式 1：通过 MCP 调用

# Claude MCP 直连
obsidian call-tool execute_cadquery \
  --benchmark-sample-id sample_001

# 或通过代码调用
await session.call_tool('execute_cadquery', {
    'code': '...',
    'benchmark_sample_id': 'sample_001',
    'timeout_seconds': 120
})

方式 2：通过全链路

curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{"role":"user","content":"..."}],
    "benchmark_sample_id": "sample_001"
  }'

评估输出

ExecutionEvaluation (mode=ground_truth)

{
  "mode": "ground_truth",
  "status": "success",
  "score": 0.92,
  "passed": true,
  "details": {
    "metric_name": "step_data_jaccard_v1",
    "benchmark_sample_id": "sample_001",
    "gt_file": "L1/steps/sample_001.step",
    "generated_file": "model.step",
    "threshold": 0.85,
    "common_features": 120,
    "total_features": 130,
    "jaccard": 0.923
  }
}

数据库存储

位置： storage/cad_files/{job_id}/

文件：

  • model.step - 生成的 STEP 文件
  • evaluation.json - 评估结果

与 LLM Judge 的对比

维度Ground TruthLLM Judge
评估方式与标准答案对比LLM 语义分析
准确性高（基于客观几何）中（基于 LLM 理解）
适用场景有标准答案的基准测试无标准答案的真实场景
计算成本低（几何解析）高（LLM API 调用）
可扩展性有限（需要 GT）高（任何场景）
评分维度单一几何相似度多维度（5 个维度）

评估优先级

if benchmark_sample_id is not None:
    # 优先使用 Ground Truth 评估
    return ground_truth_evaluator.evaluate(...)
elif SANDBOX_MCP_LLM_JUDGE_ENABLED:
    # 无 GT 时使用 LLM Judge
    return llm_judge_evaluator.evaluate(...)
else:
    # 无评估
    return ExecutionEvaluation(mode="none", status="not_requested")
