ACI 四大支柱

ACI 四大支柱

更新日期：2026-03-12 来源：docs/work_logs/WORK_LOG_2026_03_05_CAD_ACTION_INTERFACE.md

概述

ACI (Agent-Computer Interface) 是 LLM 与 CAD 环境的交互接口，通过四大支柱设计实现智能化的 CAD 建模能力。

四大支柱

1. 可操作 (Actionable)

定义： LLM 发出离散的、可组合的 CAD 动作，而非完整脚本。

核心概念：

  • 将复杂的 CAD 操作拆解为基础动作单元
  • 每个动作都有明确的输入和输出
  • 动作之间可以组合形成复杂的建模流程

示例动作类型：

  • 草图：create_sketch, add_rectangle, add_circle, add_polygon
  • 实体：extrude, cut_extrude, revolve, loft
  • 特征：fillet, chamfer, hole, pattern_linear, pattern_circular
  • 状态：rollback, snapshot

优势：

  • 降低 LLM 推理复杂度
  • 每个动作可独立验证
  • 支持增量式建模

2. 可观测 (Observable)

定义： 系统在每次动作后返回结构化的状态快照。

状态快照包含：

信息类型字段说明
步骤信息step, features当前步骤和已执行特征
几何信息geometrysolids, faces, edges, volume, bbox, center_of_mass
问题检测issues模型有效性问题列表
视觉反馈imagesiso, top, front, right, section 视图

示例快照：

{
  "step": 5,
  "features": [
    "Sketch@XY",
    "Extrude(20)",
    "Hole(d=5)",
    "Fillet(r=2)"
  ],
  "geometry": {
    "solids": 1,
    "faces": 42,
    "edges": 120,
    "volume": 32000,
    "bbox": [80, 40, 20],
    "center_of_mass": [40, 20, 10]
  },
  "issues": [],
  "images": [
    "iso.png",
    "top.png",
    "front.png",
    "right.png",
    "section.png"
  ]
}

优势：

  • LLM 可实时了解模型状态
  • 支持智能决策和错误检测
  • 提供多角度视觉反馈

3. 可回滚 (Reversible)

定义： 动作历史允许回退到任意之前的状态。

实现机制：

  • 维护完整的动作历史记录
  • 通过回放机制实现 rollback
  • 支持指定步骤回退

操作示例：

// 回退到第 2 步
{
  "action": "rollback",
  "step": 2
}

优势：

  • 容错能力强
  • 支持探索性尝试
  • 降低错误成本

4. 可组合 (Composable)

定义： 动作可以引用并构建在之前的结果上。

组合方式：

  • 通过 feature 引用关联动作
  • 支持模式化操作（线性、圆形阵列）
  • 参数化设计

示例：

// 基于之前的 hole 创建线性阵列
{
  "action": "pattern_linear",
  "feature": "hole_1",
  "count": 4,
  "spacing": 30,
  "direction": "X"
}

优势：

  • 支持复杂设计模式
  • 减少重复代码
  • 提高建模效率

架构对比

Before: 一次性脚本生成

User Request
    ↓
LLM
    ↓
Full CadQuery Script
    ↓
Sandbox
    ↓
STEP File

问题：

  • 无法中途观察状态
  • 错误难以定位
  • 无法局部修改
  • 探索成本高

After: 迭代式动作交互

User Request
    ↓
LLM (Spec Agent)
    ↓
Action 1 → Sandbox CAD Runtime → State + Observation
    ↓
LLM observes and decides next action
    ↓
Action 2 → Sandbox CAD Runtime → State + Observation
    ↓
... (迭代循环)
    ↓
Final Model

优势：

  • 每步都有状态反馈
  • 错误即时发现
  • 支持局部调整
  • 探索成本低

扩展功能 (Phase B 实现)

智能建议

基于几何分析和当前状态，自动提供改进建议：

{
  "suggestions": [
    "建议添加圆角以提高强度",
    "考虑添加倒角便于装配",
    "检测到壁厚不均，建议优化"
  ]
}

完整性诊断

评估模型是否完成，以及还缺少哪些特征：

{
  "completeness": {
    "expected_steps": 8,
    "current_step": 5,
    "missing_features": ["fillets", "chamfers"],
    "can_continue": true,
    "confidence": 0.75
  }
}

自我调整

支持修改之前动作的参数，而非完全回退：

{
  "action": "modify_action",
  "target_step": 3,
  "new_params": {
    "radius": 3,
    "target": "all_outer_edges"
  }
}

会话持久化

跨 MCP 调用保持会话状态：

  • session_id 标识会话
  • 完整的 action_history 追踪
  • 支持 clear_session 重新开始
