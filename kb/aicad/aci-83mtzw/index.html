ACI 核心概念

什么是 ACI？

ACI (Agent-Computer Interface) 是一种让 LLM 与专业软件（如 CAD、IDE、开发工具）进行结构化交互的接口设计范式。

ACI 的历史背景

ACI 概念的演变：

阶段年代核心思想代表系统
早期2000-2010命令行接口Unix Shell, Windows CMD
图形时代2010-2020GUI 界面鼠标操作、菜单栏
API 时代2010-2020编程接口REST API, SDK
插件时代2015-2020插件系统IDE 插件、浏览器扩展
AI 时代2020-至今ACIClaude Code, Cursor, Windsurf

ACI 的本质

ACI 的本质是 将领域专业知识转化为 LLM 可理解的结构化接口。

传统 CAD（GUI）：
┌──────────────────────────────────┐
│  用户         │
│  ↓              │
│  鼠标操作      │ ← 灵活但难以自动化
│  → 复杂       │
│  ↓              │
│  结果         │
└──────────────────────────────────┘

ACI（结构化）：
┌──────────────────────────────────┐
│  LLM Agent      │
│  ↓              │
│  理解意图        │
│  → 规划动作       │ ← 可自动化、可推理
│  ↓              │
│  执行动作        │
│  ↓              │
│  观察状态        │
│  ↓              │
│  决策下一步       │
│  → 结果         │
└──────────────────────────────────┘

2. ACI 的 Prompt 设计模式

基础 Prompt

你是 CAD 建模助手，使用以下 ACI 动作接口：

可用动作：
- create_sketch(plane)
- add_rectangle(width, height, centered)
- extrude(distance, direction)
- fillet(radius, target)
- hole(diameter, depth, position)

使用规则：
1. 每次只执行一个动作
2. 观察返回的 snapshot 和 suggestions
3. 基于反馈决定下一步

系统化 Prompt

ACI 工作流程：

第 1 步：分析需求
- 理解用户想要创建什么
- 确定基本参数（尺寸、材料、精度）

第 2 步：规划动作序列
- 将复杂需求分解为动作序列
- 考虑依赖关系（草图 → 实体 → 特征）
- 预估所需步骤数

第 3 步：迭代执行
- 逐个执行动作
- 每步观察状态快照
- 处理错误和异常

第 4 步：验证与优化
- 检查是否符合需求
- 根据建议进行优化
- 确认模型质量

3. ACI 的思维链（Chain of Thought）

用户: "创建一个带四个安装孔的板"

思考 1 (Th1): [分析需求]
我需要创建一个矩形板，四角各有一个安装孔。
基本尺寸需要确定。

思考 2 (Th2): [确定参数]
假设板的尺寸为 100x60mm，厚度为 8mm。
安装孔直径为 6mm，需要距离边缘 15mm。
孔的位置应该在 (15,15), (85,15), (15,45), (85,45)。

思考 3 (Th3): [规划动作]
动作序列：
1. create_sketch(plane="XY")
2. add_rectangle(width=100, height=60, centered=True)
3. extrude(distance=8, direction="up")
4. create_sketch(plane=">Z")
5. hole(diameter=6, position=[15,15])
6. hole(diameter=6, position=[85,15])
7. hole(diameter=6, position=[15,45])
8. hole(diameter=6, position=[85,45])
9. fillet(radius=2, target="all_outer_edges")

思考 4 (Th4): [执行第 1 个动作]
执行: create_sketch(plane="XY")
观察: 草图已创建

4. ACI 的错误处理

# 错误分类和处理策略
errors = {
    "geometry_invalid": {
        "severity": "error",
        "action": "rollback",
        "message": "几何形状无效，无法继续"
    },
    "parameter_out_of_range": {
        "severity": "warning",
        "action": "adjust",
        "message": "参数超出范围，已自动调整"
    },
    "feature_creation_failed": {
        "severity": "error",
        "action": "retry",
        "message": "特征创建失败，正在重试"
    }
}

def handle_error(error, current_state):
    """ACI 错误处理"""
    error_type = classify_error(error)
    strategy = errors[error_type]

    if strategy["action"] == "rollback":
        # 回退到上一步
        rollback_to(current_state["previous_step"])
    elif strategy["action"] == "adjust":
        # 调整参数
        adjusted_params = adjust_parameters(error, current_state)
        return retry_with(adjusted_params)

4. 设计思维的迁移

传统 CAD 设计思维

设计师的工作流程：
1. 看需求（图纸、文字描述）
2. 在脑海中想象 3D 形状
3. 思考如何构建：
   - 先创建草图
   - 再拉伸成实体
   - 添加圆角
   - 打孔
4. 边做边调整参数
5. 边预览效果
6. 完成后标注尺寸

特点：

  • 经验驱动：依赖设计师的经验和直觉
  • 视觉反馈：通过 GUI 实时看到效果
  • 试错过程：多次调整参数
  • 难以文档化：很多决策是隐性的

ACI 设计思维

ACI Agent 的工作流程：
1. 理解需求（自然语言）
2. 拆解为标准 CAD 操作（调用 API）
3. 规划动作序列（结构化思考）
4. 执行并观察：
   - 调用 action API
   - 获取状态快照
   - 获取 AI 建议
5. 基于反馈调整：
   - 如果 suggestion 建议添加圆角 → 执行 fillet
   - 如果 snapshot 显示尺寸不对 → 重新建模
6. 验证：检查完整性指标

特点：

  • 结构化推理：每步都有明确的 reasoning
  • 可解释性：可以查看动作历史
  • 可复制：相同的推理可以重复执行
  • 可审计：每个决策都有记录

ACI 与传统 CAD 的对比

维度对比表

维度传统 CAD（GUI）ACI（基于 Agent）
交互方式鼠标/键盘自然语言对话
学习曲线陡峭（需学习软件）低（会自然语言）
灵活性高（可任意操作）中（受限于 API）
自动化需要脚本/宏可全自动（LLM 决策）
可扩展性需要编程/插件易扩展（添加新 action）
版本控制手动保存集成 Git
协作文件共享多人并行工作
可解释性隐性（难以追溯）显性（有 reasoning 日志）
错误恢复Ctrl+Z（有限）历史回退（完整）

ACI 的设计原则

1. 原子化（Atomicity）

每个动作应该是原子的、不可分割的最小单位。

# 好的设计
good_action = {
    "action": "create_sketch",
    "plane": "XY"
}

# 不好的设计（多个动作混合）
bad_action = {
    "action": "create_sketch_and_add_rect_and_extrude",
    "sketch": "XY",
    "rect": [80, 40],
    "extrude": {"distance": 5}
}

2. 幂等性（Idempotency）

相同动作重复执行应产生相同结果。

# 幂等性保证
def execute_action(action, state):
    # 检查是否已经执行过相同动作
    if is_already_executed(action, state):
        return get_cached_result(action, state)
    return execute_fresh(action, state)

3. 可组合性（Composability）

动作可以组合成复杂模式。

# 组合模式
def create_plate_with_holes():
    """创建带孔的板 - 可组合的模式"""
    actions = [
        create_sketch("XY"),
        add_rectangle(100, 60),
        extrude(5),
        create_sketch(">Z"),
        pattern_linear_holes(hole_positions)
    ]
    return execute_sequence(actions)

4. 可逆性（Reversibility）

大部分操作应该可逆，允许回退。

# 可逆操作映射
reversible_actions = {
    "create_sketch": "delete_sketch",
    "add_rectangle": "remove_rectangle",
    "extrude": "extrude",
    "fillet": "remove_fillet",
    "hole": "fill_hole"
}

# 回退操作
def rollback_to_step(target_step):
    """回退到指定步骤"""
    current_step = get_current_step()
    steps_to_undo = current_step - target_step

    for _ in range(steps_to_undo):
        action = pop_last_action()
        reverse_action = get_reverse_action(action)
        execute(reverse_action)

5. 可观测性（Observability）

每次执行后应该返回丰富的状态信息。

{
  "action": "extrude",
  "success": true,
  "snapshot": {
    "geometry": {
      "solids": 1,
      "faces": 6,
      "volume": 32000
    },
    "features": ["Extrude(5)", "Box(100x60x5)"],
    "images": ["iso.png", "top.png", "front.png", "right.png"]
  },
  "suggestions": [
    "建议添加圆角以提高强度",
    "考虑添加安装孔"
  ],
  "completeness": {
    "expected_steps": 6,
    "current_step": 3,
    "can_continue": true,
    "confidence": 0.6
  }
}
