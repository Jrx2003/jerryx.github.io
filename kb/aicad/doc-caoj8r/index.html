评价系统概述

核心目标

让 LLM 对生成的 CAD 模型进行准确、一致、可解释的质量评价。

评估系统架构

┌─────────────────────────────────────────────┐
│           Sandbox/ACI               │
│              (被评估对象)              │
└──────────────────┬──────────────────────┘
                 │ 产物收集
                 ↓
┌──────────────────────────────────────┐
│       Evidence Builder             │
│   (收集评估所需的证据)          │
└──────────────────┬──────────────────┘
                 │
                 ↓
┌──────────────────────────────────────┐
│      Rule Evaluator            │
│   (硬规则检查：无 STEP、超时等)  │
└──────────────────┬──────────────────┘
                 │
                 ↓
┌──────────────────────────────────────┐
│       LLM Judge               │
│   (语义评估：理解、准确度等)        │
└──────────────────┬──────────────────┘
                 │
                 ↓
┌──────────────────────────────────────┐
│      Score Merger               │
│   (合并规则和 LLM 分数)             │
└──────────────────┬──────────────────┘
                 │
                 ↓
┌──────────────────────────────────────┐
│       Final Evaluation            │

评价维度设计

核心维度定义

维度权重说明评估要点
需求符合度0.35模型是否符合用户的原始需求尺寸、形状、位置、数量
几何合理性0.25几何结构是否合理拓扑正确、比例协调、无几何异常
可制造性0.20模型是否适合标准制造工艺壁厚均匀、无锐角、可脱模
代码质量0.10生成代码的质量代码简洁、可读、高效
执行稳定性0.10生成过程的稳定性无错误、超时、重试次数

维度评分标准

需求符合度

高 (1.0):
- 关键尺寸与约束完全满足
- 所有要求特征都已实现
- 无多余或遗漏

中 (0.5):
- 主形状和尺寸正确
- 次要特征基本实现
- 有小的细节偏差

低 (0.0):
- 核心形状或尺寸不正确
- 缺少关键特征
- 严重偏离需求

几何合理性

高 (1.0):
- 拓扑正确，无自相交
- 比例协调，无异常比例
- 布局合理

中 (0.5):
- 拓扑基本正确
- 有局部小比例不协调
- 无明显几何异常

低 (0.0):
- 存在明显的几何错误
- 比例严重失调
- 存在无法制造的结构

LLM Judge Prompt 设计

Prompt 结构

你是一个专业的 CAD 模型评估专家。

## 评估任务
评估用户生成的 CAD 模型是否符合给定的需求。

## 评估维度
你需要从以下 5 个维度评估模型：

1. **需求符合度 (35%)**
   模型是否符合用户的原始需求？
   关键尺寸、形状、位置、数量是否正确？

2. **几何合理性 (25%)**
   模型的几何结构是否合理？
   拓扑正确、比例协调、无几何异常？

3. **可制造性 (20%)**
   模型是否适合标准制造工艺？
   壁厚均匀、无锐角、可脱模？

4. **代码质量 (10%)**
   生成代码的质量如何？
   代码简洁、可读、高效？

5. **执行稳定性 (10%)**
   生成过程的稳定性如何？
   无错误、超时、重试次数？

## 评分标准
对每个维度，给出 0.0 到 1.0 的分数和详细解释。

### Self-Consistency 检查

自我一致性检查

作为 LLM Judge，在给出评分前先自我检查：

  1. 我是否正确理解了需求？
  2. 我的评分是否一致？
  3. 我是否考虑了所有相关信息？
  4. 我的推理是否有逻辑漏洞？

如果发现不一致，请在 reasoning_brief 中说明。

## 评分算法设计

### 1. 加权平均

```python
def calculate_overall_score(dimension_scores: dict, weights: dict) -> float:
    """计算加权总分"""

    total_score = 0.0
    for dimension, score in dimension_scores.items():
        weight = weights.get(dimension, 0)
        total_score += score * weight

    return total_score

2. 硬规则门限

def apply_hard_gates(score: float, rules: dict) -> float:
    """应用硬性规则门限"""

    final_score = score

    # 无 STEP 产物封顶
    if "missing_step" in rules and rules["missing_step_max_score"] < 1.0:
        final_score = min(score, rules["missing_step_max_score"])

    # 空文件封顶
    if "empty_artifact" in rules and rules["empty_artifact_max_score"] < 1.0:
        final_score = min(score, rules["empty_artifact_max_score"])

    # 超时封顶
    if "timeout" in rules and rules["timeout_max_score"] < 1.0:
        final_score = min(score, rules["timeout_max_score"])

    # 执行错误封顶
    if "execution_error" in rules and rules["execution_error_max_score"] < 1.0:
        final_score = rules["execution_error_max_score"]

    return final_score

3. 置信度调整

def adjust_for_confidence(
    base_score: float,
    confidence: float,
    low_threshold: float = 0.65
) -> tuple[float, float]:
    """根据置信度调整分数"""

    if confidence < low_threshold:
        # 低置信度时降低分数
        adjusted_score = base_score * confidence
        return adjusted_score, confidence

    # 高置信度保持或轻微提升
    return base_score, confidence

评估管道设计

1. 串行评估

async def evaluate_serial(
    evidence: EvidenceBundle,
    rule_evaluator: RuleEvaluator,
    llm_evaluator: LLMJudgeEvaluator,
    score_merger: ScoreMerger
) -> Evaluation:
    """串行评估流程"""

    # 1. 规则评估
    rule_result = rule_evaluator.evaluate(evidence)

    # 2. 如果规则评估通过，进行 LLM 评估
    if rule_result.passed:
        llm_result = await llm_evaluator.evaluate(evidence)
    else:
        llm_result = None

    # 3. 合并分数
    evaluation = score_merger.merge(rule_result, llm_result)

    return evaluation

2. 并行评估

async def evaluate_parallel(
    evidence_list: list[EvidenceBundle],
    evaluators: list[Evaluator]
) -> list[Evaluation]:
    """并行评估多个证据"""

    # 为每个证据创建评估任务
    tasks = []
    for evidence in evidence_list:
        for evaluator in evaluators:
            task = evaluator.evaluate(evidence)
            tasks.append(task)

    # 并行执行
    results = await asyncio.gather(*tasks)

    return results

3. 批量评估

async def evaluate_batch(
    evidence_list: list[EvidenceBundle],
    max_concurrent: int = 5
) -> list[Evaluation]:
    """批量评估（带并发限制）"""

    # 使用信号量控制并发
    semaphore = asyncio.Semaphore(max_concurrent)

    async def evaluate_one(evidence):
        async with semaphore:
            return await llm_judge.evaluate(evidence)

    # 创建任务
    tasks = [evaluate_one(evidence) for evidence in evidence_list]
    results = await asyncio.gather(*tasks)

    return results

评估质量控制

1. 评估一致性验证

class EvaluationConsistencyChecker:
    """评估一致性检查器"""

    def __init__(self):
        self.evaluation_history = []

    def check_consistency(
        self,
            new_evaluation: Evaluation,
            old_evaluation: Optional[Evaluation]
        ) -> ConsistencyReport:
        """检查评估结果的一致性"""

        if not old_evaluation:
            return ConsistencyReport(consistent=True)

        # 检查分数变化范围
        score_delta = abs(new_evaluation.score - old_evaluation.score)
        if score_delta > 0.3:
            return ConsistencyReport(
                consistent=False,
                reason=f"分数大幅上升: {old_evaluation.score} → {new_evaluation.score}"
            )

        # 检查维度分数变化
        for dimension, new_score in new_evaluation.dimension_scores.items():
            old_score = old_evaluation.dimension_scores.get(dimension, 0)
            delta = abs(new_score - old_score)
            if delta > 0.4:
                return ConsistencyReport(
                    consistent=False,
                    reason=f"{dimension} 分数大幅上升: {old_score} → {new_score}"
                )

        return ConsistencyReport(consistent=True)

2. 离群值检测

import numpy as np

class OutlierDetector:
    """离群值检测"""

    def __init__(self, window_size: int = 50):
        self.window_size = window_size
        self.score_history = []

    def is_outlier(self, score: float) -> bool:
        """检测分数是否为离群值"""

        if len(self.score_history) < self.window_size:
            return False

        # 计算窗口内分数的统计量
        window = self.score_history[-self.window_size:]
        mean = np.mean(window)
        std = np.std(window)

        # 使用 3-sigma 规则
        lower_bound = mean - 3 * std
        upper_bound = mean + 3 * std

        # 判断是否为离群
        is_outlier = score < lower_bound or score > upper_bound

        # 更新历史
        self.score_history.append(score)

        return is_outlier

        self,

            duration_seconds: float
    ):
        """记录评估时间"""

    def get_performance_stats(self) -> dict:

        if not self.evaluation_times:
            return {}

        eval_times = [e["duration"] for e in self.evaluation_times]
