ACI 会话管理

ACI 会话管理

更新日期：2026-03-12 来源：docs/work_logs/WORK_LOG_2026_03_05_CAD_ACTION_INTERFACE.md

概述

ACI 会话管理允许在多个 MCP 调用之间保持建模状态，实现跨调用的持续建模能力。

SessionManager 架构

位置

services/sandbox/mcp_server/src/sandbox_mcp_server/service.py

核心数据结构

class SessionState(BaseModel):
    """会话状态模型"""
    history: list[ActionHistoryEntry]  # 完整动作序列
    current_step: int                 # 当前步骤计数器
    created_at: datetime               # 会话创建时间戳

class ActionHistoryEntry(BaseModel):
    """单条动作历史记录"""
    step: int                        # 步骤编号
    action_type: CADActionType         # 动作类型
    params: dict                      # 动作参数
    snapshot: CADStateSnapshot         # 执行后状态快照
    success: bool                    # 是否成功
    error: str | None               # 错误信息（如有）

SessionManager API

class SessionManager:
    """ACI 会话管理器"""

    def __init__(self, max_history_length: int = 50):
        self._sessions: dict[str, SessionState] = {}
        self.max_history_length = max_history_length

    def get_or_create_session(self, session_id: str) -> SessionState:
        """获取或创建会话"""
        if session_id not in self._sessions:
            self._sessions[session_id] = SessionState(
                history=[],
                current_step=0,
                created_at=datetime.now()
            )
        return self._sessions[session_id]

    def clear_session(self, session_id: str) -> None:
        """清除会话状态"""
        if session_id in self._sessions:
            del self._sessions[session_id]

    def append_action(self, session_id: str, entry: ActionHistoryEntry) -> None:
        """追加动作到历史（max 50 entries）"""
        session = self.get_or_create_session(session_id)
        session.history.append(entry)
        session.current_step += 1
        # 限制历史长度
        if len(session.history) > self.max_history_length:
            session.history = session.history[-self.max_history_length:]

    def get_session_history(self, session_id: str) -> list[ActionHistoryEntry]:
        """获取完整动作历史"""
        session = self.get_or_create_session(session_id)
        return session.history

会话操作

CLEAR_SESSION

清除当前会话，重新开始建模。

请求：

{
  "action": "clear_session"
}

效果：

  • 删除会话中的所有历史记录
  • 重置步骤计数器到 0
  • 后续动作开始全新建模

GET_HISTORY

获取完整动作历史。

请求：

{
  "action": "get_history",
  "session_id": "my_model",
  "include_history": true
}

响应：

{
  "action_history": [
    {
      "step": 1,
      "action_type": "create_sketch",
      "params": {"plane": "XY"},
      "snapshot": {...},
      "success": true,
      "error": null
    },
    {
      "step": 2,
      "action_type": "add_rectangle",
      "params": {"width": 80, "height": 40, "centered": true},
      "snapshot": {...},
      "success": true,
      "error": null
    }
  ]
}

MODIFY_ACTION

修改之前动作的参数，无需完全回退。

请求：

{
  "action": "modify_action",
  "target_step": 3,
  "new_params": {
    "radius": 3,
    "target": "all_outer_edges"
  }
}

处理流程：

  1. 验证 target_step 参数
  2. 获取会话历史
  3. 合并 new_params 与原始动作参数
  4. 从修改后的动作重新构建模型（重放从步骤 1 到目标步骤）
  5. 更新会话历史中的条目
  6. 返回更新的快照和完整性

会话持久化机制

当前实现

  • 存储方式： 内存中（Python dict）
  • 生命周期： MCP 服务器运行期间
  • 限制： 服务器重启后丢失

未来扩展方向

方案 1：Redis 持久化

class RedisSessionManager(SessionManager):
    """Redis 持久化会话管理"""

    def __init__(self, redis_url: str):
        self.redis = redis.from_url(redis_url)

    def get_or_create_session(self, session_id: str) -> SessionState:
        # 从 Redis 读取或创建
        data = self.redis.get(f"session:{session_id}")
        if data:
            return SessionState.parse_raw(data)
        return SessionState(history=[], current_step=0)

    def save_session(self, session_id: str, session: SessionState) -> None:
        # 保存到 Redis
        self.redis.setex(
            f"session:{session_id}",
            3600,  # 1 小时过期
            session.json()
        )

方案 2：数据库持久化

class DatabaseSessionManager(SessionManager):
    """数据库持久化会话管理"""

    def get_or_create_session(self, session_id: str) -> SessionState:
        # 从数据库查询或创建
        session = db.query(SessionState).filter_by(id=session_id).first()
        if not session:
            session = SessionState(id=session_id, history=[], current_step=0)
            db.add(session)
        return session

    def append_action(self, session_id: str, entry: ActionHistoryEntry) -> None:
        # 追加动作并持久化
        session = self.get_or_create_session(session_id)
        session.history.append(entry)
        session.current_step += 1
        db.commit()

会话使用模式

模式 1：单会话多步建模

# 第一次调用
apply_cad_action({
    "session_id": "phone_case",
    "actions": [{"action": "create_sketch", "plane": "XY"}, ...]
})

# 第二次调用（基于第一次结果）
apply_cad_action({
    "session_id": "phone_case",
    "actions": [{"action": "extrude", "distance": 2}, ...]
})

# 第三次调用（基于前两次结果）
apply_cad_action({
    "session_id": "phone_case",
    "actions": [{"action": "fillet", "radius": 0.5}, ...]
})

模式 2：修改已执行动作

# 原始序列
step 1: create_sketch
step 2: add_rectangle
step 3: extrude (distance=10)
step 4: fillet (radius=1)

# 发现需要修改 step 3
apply_cad_action({
    "session_id": "phone_case",
    "action": "modify_action",
    "target_step": 3,
    "new_params": {"distance": 15}
})

# 结果：重放并更新
step 1: create_sketch
step 2: add_rectangle
step 3: extrude (distance=15)  ← 已修改
step 4: fillet (radius=1)       ← 保持原样

模式 3：查看历史并继续

# 查看历史
get_history({"session_id": "phone_case"})
# 返回完整历史和快照

# 基于历史决策下一步
apply_cad_action({
    "session_id": "phone_case",
    "actions": [{"action": "hole", "diameter": 2}, ...]
})

模式 4：完全重新开始

# 清除所有历史
apply_cad_action({
    "action": "clear_session"
})

# 从头开始新建模
apply_cad_action({
    "session_id": "phone_case",
    "actions": [{"action": "create_sketch", "plane": "XY"}, ...]
})

性能考虑

内存使用

  • 假设： 100 个并发会话，每个平均 20 个历史条目
  • 估算： 100 * 20 * ~2KB = ~4MB（可接受）

历史长度限制

  • 当前： 50 条目/会话
  • 建议： 可配置，基于实际使用调整

过期策略

  • 内存： LRU 策略或定期清理
  • Redis： TTL 自动过期
  • 数据库： 定期清理过期会话
