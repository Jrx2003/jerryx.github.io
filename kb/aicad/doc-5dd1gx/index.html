CLI 设计原则

1. 自描述性 (Self-Describing)

CLI 工具应该能够通过 --help 标志自动生成文档，LLM 可以通过阅读帮助理解工具能力。

示例：

$ cadcli --help
CAD CLI - 用于 3D 建模的命令行工具

USAGE:
  cadcli [OPTIONS] COMMAND [ARGS]...

OPTIONS:
  --help      显示帮助信息并退出
  --version   显示版本信息并退出

COMMANDS:
  create      创建新的 CAD 模型
  modify      修改现有 CAD 模型
  export      导出 CAD 模型到文件
  validate    验证 CAD 模型的有效性

设计要点：

  • 使用层级结构：命令 -> 子命令
  • 每个命令有独立的 --help
  • 帮助文档包含示例

2. 结构化输出

LLM 是基于文本的，但 CAD 工具需要处理复杂的数据结构。

输出格式选择：

格式适用场景LLM 友好度
JSON程序化消费高
表格简单列表显示中
人类可读终端直接查看高

JSON 输出示例：

$ cadcli create plate 80 40 5
{
  "success": true,
  "model_id": "plate_80x40x5",
  "geometry": {
    "type": "box",
    "dimensions": [80, 40, 5],
    "volume": 16000,
    "bounding_box": {
      "min": [0, 0, 0],
      "max": [80, 40, 5]
    }
  },
  "export_paths": [
    "/tmp/plate_80x40x5.step",
    "/tmp/plate_80x40x5.stl"
  ]
}

3. 交互式确认

对破坏性或重要操作，应该要求用户确认。

示例：

$ cadcli delete model_12345
⚠️  即将删除模型 model_12345
此操作不可逆。
确认删除？[y/N]: y
✓ 模型已删除

4. 进度反馈

长时间运行的操作应显示进度条或状态更新。

示例：

$ cadcli export model_12345 output.step
[=====>   ] 75% 正在导出...
正在生成 STEP 文件...
✓ 导出完成: output.step (2.3 MB)

LLM 与 CLI 的交互模式

模式 1: 直接命令生成

LLM 直接生成 CLI 命令字符串，通过系统执行。

优点：

  • 实现简单
  • LLM 控制完整流程

缺点：

  • 难以处理复杂交互
  • 无法利用结构化输出

示例：

# LLM 生成的命令
command = "cadcli create plate --length 80 --width 40 --thickness 5"

# 执行
import subprocess
result = subprocess.run(command, shell=True, capture_output=True)
print(result.stdout)

模式 2: 结构化函数调用

LLM 使用 Function Calling 或 Tool Use 调用 CLI 的特定功能。

优点：

  • 可以结构化处理参数
  • 支持并行调用
  • 可以处理复杂交互

实现架构：

{
  "name": "cad_create_plate",
  "description": "创建矩形板",
  "parameters": {
    "type": "object",
    "properties": {
      "length": {"type": "number", "description": "板长度 (mm)"},
      "width": {"type": "number", "description": "板宽度 (mm)"},
      "thickness": {"type": "number", "description": "板厚度 (mm)"}
    },
    "required": ["length", "width", "thickness"]
  }
}

模式 3: 混合交互

结合命令生成和结构化输出，LLM 先生成命令，解析结构化输出。

优点：

  • 灵活性高
  • 可以利用结构化输出
  • 逐步引导复杂操作

示例：

# 1. LLM 生成探索命令
$ cadcli explore model_12345

# 2. 解析 JSON 输出获取模型信息
{
  "features": ["box", "cylinders", "holes"],
  "summary": "包含 1 个长方体、3 个圆柱和 4 个孔"
}

# 3. 基于信息生成修改命令
$ cadcli modify model_12345 add-hole --x 20 --y 15 --diameter 5

CAD CLI 工具设计要点

1. 命令设计

create 命令

功能： 创建新的 CAD 模型

参数设计：

cadcli create [TYPE] [OPTIONS]

类型:
  box         长方体
  cylinder     圆柱
  plate        板（常用配置）
  bracket      支架

通用选项:
  --length FLOAT       长度 (mm)
  --width FLOAT        宽度 (mm)
  --height FLOAT       高度 (mm)
  --radius FLOAT       半径 (mm)
  --thickness FLOAT    厚度 (mm)
  --output PATH       输出文件路径

孔配置（仅 plate/bracket）:
  --hole-count INT          孔数量
  --hole-diameter FLOAT    孔径
  --hole-pattern TYPE      孔模式: grid|circle|linear

示例：

# 创建简单长方体
cadcli create box --length 80 --width 40 --thickness 5

# 创建带孔的板
cadcli create plate --length 100 --width 60 --thickness 8 \
  --hole-count 4 --hole-diameter 5 --hole-pattern grid

modify 命令

功能： 修改现有 CAD 模型

参数设计：

cadcli modify MODEL_ID [OPERATION] [OPTIONS]

操作:
  add-hole       添加孔
  add-fillet     添加圆角
  chamfer         添加倒角
  scale           缩放
  move            移动
  rotate          旋转

通用选项:
  --value FLOAT      操作值
  --target INT      目标索引
  --direction VECTOR 方向向量
  --axis AXIS        旋转轴
  --angle FLOAT     旋转角度

示例：

# 添加圆角
cadcli modify model_12345 add-fillet --radius 2 --all-edges

# 绕 Z 轴旋转 45 度
cadcli modify model_12345 rotate --axis Z --angle 45

# 缩放 1.5 倍
cadcli modify model_12345 scale --factor 1.5

export 命令

功能： 导出 CAD 模型到各种格式

参数设计：

cadcli export MODEL_ID [OPTIONS]

选项:
  --format TYPE      输出格式: step|stl|obj|amf
  --output PATH      输出文件路径
  --quality INT       质量/精度等级 (1-10)
  --binary           二进制模式（ASCII 的替代）

示例：

# 导出为 STEP
cadcli export model_12345 --format step --output plate.step

# 高质量 STL 导出
cadcli export model_12345 --format stl --quality 10 --binary

validate 命令

功能： 验证 CAD 模型的有效性

验证项目：

cadcli validate MODEL_ID [OPTIONS]

选项:
  --geometry          检查几何有效性
  --manifold          检查流形（manifold）
  --non-intersecting  检查自相交
  --volume FLOAT      检查体积范围
  --min-thickness     检查最小壁厚

示例：

# 全面验证
cadcli validate model_12345 --geometry --manifold --min-thickness 2.0

# 快速几何检查
cadcli validate model_12345 --geometry

错误代码体系

    "GEOMETRY_INVALID": {
      "code": 1001,

      "user_action": "检查模型参数"

    "FILE_NOT_FOUND": {
      "code": 2001,

      "code": 3001,

      "code": 4001,

错误消息模板

错误: [CODE] MESSAGE

位置: FILE:LINE
上下文: ...

示例：

$ cadcli create plate --length -10
错误: [GEOMETRY_INVALID] 尺寸必须为正数

位置: create.py:45
上下文: 正在解析 --length 参数
建议: 使用 --length 10 而不是 -10

位置: FILE:LINE:1

3. 配置管理

配置文件

# cadcli.yaml
defaults:
  output_format: step
  output_quality: 8
  auto_confirm: false

presets:
  standard_plate:
    length: 100
    width: 60
    thickness: 8

  phone_case:
    length: 150
    width: 75
    thickness: 10
    corner_radius: 5

使用配置：

$ cadcli --config cadcli.yaml create plate --preset standard_plate

环境变量

# 默认输出目录
export CADCLI_OUTPUT_DIR=/tmp/cad_output

# 默认 STEP 格式选项
export CADCLI_STEP_EXPORT_BINARY=true

# 验证级别
export CADCLI_VALIDATION_LEVEL=strict

4. 插件系统

插件架构

cadcli/
├── plugins/
│   ├── __init__.py
│   ├── base.py           # 插件基类
│   ├── validators/       # 验证器插件
│   └── exporters/        # 导出器插件
├── core/
└── cli.py

插件接口

from abc import ABC, abstractmethod

class PluginBase(ABC):
    @abstractmethod
    def get_name(self) -> str:
        """获取插件名称"""
        pass

    @abstractmethod
    def get_version(self) -> str:
        """获取插件版本"""
        pass

    @abstractmethod
    def execute(self, context: dict) -> dict:
        """执行插件功能"""
        pass

class ValidatorPlugin(PluginBase):
    @abstractmethod
    def validate(self, model_data: dict) -> ValidationResult:
        pass

class ExporterPlugin(PluginBase):
    @abstractmethod
    def export(self, model_data: dict, format: str, output: str) -> bool:
        pass

实现建议

简单场景：直接命令生成

适用场景：

  • 单一操作
  • 参数明确的任务
  • 快速原型验证

实现示例：

# 简单创建板
cadcli create plate 80 40 5

复杂场景：结构化函数调用

适用场景：

  • 多步工作流
  • 需要错误处理
  • 需要结构化返回

实现示例：

# 通过 MCP 或 Function Calling
{
  "name": "create_mounting_plate",
  "description": "创建安装板",
  "steps": [
    {"command": "create_plate", "params": {...}},
    {"command": "add_holes", "params": {...}},
    {"command": "add_fillets", "params": {...}}
  ]
}

高级场景：混合交互

适用场景：

  • 需要逐步探索
  • 需要人工干预
  • 需要复杂决策

实现示例：

# 1. 探索现有模型
cadcli list models

# 2. 查看模型详情
cadcli show model_12345

# 3. 基于信息决定下一步
cadcli add-feature model_12345 --type stiffener

最佳实践

1. 命名约定

文件命名：

{模型名}_{类型}.{格式}
示例: plate_001.step, cylinder_002.stl

ID 生成：

import uuid

model_id = str(uuid.uuid4())  # 如 "a1b2c3d4-e5f6-7890"

2. 向后兼容性

  • 支持旧版命令格式
  • 弃用警告而非错误
  • 迁移指南

3. 测试策略

单元测试

import unittest

class TestCreatePlate(unittest.TestCase):
    def test_create_default_plate(self):
        """测试默认参数创建板"""
        result = cadcli.create_plate()
        self.assertTrue(result['success'])

    def test_create_custom_plate(self):
        """测试自定义参数创建板"""
        result = cadcli.create_plate(length=100, width=80)
        self.assertEqual(result['geometry']['dimensions'], [100, 80, 10])

集成测试

# 测试完整工作流
$ bash tests/integration/test_plate_workflow.sh

# 验证输出文件
$ cadcli export test_model.step
$ python -c "import cadquery as cq; cq.importers.importStep('test_model.step')"
