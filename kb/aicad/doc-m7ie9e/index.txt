## 目的

本手册用于两类可复现验证：

1. **Claude Code 直连** `sandbox_mcp_server` 的 `execute_cadquery` 工具
2. **启动项目全链路**（API Gateway → Main Agent → Sub Agent → MCP → Sandbox）并验证 `llm_judge` 结果落盘

---

## 前置条件

### 1. 安装依赖

```bash
cd /Users/jerryx/code/aicad.app.agent
uv sync
```

---

### 2. 加载环境变量

建议用 `source .env`，不要用 `uv run --env-file .env`

```bash
set -a
source .env
set +a
```

---

### 3. 确认关键变量

```bash
echo "$GLM_API_KEY" | wc -c
echo "$LLM_STANDARD_PROVIDER"
echo "$LLM_STANDARD_MODEL"
```

**期望：**
- `GLM_API_KEY` 非空
- `LLM_STANDARD_PROVIDER=glm`
- `LLM_STANDARD_MODEL=glm-4.7`

---

## 场景 1：Claude Code 直连 MCP 测试（不经过主流程）

### 1. 注册 MCP Server

```bash
cd /Users/jerryx/code/aicad.app.agent
claude mcp remove -s local aicad-sandbox >/dev/null 2>&1 || true
claude mcp add -s local aicad-sandbox -- uv run python -m sandbox_mcp_server
claude mcp get aicad-sandbox
```

---

### 2. 执行测试并保存输出

```bash
cd /Users/jerryx/code/aicad.app.agent
mkdir -p .tmp/claude_proof
export SANDBOX_MCP_LLM_JUDGE_ENABLED=true
export SANDBOX_MCP_BENCHMARK_INDEX_PATH=
```

---

#### Case A：执行成功但不符合 requirement（应低 `requirement_fidelity`）

```bash
claude -p --permission-mode bypassPermissions --output-format json \
"请调用 aicad-sandbox 的 execute_cadquery 工具，参数如下：
code=\"import cadquery as cq
result = cq.Workplane('XY').box(10,10,3)
\"
timeout_seconds=120
requirement_text=\"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔。\"
调用后仅输出JSON对象，字段：
success,error_code,evaluation_mode,evaluation_status,evaluation_score,requirement_text_present,requirement_context_source,requirement_fidelity,major_issue_1。" \
> .tmp/claude_proof/mismatch.json
```

---

#### Case B：执行成功且符合 requirement（应高 `requirement_fidelity`）

```bash
claude -p --permission-mode bypassPermissions --output-format json \
"请调用 aicad-sandbox 的 execute_cadquery 工具，参数如下：
code=\"import cadquery as cq
length = 60
width = 40
thickness = 5
hole_d = 6
offset_x = length / 2 - 8
offset_y = width / 2 - 8
result = (
    cq.Workplane('XY')

)
\"
timeout_seconds=120
requirement_text=\"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔。\"
调用后仅输出JSON对象，字段：
success,evaluation_mode,evaluation_status,evaluation_score,requirement_fidelity,major_issue_1。" \

## 场景 2：全流程联调（API/MainAgent/SubAgent/MCP）

### 1. 启动基础设施

```bash
cd /Users/jerryx/code/aicad.app.agent
cd infra && docker compose up -d && cd ..
./scripts/run_migrations.sh
```

---

### 2. 终端 A：启动 API Gateway

```bash
cd /Users/jerryx/code/aicad.app.agent
set -a
source .env
set +a
export SANDBOX_TYPE=mcp
export SANDBOX_MCP_LLM_JUDGE_ENABLED=true
export SANDBOX_MCP_BENCHMARK_INDEX_PATH=
export SANDBOX_MCP_SERVER_ARGS=run,python,-m,sandbox_mcp_server
export SANDBOX_MCP_SERVER_CWD="$(pwd)"
uv run uvicorn api_gateway.main:app --host 0.0.0.0 --port 8100
```

---

### 3. 终端 B：启动 Sub Agent

```bash
cd /Users/jerryx/code/aicad.app.agent
set -a
source .env
set +a
export SANDBOX_TYPE=mcp
export SANDBOX_MCP_LLM_JUDGE_ENABLED=true
export SANDBOX_MCP_BENCHMARK_INDEX_PATH=
export SANDBOX_MCP_SERVER_ARGS=run,python,-m,sandbox_mcp_server
export SANDBOX_MCP_SERVER_CWD="$(pwd)"
uv run python -m sub_agent.main
```

---

### 4. 终端 C：发起请求并确认建模

#### 请求 1（应返回 `confirmation_required=true`）

```bash
curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[{"role":"user","content":"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔"}],
    "stream":false
  }' | jq .
```

---

#### 请求 2（确认创建任务）

```bash
RESP=$(curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[
      {"role":"user","content":"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔"},
      {"role":"assistant","content":"我将创建一个矩形安装板。\\n\\n```design-summary\\nrectangular mounting plate, length 60, width 40, thickness 5, four corner holes diameter 6\\n```"}
    ],
    "stream":false,
    "confirm_job_creation":true
  }')
echo "$RESP" | jq .
JOB_ID=$(echo "$RESP" | jq -r '.job_id')
echo "JOB_ID=$JOB_ID"
```

---

### 5. 轮询任务直到完成

```bash
while true; do
  STATUS_JSON=$(curl -sS "http://localhost:8100/v1/jobs/$JOB_ID" \
    -H "Authorization: Bearer dev_token_1")
  echo "$STATUS_JSON" | jq '{job_id,status,result}'
  STATUS=$(echo "$STATUS_JSON" | jq -r '.status')
  if [ "$STATUS" = "succeeded" ] || [ "$STATUS" = "failed" ]; then
    break
  fi
  sleep 2
done
```

## 结果验证（STEP + 评估报告）

### 本地存储（`STORAGE_TYPE=local` 时）

```bash
ls -lh "storage/cad_files/$JOB_ID"
```

**应至少包含：**
1. `model.step`
2. `evaluation.json`

---

### 检查评估关键字段

```bash
jq '{mode,status,score,details:{requirement_text_present,requirement_context_source,requirement_fidelity,requirement_text_excerpt}}' \
  "storage/cad_files/$JOB_ID/evaluation.json"
```

---

### API 返回验证

`GET /v1/jobs/{job_id}` 返回中应出现：
1. `result.cad_file`
2. `result.evaluation_report`（含可下载 URL）

---

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `llm_judge_evaluator_disabled` | `SANDBOX_MCP_LLM_JUDGE_ENABLED=false` | 设置为 `true` |
| `No such file or directory: ''` | `SANDBOX_MCP_SERVER_CWD` 未设置 | 设置为 `$(pwd)` |
| `.env` 解析 warning（含空格）| Shell 解析问题 | 优先 `source .env`；使用逗号格式 |

---

## 关键日志验真点（SubAgent 终端）

- 出现 `using_mcp_sandbox_runner` → 说明已切到 mcp 路径
- 出现 `sandbox_mcp_server_starting` → 说明确实拉起了 MCP server
- 出现 `sandbox_execution_starting / sandbox_execution_complete` → 说明最终在 Docker sandbox 执行
- 出现 `job_succeeded` → 说明完整链路成功

### 4. 终端 C：发起请求并确认建模

#### 请求 1（应返回 `confirmation_required=true`）

```bash
curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[{"role":"user","content":"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔"}],
    "stream":false
  }' | jq .
```

---

#### 请求 2（确认创建任务）

```bash
RESP=$(curl -sS -X POST http://localhost:8100/v1/chat/completions \
  -H "Authorization: Bearer dev_token_1" \
  -H "Content-Type: application/json" \
  -d '{
    "messages":[
      {"role":"user","content":"请生成一个长度60、宽40、厚度5的矩形安装板，四角各一个直径6通孔"},
      {"role":"assistant","content":"我将创建一个矩形安装板。\\n\\n```design-summary\\nrectangular mounting plate, length 60, width 40, thickness 5, four corner holes diameter 6\\n```"}
    ],
    "stream":false,
    "confirm_job_creation":true
  }')
echo "$RESP" | jq .
JOB_ID=$(echo "$RESP" | jq -r '.job_id')
echo "JOB_ID=$JOB_ID"
```

---

### 5. 轮询任务直到完成

```bash
while true; do
  STATUS_JSON=$(curl -sS "http://localhost:8100/v1/jobs/$JOB_ID" \
    -H "Authorization: Bearer dev_token_1")
  echo "$STATUS_JSON" | jq '{job_id,status,result}'
  STATUS=$(echo "$STATUS_JSON" | jq -r '.status')
  if [ "$STATUS" = "succeeded" ] || [ "$STATUS" = "failed" ]; then
    break
  fi
  sleep 2
done
```

## 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `llm_judge_evaluator_disabled` | `SANDBOX_MCP_LLM_JUDGE_ENABLED=false` | 设置为 `true` |
| `No such file or directory: ''` | `SANDBOX_MCP_SERVER_CWD` 未设置 | 设置为 `$(pwd)` |
| `.env` 解析 warning（含空格）| Shell 解析问题 | 优先 `source .env`；使用逗号格式 |
