ACI 实现细节

更新日期：2026-03-12 来源：docs/work_logs/WORK_LOG_2026_03_05_CAD_ACTION_INTERFACE.md

实施概述

CAD Action Interface (ACI) 从 Phase 1 的基础实现到 Phase 2 的完整实现，ACI 评分从 4/10 提升到 8/10（100% 提升）。

实现阶段

Phase 1: 基础实现

核心文件：

  • services/sandbox/images/cadquery/models.py - Pydantic 模型
  • services/sandbox/images/cadquery/cad_session.py - CAD 会话类
  • services/sandbox/images/cadquery/action_entrypoint.py - 动作执行入口

动作集： 15 个动作（4 草图 + 4 实体 + 5 特征 + 2 状态）

Phase 2: ACI 增强实现

新增能力：

  1. 会话管理
  2. 智能建议生成
  3. 完整性诊断
  4. 动作修改（modify_action）
  5. 历史查询（get_history）

新增文件：

  • services/sub_agent/src/sub_agent/prompts/codegen_action.md - ACI 提示词
  • services/sub_agent/src/sub_agent/action_builder.py - 动作构建工具

修改文件：

  • services/sandbox/mcp_server/src/sandbox_mcp_server/contracts.py - 新增模型
  • services/sandbox/mcp_server/src/sandbox_mcp_server/service.py - ACI 服务层
  • services/sandbox/mcp_server/src/sandbox_mcp_server/server.py - MCP 服务器

前向引用解析

为解决循环导入问题，使用字符串注解：

class ActionHistoryEntry:
    snapshot: "CADStateSnapshot"  # 字符串注解避免循环导入

class CADActionOutput:
    action_history: list["ActionHistoryEntry"]  # 字符串注解
    completeness: "CompletenessInfo | None"

架构流程图 - ACI 模式

User Request
    ↓
Spec Agent (LLM)
    ↓
apply_cad_action(session_id="my_model")
    ↓
┌─────────────────────────────────┐
│ SandboxMCPService          │
│                           │
│ 1. Get/Create Session      │
│ 2. Execute Action          │
│ 3. Generate Snapshot         │
│ 4. Append to History       │
│ 5. Generate Suggestions     │
│ 6. Generate Completeness    │
└─────────────────────────────────┘
    ↓
CADActionOutput
├─ action_history: [ActionHistoryEntry]
│   ├─ step 1: create_sketch
│   ├─ step 2: add_rectangle
│   ├─ step 3: extrude
│   └─ step N: fillet
├─ suggestions: ["Consider adding fillet..."]
├─ completeness: {expected_steps, current_step, missing_features, can_continue, confidence}
└─ snapshot: {step, features, geometry, issues, images, success, error}
    ↓
Agent Observes
    ├─ Review action_history
    ├─ Check suggestions
    ├─ Verify completeness.can_continue
    └─ Decide: Continue / Modify / Complete
    ↓
Next Action (or modify_action/get_history/clear_session)
