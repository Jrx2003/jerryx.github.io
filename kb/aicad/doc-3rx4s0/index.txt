> 更新日期：2026-03-12
> 来源：`docs/work_logs/2026_03_04_MULTIMODAL_EVAL_FLOW.md`

## 目标

将 `sandbox_mcp_server` 的 llm 评估链路从"文本证据主导"扩展为"文本 + 多角度截图证据"，并完成一次可复现的端到端实跑。

---

## 新增机制

### 1. 截图获取机制

**Sandbox 执行入口**在导出 `model.step` 后，新增四个固定视角预览图导出：

| 文件名 | 视角 | 说明 |
|--------|------|------|
| `preview_iso.png` | 等轴视图 | 整体三维效果 |
| `preview_front.png` | 前视图 | 从 Y+ 方向观察 |
| `preview_right.png` | 右视图 | 从 X+ 方向观察 |
| `preview_top.png` | 顶视图 | 从 Z+ 方向观察 |

**技术实现：**
- 采用离屏渲染方式（`matplotlib` + `Agg`）避免 GUI/X11 依赖
- Docker 镜像新增 `matplotlib-base=3.8` 依赖
- Docker runner 回收产物从"仅 step"改为"step + previews"

---

### 2. 截图进入 LLM 评估机制

**数据流：**
```
Sandbox 产物 (model.step + 4张 preview png)
    ↓
EvidenceBuilder
    ↓
EvidenceBundle.preview_images
    ├─ filename
    ├─ mime_type (image/png)
    ├─ size_bytes
    ├─ sha256
    └─ content_base64
    ↓
LLM Message (多模态)
    ├─ Text Block (评估说明 + 元数据)
    └─ Image Blocks (4张截图)
    ↓
LLM Provider (OpenAI-compatible / Anthropic / Google)
    ↓
评估结果
```

**关键改动：**
1. `EvidenceBundle` 新增 `preview_images` 字段
2. `EvidenceBuilder` 从 sandbox 产物中收集图片，限流、并纳入 `replay_key`
3. `LLMMessage` 升级为支持多模态 content（text/image part）
4. Provider 适配（OpenAI-compatible / Anthropic / Google）支持图片消息映射

---

### 3. Provider 兼容处理

#### GLM-4.7 多模态问题

**问题：** `glm-4.7` 对多模态消息返回 `400 (code: 1210)`

**修复策略：**
1. 保留"优先多模态"
2. 若多模态调用被 provider 拒绝，自动回退到文本模式继续评估
3. `judge_attempt_count` 会体现重试次数

**效果：** 在 GLM 当前模型能力下，流程从失败恢复为 `evaluation_status=success`

---

#### Kimi 温度兼容问题

**问题：** `invalid temperature: only 1 is allowed for this model`

**修复：**
```python
# libs/llm/src/llm/providers/openai_compatible.py
if model.startswith('kimi-'):
    temperature = 1.0  # Kimi 要求固定温度
```

---

## 配置

**配置位置：** `libs/common/src/common/config.py`

| 配置项 | 默认值 | 说明 |
|--------|---------|------|
| `SANDBOX_MCP_LLM_JUDGE_MAX_PREVIEW_IMAGES` | `4` | 证据中最多携带多少张截图 |
| `SANDBOX_MCP_LLM_JUDGE_MAX_PREVIEW_IMAGE_BYTES` | `600000` | 单张截图体积上限 |

## 关键更改与思路

### 关键更改

1. **Sandbox 产图与回收**
   - `services/sandbox/images/cadquery/entrypoint.py`
   - `services/sandbox/images/cadquery/Dockerfile`
   - `libs/sandbox/src/sandbox/docker_runner.py`

2. **证据模型与构建**
   - `services/sandbox/mcp_server/src/sandbox_mcp_server/llm_judge_models.py`
   - `services/sandbox/mcp_server/src/sandbox_mcp_server/evidence_builder.py`

3. **LLM 多模态协议与适配**
   - `libs/llm/src/llm/interface.py`
   - `libs/llm/src/llm/providers/openai_compatible.py`
   - `libs/llm/src/llm/providers/anthropic.py`
   - `libs/llm/src/llm/providers/google.py`

4. **评估器健壮性（多模态失败回退）**
   - `services/sandbox/mcp_server/src/sandbox_mcp_server/llm_judge.py`

---

### 设计思路

1. 先保证"截图真实可产出、可回收、可落盘"
2. 再把截图注入评估证据与多模态消息结构
3. 最后解决 provider 模型能力差异（自动降级），确保生产可用性优先

### 关键更改

   - `libs/llm/src/llm/interface.py`

   - `services/sandbox/mcp_server/src/sandbox_mcp_server/llm_judge.py`

### 设计思路

1. 先保证"截图真实可产出、可回收、可落盘"
2. 再把截图注入评估证据与多模态消息结构
3. 最后解决 provider 模型能力差异（自动降级），确保生产可用性优先
