## K8s 架构

### 控制平面 (Control Plane)

```
┌─────────────────────────────────────┐
│          Control Plane              │
├─────────────────────────────────────┤
│  API Server  ←→  etcd (存储)        │
│       ↑                             │
│  Scheduler    Controller Manager    │
└─────────────────────────────────────┘
              ↓
┌─────────────────────────────────────┐
│           Worker Nodes              │
├─────────────────────────────────────┤
│  Kubelet → Container Runtime        │
│  Kube-Proxy (网络代理)               │
└─────────────────────────────────────┘
```

### 核心组件职责

| 组件 | 职责 |
|------|------|
| API Server | 所有操作的入口，REST API |
| etcd | 分布式键值存储，保存集群状态 |
| Scheduler | Pod 调度，选择最优节点 |
| Controller Manager | 维护集群期望状态 |
| Kubelet | 节点上的 Pod 生命周期管理 |
| Kube-Proxy | 服务发现和负载均衡 |

## 核心对象

### Pod

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
spec:
  containers:
  - name: nginx
    image: nginx:1.20
    ports:
    - containerPort: 80
```

### Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.20
```

### Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  type: ClusterIP
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
```

## 控制器模式

### 声明式 vs 命令式

```
命令式: "启动3个nginx容器"
声明式: "期望状态: 3个nginx Pod副本"

K8s 通过控制循环维护期望状态:
┌──────────┐     ┌──────────┐     ┌──────────┐
│  观察    │ →  │  差异    │ →  │  行动    │
│  当前状态 │    │  分析    │    │  调整    │
└──────────┘     └──────────┘     └──────────┘
       ↑                               │
       └───────────────────────────────┘
```

### 常用控制器

| 控制器 | 用途 |
|--------|------|
| Deployment | 无状态应用管理 |
| StatefulSet | 有状态应用管理 |
| DaemonSet | 每个节点一个Pod |
| Job/CronJob | 批处理任务 |

## 网络模型

### 网络设计原则

```
1. 每个 Pod 有独立 IP (IP-per-Pod)
2. 所有 Pod 可以互相通信 (无需 NAT)
3. 节点和 Pod 可以通信

Pod IP 特点:
- 集群内唯一
- 跨节点可达
- 生命周期与 Pod 绑定
```

### Service 网络

```
ClusterIP: 集群内部虚拟 IP
NodePort: 节点端口暴露 (30000-32767)
LoadBalancer: 云厂商负载均衡
Ingress: 七层路由
```

## 面试要点

**Q1: Pod 与容器的关系？**

A:
- Pod 是最小调度单元
- 一个 Pod 可包含多个容器
- 共享网络命名空间和存储卷

**Q2: K8s 如何保证高可用？**

A:
- 多 Master 节点
- etcd 集群部署
- Pod 多副本
- 故障自动转移

**Q3: Deployment 的更新策略？**

A:
- RollingUpdate (滚动更新，默认)
- Recreate (先删后建)
- maxSurge/maxUnavailable 控制更新节奏
