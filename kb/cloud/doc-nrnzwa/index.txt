## 什么是云计算

云计算（Cloud Computing）是一种通过互联网按需提供计算资源（服务器、存储、数据库、网络、软件等）的服务模式。用户无需购买和维护物理硬件，只需按需使用、按量付费。

### 核心定义

NIST（美国国家标准与技术研究院）定义：
> 云计算是一种模型，它可以实现随时随地、便捷地、随需应变地从可配置计算资源共享池中获取所需的资源（例如，网络、服务器、存储、应用、及服务），资源能够快速供应并释放，使管理资源的工作量和与服务提供商的交互减小到最低限度。

---

## 云计算的核心特征

### 五大基本特征

| 特征 | 英文 | 说明 | 示例 |
|------|------|------|------|
| 按需自助服务 | On-demand Self-service | 用户可自动获取计算能力，无需人工干预 | 在 AWS 控制台启动 EC2 实例 |
| 广泛的网络访问 | Broad Network Access | 通过网络提供服务，支持各种客户端 | 手机、电脑、平板均可访问 |
| 资源池化 | Resource Pooling | 多租户共享资源，动态分配 | 多个用户共享物理服务器 |
| 快速弹性 | Rapid Elasticity | 资源可快速扩展或收缩 | 双 11 期间自动扩容 |
| 可计量服务 | Measured Service | 资源使用可被监控、控制和报告 | 按 CPU 使用时间计费 |

## 云计算的优势

### 对企业的好处

| 优势 | 传统 IT | 云计算 | 节省/提升 |
|------|---------|--------|-----------|
| 初始投入 | 高（购买硬件） | 低（按需付费） | 节省 70%+ |
| 部署速度 | 周/月级 | 分钟级 | 提升 90%+ |
| 扩展性 | 受限于硬件 | 无限扩展 | 灵活度 100% |
| 维护成本 | 需要专业团队 | 服务商负责 | 节省 50%+ |
| 可靠性 | 依赖单点 | 多可用区冗余 | 可用性 99.99%+ |
| 创新能力 | 受限于资源 | 快速试错 | 提升显著 |

### 对开发者的价值

1. **快速启动项目**：几分钟内创建完整开发环境
2. **专注业务逻辑**：无需关心底层基础设施
3. **弹性应对流量**：自动扩缩容应对业务高峰
4. **全球部署**：一键部署到全球多个区域
5. **丰富服务生态**：数据库、缓存、消息队列等即开即用

## 云计算的架构层次

### 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Applications)                   │
│         SaaS: 钉钉、Salesforce、Office 365                  │
├─────────────────────────────────────────────────────────────┤
│                      平台层 (Platform)                       │
│         PaaS: Heroku、Google App Engine、阿里云 EDAS        │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure)                │
│         IaaS: AWS EC2、Azure VM、阿里云 ECS                 │
├─────────────────────────────────────────────────────────────┤
│                     虚拟化层 (Virtualization)                │
│         VMware、KVM、Xen、Hyper-V                           │
├─────────────────────────────────────────────────────────────┤
│                      物理层 (Physical)                       │
│         服务器、存储、网络设备                               │
└─────────────────────────────────────────────────────────────┘
```

| 年份 | 事件 | 意义 |
|------|------|------|
| 1963 | J.C.R. Licklider 提出"星际计算机网络" | 云计算概念萌芽 |
| 1999 | Salesforce 推出 SaaS 服务 | SaaS 模式诞生 |
| 2006 | AWS 推出 S3 和 EC2 | 现代云计算开端 |
| 2010 | OpenStack 项目启动 | 开源云平台兴起 |
| 2013 | Docker 发布 | 容器化革命 |
| 2014 | Kubernetes 开源 | 容器编排标准 |
| 2015 | AWS Lambda 发布 | Serverless 兴起 |
| 2019 | 云原生概念普及 | 云原生时代到来 |
| 2023 | ChatGPT 引发 AI 热潮 | AI 基础设施需求爆发 |
| 2024-2025 | MaaS/FinOps/机密计算成熟 | 云计算新阶段 |

## 云计算的优势

### 对企业的好处

| 优势 | 传统 IT | 云计算 | 节省/提升 |
|------|---------|--------|-----------|
| 初始投入 | 高（购买硬件） | 低（按需付费） | 节省 70%+ |
| 部署速度 | 周/月级 | 分钟级 | 提升 90%+ |
| 扩展性 | 受限于硬件 | 无限扩展 | 灵活度 100% |
| 维护成本 | 需要专业团队 | 服务商负责 | 节省 50%+ |
| 可靠性 | 依赖单点 | 多可用区冗余 | 可用性 99.99%+ |
| 创新能力 | 受限于资源 | 快速试错 | 提升显著 |

### 对开发者的价值

1. **快速启动项目**：几分钟内创建完整开发环境
2. **专注业务逻辑**：无需关心底层基础设施
3. **弹性应对流量**：自动扩缩容应对业务高峰
4. **全球部署**：一键部署到全球多个区域
5. **丰富服务生态**：数据库、缓存、消息队列等即开即用

## 云计算的架构层次

### 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      应用层 (Applications)                   │
│         SaaS: 钉钉、Salesforce、Office 365                  │
├─────────────────────────────────────────────────────────────┤
│                      平台层 (Platform)                       │
│         PaaS: Heroku、Google App Engine、阿里云 EDAS        │
├─────────────────────────────────────────────────────────────┤
│                    基础设施层 (Infrastructure)                │
│         IaaS: AWS EC2、Azure VM、阿里云 ECS                 │
├─────────────────────────────────────────────────────────────┤
│                     虚拟化层 (Virtualization)                │
│         VMware、KVM、Xen、Hyper-V                           │
├─────────────────────────────────────────────────────────────┤
│                      物理层 (Physical)                       │
│         服务器、存储、网络设备                               │
└─────────────────────────────────────────────────────────────┘
```

## 云计算的关键技术

### 核心技术栈

| 技术领域 | 关键技术 | 作用 |
|----------|----------|------|
| 虚拟化 | KVM、VMware、Xen | 硬件资源抽象 |
| 容器化 | Docker、containerd | 应用级隔离 |
| 编排调度 | Kubernetes、Mesos | 容器管理 |
| 存储 | Ceph、GlusterFS | 分布式存储 |
| 网络 | SDN、VXLAN、Calico | 虚拟网络 |
| 自动化 | Terraform、Ansible | 基础设施即代码 |
