服务模式详解

服务模式详解

更新日期：2026-03-13 | 版本：v1.0 | 难度：L1（初级）

概述

云计算的服务模式定义了云服务提供商与用户之间的责任边界。根据 NIST 定义，云计算有三种主要服务模式：

  • IaaS（基础设施即服务）
  • PaaS（平台即服务）
  • SaaS（软件即服务）

此外，随着技术发展，还衍生出 CaaS、FaaS、BaaS 等新模式。

IaaS（基础设施即服务）

定义

IaaS（Infrastructure as a Service）提供虚拟化的计算资源，包括服务器、存储、网络等基础设施。用户可以在这些资源上部署和运行任意软件。

核心特点

特点说明
最高灵活性完全控制底层基础设施
按需付费按实际使用量计费
快速部署分钟级资源交付
弹性伸缩根据负载自动调整资源

典型产品

厂商产品名称核心服务
AWSAmazon EC2虚拟机、存储、网络
AzureAzure Virtual Machines虚拟机、托管磁盘
GCPCompute Engine虚拟机、GPU 实例
阿里云ECS云服务器、块存储
华为云ECS弹性云服务器

使用场景

场景说明示例
应用托管部署 Web 应用电商网站、企业官网
高性能计算科学计算、渲染基因测序、视频渲染
灾备恢复异地备份业务连续性保障
开发测试临时环境自动化测试环境

代码示例：使用 AWS CLI 创建 EC2

# 创建 EC2 实例
aws ec2 run-instances \
    --image-id ami-0c55b159cbfafe1f0 \
    --instance-type t2.micro \
    --key-name MyKeyPair \
    --security-group-ids sg-903004f8 \
    --subnet-id subnet-6e7f829e \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=MyInstance}]'

# 查看实例状态
aws ec2 describe-instances --instance-ids i-1234567890abcdef0

PaaS（平台即服务）

定义

PaaS（Platform as a Service）提供应用开发、运行和管理的完整平台。用户无需管理底层基础设施，专注于应用开发。

核心特点

特点说明
开发友好提供开发工具、框架支持
自动运维自动处理部署、扩展、补丁
多语言支持Java、Python、Node.js 等
集成服务内置数据库、缓存、消息队列

典型产品

厂商产品名称特点
GoogleApp Engine完全托管、自动扩缩
HerokuHeroku Platform开发者友好、Git 部署
AWSElastic Beanstalk与 AWS 服务深度集成
阿里云EDAS企业级分布式应用服务
腾讯云TKE Serverless容器化 PaaS

使用场景

场景说明示例
Web 应用快速部署网站企业官网、博客
API 服务后端服务托管RESTful API
微服务微服务部署Spring Cloud 应用
移动后端移动应用后端BaaS 服务

代码示例：部署到 Heroku

# 安装 Heroku CLI
brew tap heroku/brew && brew install heroku

# 登录
heroku login

# 创建应用
heroku create my-app-name

# 部署代码
git push heroku main

# 查看日志
heroku logs --tail

SaaS（软件即服务）

定义

SaaS（Software as a Service）提供完整的软件应用，用户通过浏览器或客户端直接使用，无需安装和维护。

核心特点

特点说明
即用即付订阅制付费
免维护服务商负责更新维护
跨平台支持 Web、移动端
协作能力支持多人协作

典型产品

类别代表产品功能
办公协作Office 365、钉钉、飞书文档、沟通、协作
CRMSalesforce、HubSpot客户关系管理
ERPSAP S/4HANA Cloud、金蝶云企业资源计划
存储Dropbox、百度网盘文件存储同步
视频会议Zoom、腾讯会议在线会议

使用场景

场景说明优势
企业办公日常办公工具降低 IT 成本
销售管理CRM 系统提升销售效率
团队协作项目管理远程协作
客户服务客服系统提升响应速度

新兴服务模式

CaaS（容器即服务）

属性说明
定义提供容器编排和管理平台
代表产品EKS、ACK、GKE
优势比 IaaS 更易用，比 PaaS 更灵活
适用容器化应用、微服务架构

FaaS（函数即服务）

属性说明
定义事件驱动的函数计算平台
代表产品AWS Lambda、函数计算
优势极致弹性、按调用付费
适用事件处理、API 后端、定时任务

BaaS（后端即服务）

属性说明
定义为前端应用提供后端能力
代表产品Firebase、LeanCloud
优势前端开发者无需后端知识
适用移动应用、单页应用

面试常见问题

Q1: IaaS 和 PaaS 的主要区别是什么？

答：主要区别在于管理责任边界。IaaS 提供基础设施（服务器、存储、网络），用户需要管理操作系统、运行时和应用；PaaS 提供完整的应用运行平台，用户只需关注应用代码，平台负责底层运维。

Q2: 什么场景适合使用 SaaS？

答：当企业需要标准化功能、希望降低 IT 成本、快速上线时适合使用 SaaS。例如：办公协作（钉钉）、CRM（Salesforce）、财务系统（用友云）等。

Q3: FaaS 和传统 PaaS 有什么区别？

答：FaaS 是更细粒度的计算抽象，以函数为单位运行，按调用次数计费，极致弹性；传统 PaaS 以应用为单位，通常有最小运行实例，按资源使用计费。
