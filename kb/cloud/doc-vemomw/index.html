边缘计算概述

什么是边缘计算

计算位置演进:

中心化: 所有计算在云端数据中心
    ↓
CDN: 内容分发到边缘节点
    ↓
边缘计算: 计算能力下沉到边缘

定义:
在靠近数据源的网络边缘侧，提供计算、存储、应用能力

为什么需要边缘计算

传统云计算痛点:
┌──────────┐
│ 设备端   │ ← 延迟高 (100ms+)
│ (数据源) │ ← 带宽成本大
└────┬─────┘
     │ 网络传输
     ↓
┌──────────┐
│ 云端     │ ← 隐私风险
│ (处理)   │ ← 单点故障
└──────────┘

边缘计算解决:
- 降低延迟 (10ms以内)
- 减少带宽
- 保护隐私
- 提高可靠性

架构模式

三层架构

┌─────────────────────────────────────┐
│            云端 (Cloud)              │
│   - 全局管理                         │
│   - 大数据存储                       │
│   - AI训练                          │
└──────────────┬──────────────────────┘
               │ 管理/同步
┌──────────────▼──────────────────────┐
│           边缘 (Edge)                │
│   - 边缘数据中心                     │
│   - 5G MEC                          │
│   - 区域处理                        │
└──────────────┬──────────────────────┘
               │ 低延迟通信
┌──────────────▼──────────────────────┐
│          设备端 (Device)             │
│   - IoT传感器                        │
│   - 智能终端                         │
│   - 实时响应                        │
└─────────────────────────────────────┘

边缘部署形态

形态位置距离延迟
设备边缘设备内部0跳<1ms
网关边缘现场网关1跳1-10ms
MEC基站/接入点2-5跳10-20ms
区域边缘城市数据中心多跳20-50ms

- 资源受限 (CPU/内存/存储)

解决方案:
- KubeEdge: 华为开源

分布式云

概念

分布式云 = 云计算 + 边缘计算

特点:
- 统一管理和调度
- 一致的服务体验
- 灵活的工作负载分布

代表产品:
- AWS Outposts / Wavelength / Local Zones
- Azure Stack / Arc
- Google Anthos
- 阿里云边缘容器

工作负载调度

调度策略:

延迟敏感:
  推理请求 → 边缘节点

计算密集:
  模型训练 → 云端数据中心

数据合规:
  敏感数据处理 → 本地边缘

成本优化:
  批处理任务 → 云端 (更便宜)

面试要点

Q1: 边缘计算 vs CDN 的区别？

A:

  • CDN: 内容分发，只缓存静态内容
  • 边缘计算: 计算能力，可运行动态逻辑

Q2: 边缘计算的挑战？

A:

  • 资源受限
  • 网络不稳定
  • 运维困难 (地理位置分散)
  • 安全问题 (物理接触风险)

Q3: 什么场景适合边缘计算？

A:

  • 自动驾驶 (低延迟要求)
  • 工业质检 (带宽节省)
  • AR/VR (实时渲染)
  • 智能监控 (隐私保护)
