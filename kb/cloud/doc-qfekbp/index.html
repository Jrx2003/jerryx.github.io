云计算核心概念

云计算核心概念

更新日期：2026-03-13 相关文档：虚拟化技术详解 | 网络架构与VPC | 计算服务概览

一、云计算定义

云计算是一种通过互联网提供计算服务的模式，包括服务器、存储、数据库、网络、软件、分析等。用户无需拥有和维护物理基础设施，按需使用、按量付费。

NIST 五大特征

特征说明举例
按需自助用户可自动获取计算资源AWS EC2 启动实例
广泛网络访问通过网络从任何设备访问从 PC、手机访问云服务
资源池化供应商统一管理，动态分配多租户共享物理资源
快速弹性根据负载自动扩展/收缩Auto Scaling 自动增减实例
按量计费用多少付多少按小时、按请求数计费

二、服务模式

IaaS - 基础设施即服务

提供虚拟化的计算资源，用户负责操作系统和应用。

特点优势典型产品
提供虚拟机、存储、网络灵活性最高AWS EC2、阿里云 ECS、Azure VM
用户管理 OS 和应用控制力强-
按需分配资源成本可控-

PaaS - 平台即服务

提供开发运行环境，用户只需关注代码。

特点优势典型产品
提供开发平台、数据库开发效率高Heroku、Google App Engine
隐藏基础设施细节运维简化阿里云 SAE、腾讯云 SCF
自动扩缩容弹性好-

SaaS - 软件即服务

提供完整应用，用户直接使用。

特点优势典型产品
提供完整软件零部署Office 365、钉钉
自动更新最新功能Salesforce、Gmail
多租户共享成本分摊-

三、部署模式

公有云 (Public Cloud)

第三方提供商运营，共享基础设施。

优势劣势适用场景
无需前期投入数据安全顾虑互联网应用、初创企业
弹性扩展快合规性问题波动性业务
专业技术支持性能可能受影响非核心数据业务

私有云 (Private Cloud)

企业自建或独享基础设施。

优势劣势适用场景
数据完全控制成本高金融、政府
合规性好维护复杂对安全敏感业务
性能可控扩展受限核心系统

混合云 (Hybrid Cloud)

公有云 + 私有云结合。

优势劣势适用场景
灵活选择部署架构复杂大型企业
灾备方案数据同步困难需要合规的互联网公司
成本优化运维挑战波动性 + 敏感业务

多云 (Multi Cloud)

使用多个公有云服务商。

优势劣势适用场景
避免厂商锁定管理复杂大型企业
优化成本一致性问题全球业务
区域优化学习曲线陡峭风险分散

五、面试要点

高频问题

  1. 云计算 vs 本地部署的区别？

    • 弹性：云可以自动扩缩容
    • 成本：按量付费 vs 固定投入
    • 维护：供应商负责 vs 自维护

  2. 如何选择公有云 vs 私有云？

    • 数据敏感性：敏感数据选私有云
    • 合规要求：金融等强监管选私有云
    • 成本预算：初创公司选公有云
    • 技术能力：没有运维团队选公有云

  3. 容器 vs 虚拟机？

    • 虚拟机：完整 OS，隔离性强，启动慢
    • 容器：共享内核，启动快，占用少

  4. CAP 定理在云计算中的应用？

    • Consistency（一致性）：所有节点同时看到相同数据
    • Availability（可用性）：系统始终可用
    • Partition tolerance（分区容错）：网络分区时仍能工作
    • 云系统通常选择 AP 或 CP
