## 基础概念题

### Q1: IaaS/PaaS/SaaS 的区别？

**答案:**

```
IaaS (基础设施即服务):
- 提供: 计算、存储、网络
- 管理: OS以上用户管理
- 代表: AWS EC2, 阿里云ECS

PaaS (平台即服务):
- 提供: 开发运行平台
- 管理: 应用以上用户管理
- 代表: Heroku, Google App Engine

SaaS (软件即服务):
- 提供: 完整软件应用
- 管理: 全部由厂商管理
- 代表: Salesforce, Office 365

类比:
IaaS = 租地自己建房
PaaS = 租房装修
SaaS = 酒店拎包入住
```

### Q2: 虚拟化和容器的区别？

**答案:**

| 维度 | 虚拟机 | 容器 |
|------|--------|------|
| 隔离级别 | OS级 | 进程级 |
| 启动时间 | 分钟级 | 秒级 |
| 资源占用 | 大 (GB) | 小 (MB) |
| 性能 | 有虚拟化损耗 | 接近原生 |
| 镜像大小 | 大 (GB) | 小 (MB) |

```
架构对比:

VM: App + Libs + OS → 虚拟硬件 → Host OS → 物理机
Container: App + Libs → 容器运行时 → Host OS → 物理机

容器共享主机内核，VM 有独立内核
```

### Q3: 公有云/私有云/混合云/多云的区别？

**答案:**

```
公有云: 第三方提供，多租户共享
私有云: 自建自用，单租户
混合云: 公有云 + 私有云，统一管理
多云: 多个公有云厂商

选择因素:
- 数据合规 → 私有云
- 弹性需求 → 公有云
- 既有资产 + 新业务 → 混合云
- 避免锁定 → 多云
```

## 架构设计题

### Q4: 如何设计一个高可用的云上架构？

**答案:**

```
高可用设计原则:

1. 多可用区部署
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │ 可用区A │ │ 可用区B │ │ 可用区C │
   │ 主      │ │ 备      │ │ 备      │
   └─────────┘ └─────────┘ └─────────┘

2. 无状态设计
   - 会话外置 (Redis)
   - 本地不存储数据

3. 自动故障转移
   - 健康检查
   - 自动切换

4. 限流降级
   - 熔断器
   - 限流策略
```

### Q5: 云上如何实现弹性伸缩？

**答案:**

```
伸缩类型:

水平伸缩 (Scale Out):
- 增加/减少实例数量
- 适合无状态应用
- 通过 SLB 分发流量

垂直伸缩 (Scale Up):
- 升级/降级实例规格
- 适合有状态应用
- 需要重启

自动伸缩策略:
- 基于 CPU/内存利用率
- 基于请求队列长度
- 基于定时规则
- 基于自定义指标
```

## 运维实践题

### Q6: 如何监控云上资源？

**答案:**

```
监控体系:

指标监控 (Metrics):
- 基础设施: CPU、内存、磁盘、网络
- 应用指标: QPS、延迟、错误率
- 业务指标: 订单量、用户数

日志监控 (Logging):
- 集中收集 (ELK/Loki)
- 结构化日志
- 日志告警

链路追踪 (Tracing):
- 分布式追踪 (Jaeger/SkyWalking)
- 性能瓶颈定位

告警策略:
- 多级告警 (P0/P1/P2)
- 告警收敛
- 值班轮询
```

### Q7: 云上数据备份策略？

**答案:**

```
备份 3-2-1 原则:
- 3份数据副本
- 2种不同介质
- 1份异地备份

云厂商方案:
- 自动快照
- 跨区域复制
- 版本控制
- 定期测试恢复

数据库:
- 主从复制
- 定期全量备份
- 增量备份
- Binlog 归档
```

## 安全合规题

### Q8: 云安全责任共担模型是什么？

**答案:**

```
责任划分:

云厂商负责 (云的安全):
- 物理设施
- 网络基础设施
- 虚拟化层
- 托管服务底层

用户负责 (云中的安全):
- 操作系统
- 应用和数据
- 访问控制
- 加密配置

共同责任:
- 合规认证
- 安全更新
```

### Q9: 如何保障云上数据安全？

**答案:**

```
数据安全层次:

传输安全:
- TLS 1.3 加密
- VPN/专线
- 证书管理

存储安全:
- 服务端加密 (SSE)
- 客户端加密 (CSE)
- 密钥管理 (KMS)

访问控制:
- IAM 最小权限
- 多因素认证 (MFA)
- 临时凭证

审计:
- 操作日志
- 访问记录
- 异常检测
```

监控告警:
- 全链路监控

- 预案演练

### Q11: 企业上云的步骤和注意事项？

**答案:**

```
上云步骤:

1. 评估阶段

   - 依赖分析
   - 成本评估

2. 规划阶段
   - 架构设计
   - 网络规划
   - 安全策略

3. 迁移阶段
   - 试点迁移
   - 批量迁移
   - 数据同步

4. 优化阶段
   - 成本优化
   - 性能调优
   - 架构升级

注意事项:
- 合规要求 (数据主权)
- 网络带宽规划
- 回滚方案
- 团队培训
```
