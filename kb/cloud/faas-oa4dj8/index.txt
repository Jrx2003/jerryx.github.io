## FaaS 概述

### Function as a Service

```
FaaS 是 Serverless 的核心实现

核心思想:
- 以函数为最小部署单元
- 事件驱动执行
- 用完即走，自动回收

对比:
┌───────────┬────────────┬────────────┐
│ 维度      │ 传统服务   │ FaaS       │
├───────────┼────────────┼────────────┤
│ 部署单元  │ 应用/容器  │ 函数       │
│ 运行时间  │ 7x24小时   │ 按需       │
│ 计费方式  │ 预留实例   │ 调用次数   │
│ 扩缩容    │ 手动/自动  │ 自动       │
└───────────┴────────────┴────────────┘
```

### 主流平台

| 平台 | 提供商 | 特点 |
|------|--------|------|
| AWS Lambda | AWS | 最早最成熟 |
| Azure Functions | Microsoft | 与Azure生态集成 |
| Cloud Functions | Google | 与GCP深度集成 |
| 函数计算 | 阿里云 | 国内领先 |
| SCF | 腾讯云 | 微信生态集成 |

## 核心概念

### 函数定义

```python
# AWS Lambda 示例
import json

def lambda_handler(event, context):
    """
    event: 触发事件数据
    context: 运行时上下文
    """
    # 处理逻辑
    result = process(event)

    return {
        'statusCode': 200,
        'body': json.dumps(result)
    }
```

### 运行环境

```
执行环境组成:
- 内存: 128MB - 10GB
- CPU: 与内存成正比
- 临时磁盘: 512MB - 10GB
- 超时: 最多15分钟

运行时支持:
- Node.js
- Python
- Java
- Go
- .NET
- 自定义运行时
```

## 函数生命周期

### 生命周期阶段

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│ 创建    │ → │ 执行    │ → │ 销毁    │
└─────────┘    └─────────┘    └─────────┘
     ↓              ↓              ↓
 上传代码      冷启动/热启动    超时回收
 配置触发器    处理请求
```

### 实例复用

```python
# 全局初始化 (冷启动时执行一次)
import boto3
client = boto3.client('s3')  # 可复用

def handler(event, context):
    # 每次调用执行
    response = client.get_object(...)
    return response
```

## 触发器机制

### 触发器类型

```
事件源触发:
┌──────────────┬─────────────────────┐
│ 触发器       │ 场景                │
├──────────────┼─────────────────────┤
│ API Gateway  │ HTTP/REST API       │
│ S3 事件      │ 文件上传处理        │
│ 定时触发     │ Cron 任务           │
│ 消息队列     │ 异步消息处理        │
│ 数据库变更   │ CDC 数据同步        │
│ IoT 规则     │ 设备数据处理        │
└──────────────┴─────────────────────┘
```

### 同步 vs 异步

```
同步调用:
调用方 ←────等待返回────→ 函数

异步调用:
调用方 ──→ 事件队列 ──→ 函数
  └──────────────┘
    立即返回
```

## 面试要点

**Q1: FaaS 的计费模式？**

A:
- 请求次数费用
- 计算时间费用 (GB-秒)
- 免费额度通常每月100万次请求

**Q2: 函数计算的性能优化？**

A:
- 内存越大CPU越多
- 精简依赖包
- 使用单实例多并发
- 连接池复用
- 预留实例避免冷启动

**Q3: FaaS 的存储选项？**

A:
- 临时存储 (/tmp): 实例生命周期内有效
- 对象存储 (S3/OSS): 持久化大文件
- 数据库: 结构化数据
- 缓存 (Redis): 会话状态
