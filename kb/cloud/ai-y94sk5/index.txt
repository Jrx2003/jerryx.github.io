## AI 基础设施概述

### AI 基础设施栈

```
┌─────────────────────────────────────┐
│         AI 应用层                    │
│   对话系统、代码生成、文生图         │
├─────────────────────────────────────┤
│         AI 模型层                    │
│   LLM、多模态、CV、NLP              │
├─────────────────────────────────────┤
│         AI 平台层                    │
│   训练框架、推理引擎、MLOps         │
├─────────────────────────────────────┤
│         基础设施层                   │
│   GPU集群、高速网络、存储系统       │
├─────────────────────────────────────┤
│         云原生层                     │
│   K8s、容器、调度、监控             │
└─────────────────────────────────────┘
```

### 核心组件

| 组件 | 功能 | 代表产品 |
|------|------|----------|
| 训练框架 | 分布式训练 | PyTorch, DeepSpeed, Megatron |
| 推理引擎 | 高性能推理 | vLLM, TensorRT-LLM, TGI |
| 向量数据库 | RAG 存储 | Milvus, Pinecone, Weaviate |
| 模型仓库 | 版本管理 | MLflow, Model Registry |

## MaaS 模式

### Model as a Service

```
MaaS 定义:
模型即服务，将 AI 能力通过 API 提供

商业模式:
┌──────────────┬──────────────────────┐
│ 模式         │ 说明                 │
├──────────────┼──────────────────────┤
│ 公有 API     │ OpenAI API 模式      │
│ 私有部署     │ 企业本地化部署       │
│ 混合模式     │ 基础模型公有 + 微调私有│
└──────────────┴──────────────────────┘
```

### MaaS 架构

```
┌─────────────────────────────────────┐
│           API Gateway               │
│    认证、限流、路由、计费           │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│           模型服务层                 │
│  ┌─────────┐ ┌─────────┐           │
│  │ LLM服务 │ │ 嵌入服务 │           │
│  │ 多副本  │ │ 多副本  │           │
│  └─────────┘ └─────────┘           │
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│           基础设施层                 │
│    GPU集群、自动扩缩、负载均衡       │
└─────────────────────────────────────┘
```

## AI 平台架构

### 大模型训练平台

```python
# 分布式训练配置 (DeepSpeed)
{
    "fp16": {
        "enabled": true
    },
    "zero_optimization": {
        "stage": 3,  # ZeRO-3 内存优化
        "offload_optimizer": {
            "device": "cpu"  # 优化器状态卸载到CPU
        }
    },
    "train_batch_size": "auto",
    "train_micro_batch_size_per_gpu": 4
}
```

### 推理优化平台

```
推理优化技术栈:

┌─────────────────────────────────────┐
│         服务层                       │
│   vLLM / TGI / TensorRT-LLM        │
├─────────────────────────────────────┤
│         优化层                       │
│   - 量化 (INT8/INT4/FP8)           │
│   - 剪枝                            │
│   - 蒸馏                            │
├─────────────────────────────────────┤
│         运行时                       │
│   - 批处理 (Continuous Batching)   │
│   - 投机解码                        │
│   - KV Cache 优化                   │
└─────────────────────────────────────┘
```

## 训推一体

### 概念

```
训推一体: 统一平台支持训练和推理

优势:
- 资源共享 (GPU 利用率提升)
- 模型快速迭代
- 统一调度管理
```

### 架构设计

```
┌─────────────────────────────────────────┐
│            统一调度层                    │
│       (Volcano / Yunikorn)              │
└──────────────┬──────────────────────────┘
               │
    ┌──────────┴──────────┐
    ↓                     ↓
┌──────────┐        ┌──────────┐
│ 训练集群 │        │ 推理集群 │
│ - 大显存 │        │ - 高吞吐 │
│ - NVLink │        │ - 多实例 │
│ - RDMA   │        │ - 低延迟 │
└──────────┘        └──────────┘
    │                     │
    └──────────┬──────────┘
               ↓
        ┌──────────────┐
        │  共享存储    │
        │  (模型仓库)  │
        └──────────────┘
```

## 面试要点

**Q1: MaaS 的核心挑战？**

A:
- 高并发低延迟
- 成本控制 (GPU 昂贵)
- 模型版本管理
- 多租户隔离

**Q2: 大模型推理优化手段？**

A:
- 量化压缩
- KV Cache 管理
- Continuous Batching
- 投机解码
- 模型并行

**Q3: AI 基础设施的关键指标？**

A:
- GPU 利用率
- 训练吞吐量 (samples/sec)
- 推理延迟 (TTFT, TPOT)
- 模型部署时间
- 资源成本
