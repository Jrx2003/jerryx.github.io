一、存储服务概览

存储类型对比

类型用途典型产品访问方式
对象存储非结构化数据S3、OSSAPI / SDK
块存储文件系统、数据库EBS、云盘挂载到实例
文件存储文件共享EFS、NASNFS / SMB
归档存储长期备份Glacier、Archive冷存取

二、对象存储 (Object Storage)

对象存储特点

特性说明
扁平结构没有目录层级
元数据丰富自定义标签、权限
无限扩展自动扩容
高可用多副本、跨区域

S3 核心概念

Bucket (桶)
    │
    ├─── Objects (对象)
    │    │
    │    ├── file1.jpg
    │    ├── file2.pdf
    │    └── data/
    │         └── nested.json
    │
    ├─── Versioning (版本控制)
    └─── Lifecycle (生命周期)

S3 存储类别

类别用途获取时间成本
Standard频繁访问毫秒级标准
Intelligent-Tiering访问模式未知毫秒级自动优化
Standard-IA不频繁访问毫秒级较低
One Zone-IA单区域不频繁毫秒级更低
Glacier长期归档分钟-小时最低
Deep Archive极长期归档小时最低

S3 生命周期策略

{
  "Rules": [
    {
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ],
      "Expiration": {
        "Days": 365
      }
    }
  ]
}

三、块存储 (Block Storage)

块存储特点

特性说明
块级访问固定大小的块
低延迟本地挂载
可调整在线扩容

EBS 卷类型对比

类型性能用途成本
gp3 (通用 SSD)3000-16000 IOPS通用工作负载标准
gp2 (旧版通用 SSD)根据容量变化通用较高
io1/io2 (Provisioned IOPS)最高 64000 IOPS关键数据库高
st1 (吞吐优化 HDD)高吞吐大数据流低
sc1 (冷 HDD)最低成本备份最低

EBS 性能配置

# 创建 gp3 卷，指定 IOPS 和吞吐量
aws ec2 create-volume \
    --volume-type gp3 \
    --size 100 \
    --iops 3000 \
    --throughput 125

EBS 快照

特性说明
增量备份只保存变化的数据
跨区域复制到其他区域
生命周期自动删除旧快照

七、面试要点

高频问题

  1. 对象存储 vs 块存储？

    • 对象存储：非结构化数据，API 访问
    • 块存储：文件系统、数据库，挂载访问

  2. EBS gp3 vs gp2？

    • gp3：独立配置 IOPS 和吞吐量，成本更低
    • gp2：IOPS 随容量增加，成本较高

  3. 何时使用 Glacier？

    • 长期备份
    • 合规要求
    • 数据极少访问

  4. EFS vs EBS？

    • EFS：多实例共享，自动扩容
    • EBS：单实例，性能可调

  5. 如何选择存储类别？

    • 频繁访问：Standard
    • 不频繁：Standard-IA
    • 归档：Glacier

八、最佳实践

  1. 使用生命周期策略：自动归档和删除
  2. 启用版本控制：防止误删除
  3. 加密数据：传输中 + 静态加密
  4. 监控存储成本：定期审计使用情况
  5. 使用多可用区：提高可用性
