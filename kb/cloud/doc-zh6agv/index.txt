## 一、存储服务概览

### 存储类型对比

| 类型 | 用途 | 典型产品 | 访问方式 |
|------|------|----------|----------|
| **对象存储** | 非结构化数据 | S3、OSS | API / SDK |
| **块存储** | 文件系统、数据库 | EBS、云盘 | 挂载到实例 |
| **文件存储** | 文件共享 | EFS、NAS | NFS / SMB |
| **归档存储** | 长期备份 | Glacier、Archive | 冷存取 |

---

## 二、对象存储 (Object Storage)

### 对象存储特点

| 特性 | 说明 |
|------|------|
| **扁平结构** | 没有目录层级 |
| **元数据丰富** | 自定义标签、权限 |
| **无限扩展** | 自动扩容 |
| **高可用** | 多副本、跨区域 |

### S3 核心概念

```
Bucket (桶)
    │
    ├─── Objects (对象)
    │    │
    │    ├── file1.jpg
    │    ├── file2.pdf
    │    └── data/
    │         └── nested.json
    │
    ├─── Versioning (版本控制)
    └─── Lifecycle (生命周期)
```

### S3 存储类别

| 类别 | 用途 | 获取时间 | 成本 |
|------|------|----------|------|
| **Standard** | 频繁访问 | 毫秒级 | 标准 |
| **Intelligent-Tiering** | 访问模式未知 | 毫秒级 | 自动优化 |
| **Standard-IA** | 不频繁访问 | 毫秒级 | 较低 |
| **One Zone-IA** | 单区域不频繁 | 毫秒级 | 更低 |
| **Glacier** | 长期归档 | 分钟-小时 | 最低 |
| **Deep Archive** | 极长期归档 | 小时 | 最低 |

### S3 生命周期策略

```json
{
  "Rules": [
    {
      "Status": "Enabled",
      "Filter": {"Prefix": "logs/"},
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ],
      "Expiration": {
        "Days": 365
      }
    }
  ]
}
```

---

## 三、块存储 (Block Storage)

### 块存储特点

| 特性 | 说明 |
|------|------|
| **块级访问** | 固定大小的块 |
| **低延迟** | 本地挂载 |
| **可调整** | 在线扩容 |

### EBS 卷类型对比

| 类型 | 性能 | 用途 | 成本 |
|------|------|------|------|
| **gp3 (通用 SSD)** | 3000-16000 IOPS | 通用工作负载 | 标准 |
| **gp2 (旧版通用 SSD)** | 根据容量变化 | 通用 | 较高 |
| **io1/io2 (Provisioned IOPS)** | 最高 64000 IOPS | 关键数据库 | 高 |
| **st1 (吞吐优化 HDD)** | 高吞吐 | 大数据流 | 低 |
| **sc1 (冷 HDD)** | 最低成本 | 备份 | 最低 |

### EBS 性能配置

```bash
# 创建 gp3 卷，指定 IOPS 和吞吐量
aws ec2 create-volume \
    --volume-type gp3 \
    --size 100 \
    --iops 3000 \
    --throughput 125
```

### EBS 快照

| 特性 | 说明 |
|------|------|
| **增量备份** | 只保存变化的数据 |
| **跨区域** | 复制到其他区域 |
| **生命周期** | 自动删除旧快照 |

---

## 四、文件存储 (File Storage)

### 文件存储特点

| 特性 | 说明 |
|------|------|
| **文件系统** | 完整的目录结构 |
| **共享访问** | 多实例同时访问 |
| **自动扩容** | 无需手动扩容 |

### EFS vs 文件存储服务

| 特性 | AWS EFS | 阿里云 NAS | GCP Filestore |
|------|----------|-------------|---------------|
| 协议 | NFS | NFS/SMB | NFS |
| 性能 | 动态调整 | 预配置 | 预配置 |
| 成本 | 按 GB + 读写 | 按 GB | 按 GB + 性能 |
| 区域 | 多可用区 | 单/多可用区 | 单可用区 |

---

## 五、存储网关

### 存储网关类型

| 类型 | 说明 | 用途 |
|------|------|------|
| **File Gateway** | NFS/SMB 文件共享 | 备份、归档 |
| **Volume Gateway** | iSCSI 块存储 | 数据库、虚拟机 |
| **Tape Gateway** | VTL 磁带备份 | 长期归档 |

---

## 六、数据传输

### 传输方式

| 方式 | 带宽 | 成本 | 适用场景 |
|------|------|------|----------|
| **公网** | 取决于网络 | 按流量 | 少量数据 |
| **VPN** | 加密通道 | VPN 费用 | 安全传输 |
| **Direct Connect** | 专线 | 专线费用 | 大量数据 |
| **Snowball** | 物理设备 | 固定费用 | PB 级数据 |

### Snowball 系列

| 设备 | 容量 | 用途 |
|------|------|------|
| **Snowball Edge** | 8TB - 100TB | 边缘计算 |
| **Snowcone** | 8TB | 小规模迁移 |
| **Snowmobile** | 100PB | 超大规模 |

---

## 七、面试要点

### 高频问题

1. **对象存储 vs 块存储？**
   - 对象存储：非结构化数据，API 访问
   - 块存储：文件系统、数据库，挂载访问

2. **EBS gp3 vs gp2？**
   - gp3：独立配置 IOPS 和吞吐量，成本更低
   - gp2：IOPS 随容量增加，成本较高

3. **何时使用 Glacier？**
   - 长期备份
   - 合规要求
   - 数据极少访问

4. **EFS vs EBS？**
   - EFS：多实例共享，自动扩容
   - EBS：单实例，性能可调

5. **如何选择存储类别？**
   - 频繁访问：Standard
   - 不频繁：Standard-IA
   - 归档：Glacier

---

## 八、最佳实践

1. **使用生命周期策略**：自动归档和删除
2. **启用版本控制**：防止误删除
3. **加密数据**：传输中 + 静态加密
4. **监控存储成本**：定期审计使用情况
5. **使用多可用区**：提高可用性

## 七、面试要点

### 高频问题

1. **对象存储 vs 块存储？**
   - 对象存储：非结构化数据，API 访问
   - 块存储：文件系统、数据库，挂载访问

2. **EBS gp3 vs gp2？**
   - gp3：独立配置 IOPS 和吞吐量，成本更低
   - gp2：IOPS 随容量增加，成本较高

3. **何时使用 Glacier？**
   - 长期备份
   - 合规要求
   - 数据极少访问

4. **EFS vs EBS？**
   - EFS：多实例共享，自动扩容
   - EBS：单实例，性能可调

5. **如何选择存储类别？**
   - 频繁访问：Standard
   - 不频繁：Standard-IA
   - 归档：Glacier

## 八、最佳实践

1. **使用生命周期策略**：自动归档和删除
2. **启用版本控制**：防止误删除
3. **加密数据**：传输中 + 静态加密
4. **监控存储成本**：定期审计使用情况
5. **使用多可用区**：提高可用性
