## 一、计算服务类型

### 1. 虚拟机 (VM)

最基础的计算服务，提供虚拟机实例。

| 特性 | 说明 |
|------|------|
| **弹性** | 按需创建/销毁 |
| **灵活** | 自选 OS、CPU、内存、存储 |
| **计费** | 按小时/按秒计费 |

### 2. 容器服务

基于容器的计算服务，自动编排管理。

| 服务 | 描述 |
|------|------|
| **EKS / AKS / GKE / ACK** | 托管 K8s 服务 |
| **Fargate / Serverless Containers** | 无需管理节点 |

### 3. 函数计算 (FaaS)

事件驱动的无服务器计算。

| 特性 | 说明 |
|------|------|
| **按量计费** | 按调用次数和执行时长 |
| **自动扩缩容** | 无需配置 |
| **事件驱动** | HTTP 请求、消息触发 |

### 4. 专用主机

物理服务器独占使用。

| 用途 | 说明 |
|------|------|
| **许可合规** | 需要绑定物理 CPU 的许可证 |
| **性能保障** | 无其他租户干扰 |
| **安全合规** | 满足特定合规要求 |

---

## 二、虚拟机实例类型

### 实例类型分类

| 类型 | 用途 | 代表实例 |
|------|------|----------|
| **通用型** | 均衡计算、内存 | t3、m5、c5 |
| **计算优化** | 高性能计算 | c5、c6g |
| **内存优化** | 内存密集型应用 | r5、r6g |
| **存储优化** | 高 IOPS、吞吐量 | i3、i3en |
| **加速计算** | GPU/FPGA 加速 | p3、p4、g4dn |
| **突发型** | 低负载、成本低 | t3、t3a |

### 实例命名规范 (AWS)

```
  c    6   g    .   medi um  .  2xlarge
  │    │    │        │          │
  │    │    │        │          └─ 实例大小 (nano, micro, small, medium, large, xlarge)
  │    │    │        └─ 实例规格
  │    │    └─ 代数 (5th, 6th generation)
  │    └─ 实例类型 (c=计算, m=通用, r=内存, i=存储, p=GPU)
  └─ 厂商前缀 (c=EC2)
```

### 竞价实例 (Spot Instance)

使用闲置容量，成本大幅降低。

| 特性 | 说明 |
|------|------|
| **价格** | 最高可达按需价格的 90% 折扣 |
| **中断** | 云服务商可随时回收 |
| **适用** | 容错、灵活的工作负载 |

```bash
# AWS 竞价实例请求示例
aws ec2 request-spot-instances \
    --spot-price "0.05" \
    --instance-count 1 \
    --launch-specification "..."
```

### 预留实例 (Reserved Instance)

长期使用可获得折扣。

| 类型 | 折扣范围 | 灵活性 |
|------|----------|--------|
| **标准预留** | 最高 75% | 固定期、固定实例 |
| **可转换预留** | 最高 54% | 可转换实例类型 |
| **Savings Plans** | 最高 72% | 灵活使用 |

---

## 三、实例生命周期

### 实例状态

```
pending ──→ running ──→ shutting-down ──→ terminated
              │              │
              │              └─→ stopped
              └─→ stopping ──→ stopped
```

| 状态 | 说明 |
|------|------|
| **pending** | 实例启动中 |
| **running** | 实例运行中 |
| **stopping** | 实例停止中 |
| **stopped** | 实例已停止（数据保留）|
| **terminated** | 实例已终止（数据清除）|

## 四、自动扩缩容

### Auto Scaling 组件

```
             ┌─────────────┐
             │   用户请求   │
             └──────┬──────┘
                    │
             ┌──────▼─────────────────────────┐
             │      Load Balancer (ALB)       │
             └──────┬─────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐      ┌───▼───┐      ┌───▼───┐
│ Auto  │      │ Auto  │      │ Auto  │
│ Scaling Group│ Scaling Group│ Scaling Group│
│ (3 instances)│ (2 instances)│ (1 instance)│
└───┬───┘      └───┬───┘      └───┬───┘
    │               │               │
    └───────────────┼───────────────┘
                    │
            ┌───────▼────────┐
            │  Scaling Policy│
            │  (扩缩容策略) │
            └───────┬────────┘
                    │
            ┌───────▼────────┐
            │ CloudWatch     │
            │  (监控指标)    │
            └───────────────┘
```

### 扩缩容策略

| 策略 | 触发条件 | 操作 |
|------|----------|------|
| **目标追踪** | CPU > 70% | 调整到目标 70% |
| **步进策略** | CPU > 80% | 增加 2 个实例 |
| **简单策略** | CPU > 75% | 增加到 5 个实例 |

## 五、函数计算 (FaaS)

### FaaS 特性

| 特性 | 说明 |
|------|------|
| **无服务器** | 无需管理服务器 |
| **事件驱动** | HTTP、定时、消息触发 |
| **按需计费** | 按调用次数和执行时长 |
| **自动扩缩容** | 无需配置 |

### 函数计算对比

| 服务 | 冷启动 | 最大执行时间 | 语言支持 |
|------|---------|--------------|----------|
| **AWS Lambda** | 秒级 | 15 分钟 | 多语言 |
| **阿里云 FC** | 秒级 | 15 分钟 | 多语言 |
| **Azure Functions** | 秒级 | 10 分钟 | 多语言 |
| **GCP Cloud Functions** | 毫秒级 | 60 分钟 | 多语言 |

### Lambda 函数示例

```python
import json

def lambda_handler(event, context):
    # 解析事件
    name = event.get('name', 'World')

    # 返回响应
    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': f'Hello, {name}!'
        })
    }
```

## 六、GPU 实例

### GPU 实例类型

| 实例 | GPU | 显存 | 用途 |
|------|-----|------|------|
| **g4dn.xlarge** | 1x T4 | 16GB | 推理、轻量训练 |
| **p3.2xlarge** | 1x V100 | 32GB | 中等训练 |
| **p3.8xlarge** | 4x V100 | 128GB | 大规模训练 |
| **p4d.24xlarge** | 8x A100 | 320GB | 超大规模训练 |

### GPU 使用场景

| 场景 | 推荐实例 |
|------|----------|
| **模型推理** | g4dn、g5 |
| **深度学习训练** | p3、p4、g5 |
| **机器视觉** | p3、p4 |
| **自然语言处理** | p4、p5 |

## 七、面试要点

### 高频问题

1. **何时使用竞价实例？**
   - 容错能力强的工作负载
   - 批处理作业
   - 成本敏感场景

2. **Auto Scaling 如何工作？**
   - CloudWatch 监控指标
   - 达到阈值触发扩缩容策略
   - Auto Scaling Group 增减实例

3. **Lambda 冷启动如何优化？**
   - 保持热实例
   - 优化依赖包大小
   - 使用 Provisioned Concurrency

4. **如何选择实例类型？**
   - 通用型：Web 应用
   - 计算优化：科学计算
   - 内存优化：数据库
   - 存储优化：高 IOPS 应用

---

## 八、最佳实践

1. **使用 Auto Scaling**：根据负载自动调整
2. **合理使用竞价实例**：成本优化
3. **预留长期实例**：降低成本
4. **监控资源使用**：避免过度配置
5. **使用 Spot Fleet**：混合按需和竞价

              └─→ stopping ──→ stopped
```

| 状态 | 说明 |
|------|------|
| **pending** | 实例启动中 |
| **running** | 实例运行中 |
| **stopping** | 实例停止中 |
| **stopped** | 实例已停止（数据保留）|
| **terminated** | 实例已终止（数据清除）|

## 四、自动扩缩容

### Auto Scaling 组件

```
             ┌─────────────┐
             │   用户请求   │
             └──────┬──────┘
                    │
             ┌──────▼─────────────────────────┐
             │      Load Balancer (ALB)       │
             └──────┬─────────────────────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐      ┌───▼───┐      ┌───▼───┐
│ Auto  │      │ Auto  │      │ Auto  │
│ Scaling Group│ Scaling Group│ Scaling Group│
│ (3 instances)│ (2 instances)│ (1 instance)│
└───┬───┘      └───┬───┘      └───┬───┘
    │               │               │
    └───────────────┼───────────────┘
                    │
            ┌───────▼────────┐
            │  Scaling Policy│
            │  (扩缩容策略) │
            └───────┬────────┘
                    │
            ┌───────▼────────┐
            │ CloudWatch     │
            │  (监控指标)    │
            └───────────────┘
```

### 扩缩容策略

| 策略 | 触发条件 | 操作 |
|------|----------|------|
| **目标追踪** | CPU > 70% | 调整到目标 70% |
| **步进策略** | CPU > 80% | 增加 2 个实例 |
| **简单策略** | CPU > 75% | 增加到 5 个实例 |

## 七、面试要点

### 高频问题

1. **何时使用竞价实例？**
   - 容错能力强的工作负载
   - 批处理作业
   - 成本敏感场景

2. **Auto Scaling 如何工作？**
   - CloudWatch 监控指标
   - 达到阈值触发扩缩容策略
   - Auto Scaling Group 增减实例

3. **Lambda 冷启动如何优化？**
   - 保持热实例
   - 优化依赖包大小
   - 使用 Provisioned Concurrency

4. **如何选择实例类型？**
   - 通用型：Web 应用
   - 计算优化：科学计算
   - 内存优化：数据库
   - 存储优化：高 IOPS 应用

## 八、最佳实践

1. **使用 Auto Scaling**：根据负载自动调整
2. **合理使用竞价实例**：成本优化
3. **预留长期实例**：降低成本
4. **监控资源使用**：避免过度配置
5. **使用 Spot Fleet**：混合按需和竞价
