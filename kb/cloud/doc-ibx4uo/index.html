计算服务概览

计算服务概览

更新日期：2026-03-13 相关文档：云计算核心概念 | 虚拟化技术详解 | Kubernetes架构详解

一、计算服务类型

1. 虚拟机 (VM)

最基础的计算服务，提供虚拟机实例。

特性说明
弹性按需创建/销毁
灵活自选 OS、CPU、内存、存储
计费按小时/按秒计费

2. 容器服务

基于容器的计算服务，自动编排管理。

服务描述
EKS / AKS / GKE / ACK托管 K8s 服务
Fargate / Serverless Containers无需管理节点

3. 函数计算 (FaaS)

事件驱动的无服务器计算。

特性说明
按量计费按调用次数和执行时长
自动扩缩容无需配置
事件驱动HTTP 请求、消息触发

4. 专用主机

物理服务器独占使用。

用途说明
许可合规需要绑定物理 CPU 的许可证
性能保障无其他租户干扰
安全合规满足特定合规要求

二、虚拟机实例类型

实例类型分类

类型用途代表实例
通用型均衡计算、内存t3、m5、c5
计算优化高性能计算c5、c6g
内存优化内存密集型应用r5、r6g
存储优化高 IOPS、吞吐量i3、i3en
加速计算GPU/FPGA 加速p3、p4、g4dn
突发型低负载、成本低t3、t3a

实例命名规范 (AWS)

  c    6   g    .   medi um  .  2xlarge
  │    │    │        │          │
  │    │    │        │          └─ 实例大小 (nano, micro, small, medium, large, xlarge)
  │    │    │        └─ 实例规格
  │    │    └─ 代数 (5th, 6th generation)
  │    └─ 实例类型 (c=计算, m=通用, r=内存, i=存储, p=GPU)
  └─ 厂商前缀 (c=EC2)

竞价实例 (Spot Instance)

使用闲置容量，成本大幅降低。

特性说明
价格最高可达按需价格的 90% 折扣
中断云服务商可随时回收
适用容错、灵活的工作负载
# AWS 竞价实例请求示例
aws ec2 request-spot-instances \
    --spot-price "0.05" \
    --instance-count 1 \
    --launch-specification "..."

预留实例 (Reserved Instance)

长期使用可获得折扣。

类型折扣范围灵活性
标准预留最高 75%固定期、固定实例
可转换预留最高 54%可转换实例类型
Savings Plans最高 72%灵活使用

三、实例生命周期

实例状态

pending ──→ running ──→ shutting-down ──→ terminated
              │              │
              │              └─→ stopped
              └─→ stopping ──→ stopped
状态说明
pending实例启动中
running实例运行中
stopping实例停止中
stopped实例已停止（数据保留）
terminated实例已终止（数据清除）

Lambda 函数示例

import json

def lambda_handler(event, context):
    # 解析事件
    name = event.get('name', 'World')

    # 返回响应
    return {
        'statusCode': 200,
        'body': json.dumps({
            'message': f'Hello, {name}!'
        })
    }

七、面试要点

高频问题

  1. 何时使用竞价实例？

    • 容错能力强的工作负载
    • 批处理作业
    • 成本敏感场景

  2. Auto Scaling 如何工作？

    • CloudWatch 监控指标
    • 达到阈值触发扩缩容策略
    • Auto Scaling Group 增减实例

  3. Lambda 冷启动如何优化？

    • 保持热实例
    • 优化依赖包大小
    • 使用 Provisioned Concurrency

  4. 如何选择实例类型？

    • 通用型：Web 应用
    • 计算优化：科学计算
    • 内存优化：数据库
    • 存储优化：高 IOPS 应用

八、最佳实践

  1. 使用 Auto Scaling：根据负载自动调整
  2. 合理使用竞价实例：成本优化
  3. 预留长期实例：降低成本
  4. 监控资源使用：避免过度配置
  5. 使用 Spot Fleet：混合按需和竞价
