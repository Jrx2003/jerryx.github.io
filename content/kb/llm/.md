## 优化器与学习率调度

### AdamW 优化器

**Adam with Weight Decay**

```python
# PyTorch 实现
optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=1e-4,
    betas=(0.9, 0.95),  # β2 通常设为 0.95 或 0.999
    eps=1e-8,
    weight_decay=0.1   # 权重衰减
)
```

**参数说明**:

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| lr | 1e-4 ~ 6e-4 | 峰值学习率 |
| β1 | 0.9 | 一阶矩衰减 |
| β2 | 0.95 ~ 0.999 | 二阶矩衰减 |
| eps | 1e-8 | 数值稳定 |
| weight_decay | 0.1 | L2 正则化 |

**Adam vs AdamW**:

```
Adam: 权重衰减应用在梯度上
AdamW: 权重衰减与梯度解耦，更稳定

推荐: 大模型预训练使用 AdamW
```

### 学习率调度

#### 1. 线性预热 + 余弦衰减

```python
def get_lr_scheduler(optimizer, num_warmup_steps, num_training_steps):
    """线性预热 + 余弦衰减"""
    def lr_lambda(current_step):
        if current_step < num_warmup_steps:
            # 线性预热
            return current_step / num_warmup_steps
        else:
            # 余弦衰减
            progress = (current_step - num_warmup_steps) / (num_training_steps - num_warmup_steps)
            return 0.5 * (1 + math.cos(math.pi * progress))

    return torch.optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)
```

**学习率曲线**:

```
学习率
  │    ╭────╮
  │   ╱      ╲_______
  │  ╱                 ╲
  │ ╱                     ╲
  │╱                         ╲___
  └──────────────────────────────→ 步数
    预热期              衰减期
```

#### 2. Warmup-Stable-Decay (WSD)

```
学习率
  │        ╭──────────────╮
  │       ╱                ╲
  │      ╱                  ╲
  │     ╱                    ╲
  │    ╱                      ╲
  │   ╱                        ╲
  │  ╱                          ╲
  │ ╱                            ╲
  │╱                              ╲
  └────────────────────────────────→ 步数
    预热期    稳定期      衰减期
```

**优点**: 可以在稳定期任意点恢复训练，便于实验。

### 学习率与 batch size 的关系

**线性缩放规则**:

```
当 batch size 扩大 k 倍时，学习率也扩大 k 倍

lr_new = lr_base × (batch_size_new / batch_size_base)
```

**平方根缩放** (当 batch size 很大时):

```
lr_new = lr_base × sqrt(batch_size_new / batch_size_base)
```

## 混合精度训练

### 为什么需要混合精度

| 精度 | 内存 | 速度 | 范围 |
|------|------|------|------|
| FP32 | 4 bytes | 基准 | 大 |
| FP16 | 2 bytes | 2x | 小 |
| BF16 | 2 bytes | 2x | 与 FP32 相同 |

**优势**:
- 内存减半
- 计算速度提升 (Tensor Core)
- 可训练更大模型

### FP16 的挑战

**梯度下溢 (Underflow)**:

```
FP16 最小正规格化数: 2^-14 ≈ 6.1e-5
小梯度可能直接变为 0
```

**解决方案**:

1. **损失缩放 (Loss Scaling)**
   ```python
   # 前向传播
   with autocast():
       loss = model(input)

   # 放大损失
   scaled_loss = loss * scaler.get_scale()
   scaled_loss.backward()

   # 梯度更新后缩放回正常范围
   scaler.step(optimizer)
   scaler.update()
   ```

2. **BF16 (Brain Float 16)**
   ```
   BF16 指数位与 FP32 相同，不会下溢
   但精度较低 (7 位尾数 vs 10 位)
   ```

### PyTorch 混合精度实现

```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for batch in dataloader:
    optimizer.zero_grad()

    # 自动选择 FP16/FP32
    with autocast(dtype=torch.float16):
        outputs = model(batch)
        loss = criterion(outputs, labels)

    # 缩放损失并反向传播
    scaler.scale(loss).backward()

    # 梯度裁剪 (可选)
    scaler.unscale_(optimizer)
    torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

    # 更新参数
    scaler.step(optimizer)
    scaler.update()
```

## 分布式训练

### 数据并行 (Data Parallelism, DP)

```
┌─────────────────────────────────────────────────┐
│                  数据并行                        │
├─────────────────────────────────────────────────┤
│                                                  │
│  Batch ──→ Split ──→ GPU 0  GPU 1  GPU 2  GPU 3 │
│            [B/4]    ┌───┐  ┌───┐  ┌───┐  ┌───┐ │
│                     │Model│  │Model│  │Model│  │Model│ │
│                     └───┘  └───┘  └───┘  └───┘ │
│                       │      │      │      │   │
│                     Loss   Loss   Loss   Loss  │
│                       │      │      │      │   │
│                       └──────┴──────┴──────┘   │
│                              │                 │
│                           All-Reduce           │
│                              │                 │
│                           Update               │
└─────────────────────────────────────────────────┘
```

**PyTorch DDP**:

```python
import torch
import torch.distributed as dist
from torch.nn.parallel import DistributedDataParallel as DDP

# 初始化进程组
dist.init_process_group(backend='nccl')

# 包装模型
model = DDP(model, device_ids=[local_rank])

# 训练 (与普通训练相同)
for batch in dataloader:
    loss = model(batch)
    loss.backward()
    optimizer.step()
    # DDP 自动处理梯度同步
```

### 完全分片数据并行 (FSDP)

**Fully Sharded Data Parallel**

```
DDP: 每个 GPU 保存完整模型副本
FSDP: 模型参数分片存储，计算时按需收集

内存节省: 与 GPU 数量成正比
```

```python
from torch.distributed.fsdp import FullyShardedDataParallel as FSDP
from torch.distributed.fsdp.wrap import transformer_auto_wrap_policy

model = FSDP(
    model,
    auto_wrap_policy=transformer_auto_wrap_policy,
    mixed_precision=torch.bfloat16,
    device_id=torch.cuda.current_device(),
    limit_all_gathers=True
)
```

## 梯度累积与裁剪

### 梯度累积

**解决显存不足，模拟大 batch size**:

```python
accumulation_steps = 4  # 累积 4 步

for i, batch in enumerate(dataloader):
    loss = model(batch) / accumulation_steps  # 缩放损失
    loss.backward()

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()
```

**有效 batch size**:
```
effective_batch_size = per_gpu_batch_size × num_gpus × accumulation_steps
```

### 梯度裁剪

**防止梯度爆炸**:

```python
# 按范数裁剪
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# 按值裁剪
torch.nn.utils.clip_grad_value_(model.parameters(), clip_value=1.0)
```

**推荐值**:
- GPT 系列: 1.0
- BERT: 1.0
- 一般范围: 0.5 - 5.0

## 数据并行与模型并行

### 张量并行 (Tensor Parallelism)

**将层内计算分到多个 GPU**:

```
Linear 层: Y = XW

W 分片:
GPU 0: W1  GPU 1: W2

计算:
Y = [XW1 | XW2]  (列并行)
或
Y = XW1 + XW2    (行并行)
```

**Megatron-LM 实现**:

```python
# 列并行线性层
class ColumnParallelLinear(nn.Module):
    def __init__(self, input_size, output_size, world_size):
        super().__init__()
        # 每个 GPU 负责部分输出维度
        self.output_size_per_partition = output_size // world_size
        self.weight = nn.Parameter(torch.empty(
            self.output_size_per_partition, input_size
        ))

    def forward(self, x):
        # 本地计算
        output_parallel = F.linear(x, self.weight)
        # 收集所有 GPU 的结果
        output = gather_from_tensor_model_parallel_region(output_parallel)
        return output
```

### 流水线并行 (Pipeline Parallelism)

**将模型层分到多个 GPU**:

```
GPU 0: Layer 1-4    GPU 1: Layer 5-8    GPU 2: Layer 9-12

输入 → [Layers 1-4] → [Layers 5-8] → [Layers 9-12] → 输出
       GPU 0          GPU 1          GPU 2
```

**GPipe / PipeDream**:

```python
# 简单流水线实现
class PipelineParallel(nn.Module):
    def __init__(self, layers, num_stages):
        super().__init()
        self.stages = nn.ModuleList([
            layers[i::num_stages] for i in range(num_stages)
        ])

    def forward(self, x, chunk_size=1):
        # 微批次流水线
        chunks = x.split(chunk_size)
        outputs = []

        for chunk in chunks:
            for stage in self.stages:
                chunk = stage(chunk)
            outputs.append(chunk)

        return torch.cat(outputs)
```

### 3D 并行

```
数据并行 (DP) × 张量并行 (TP) × 流水线并行 (PP)

示例: 1024 GPU
- TP: 8 (单节点内)
- PP: 16 (跨节点)
- DP: 8 (全局)

总 GPU = 8 × 16 × 8 = 1024
```

## 训练稳定性技巧

### 1. 权重初始化

**Xavier / Glorot 初始化**:

```python
# 适用于 tanh/sigmoid
nn.init.xavier_uniform_(tensor, gain=1.0)
```

**Kaiming / He 初始化**:

```python
# 适用于 ReLU/GELU
nn.init.kaiming_uniform_(tensor, mode='fan_in', nonlinearity='relu')
```

**GPT-2/3 初始化**:

```python
# 残差层权重按 1/sqrt(N) 缩放
# N 是残差层数
self.c_proj.weight.data.normal_(mean=0.0, std=0.02)
self.c_proj.weight.data.div_(math.sqrt(2 * self.num_layers))
```

### 2. 嵌入层稳定性

**嵌入层梯度裁剪**:

```python
# 单独对嵌入层进行梯度裁剪
for name, param in model.named_parameters():
    if 'embed' in name:
        param.grad.data.clamp_(-0.1, 0.1)
```

### 3. 学习率预热

**防止早期训练不稳定**:

```python
# 前 2000 步线性增加学习率
warmup_steps = 2000
if step < warmup_steps:
    lr = base_lr * (step / warmup_steps)
```

### 4. 检查 NaN/Inf

```python
def check_nan_inf(model):
    """检查模型参数中的 NaN/Inf"""
    for name, param in model.named_parameters():
        if torch.isnan(param).any():
            print(f"NaN detected in {name}")
            return False
        if torch.isinf(param).any():
            print(f"Inf detected in {name}")
            return False
    return True
```

## 面试要点

### 高频问题

**Q1: AdamW 相比 Adam 的优势？**

A: AdamW 将权重衰减与梯度更新解耦，避免了 Adam 中权重衰减与学习率耦合的问题，在大模型训练中更稳定。

**Q2: 为什么要使用学习率预热？**

A: 训练初期，模型参数随机初始化，梯度可能很大。线性预热可以逐步增加学习率，防止早期训练不稳定和损失震荡。

**Q3: 混合精度训练中为什么要损失缩放？**

A: FP16 的表示范围小，小梯度可能下溢为 0。损失缩放将损失乘以一个大的因子，使梯度也相应放大，避免下溢。

**Q4: DDP 和 FSDP 的区别？**

A:
- DDP: 每个 GPU 保存完整模型，只同步梯度
- FSDP: 模型参数分片存储，计算时收集，内存效率更高

**Q5: 梯度累积的原理和作用？**

A: 将多个小 batch 的梯度累积后再更新参数，等效于使用更大的 batch size，解决显存不足问题。
