## 一、Serverless 2.0

### 极致弹性

| 特性 | 说明 |
|------|------|
| **毫秒级启动** | 预热实例、编译优化 |
| **自动扩缩容** | 基于流量自动调整 |
| **按量计费** | 精确到毫秒 |

### Serverless 容器

| 服务 | 描述 |
|------|------|
| **AWS Fargate** | 无需管理节点的容器 |
| **Azure Container Apps** | 微服务 Serverless 容器 |
| **GCP Cloud Run** | 容器化应用 Serverless |

### 冷启动优化

```
冷启动流程：

应用代码
    │
    ▼
1. 下载镜像 (2-3s)
    │
    ▼
2. 创建容器 (1-2s)
    │
    ▼
3. 初始化运行时 (1-2s)
    │
    ▼
4. 加载依赖 (2-5s)
    │
    ▼
5. 应用初始化 (1-3s)

总耗时：7-15s
```

| 优化方法 | 效果 |
|----------|------|
| **SnapStart** | 将冷启动降低到 < 100ms |
| **预置并发** | 保持热实例池 |
| **精简依赖** | 减少镜像和代码大小 |
| **AOT 编译** | 减少运行时初始化 |

## 二、WebAssembly (WASM)

### WASM vs 容器

| 维度 | 容器 | WASM |
|------|-------|------|
| 启动时间 | 秒级 | 毫秒级 |
| 镜像大小 | MB 级 | KB 级 |
| 隔离性 | 进程级 | 沙箱隔离 |
| 兼容性 | 依赖 OS | 跨平台 |

### WASM 应用场景

| 场景 | 说明 |
|------|------|
| **边缘计算** | 资源受限环境 |
| **Serverless** | 降低冷启动 |
| **微前端** | 按需加载组件 |
| **安全隔离** | 不可信代码执行 |

## 八、面试要点

### 高频问题

1. **Serverless 冷启动如何优化？**
   - SnapStart 预编译镜像
   - 预置并发保持热实例
   - 精简代码和依赖

2. **WASM 相比容器的优势？**
   - 启动更快（毫秒级）
   - 占用更小（KB 级）
   - 跨平台兼容

3. **边缘计算的应用场景？**
   - IoT 设备
   - 实时分析
   - 低延迟应用（AR/VR）

4. **FinOps 的核心是什么？**
   - 云成本可视化
   - 优化资源使用
   - 预测和规划

5. **机密计算解决了什么问题？**
   - 数据使用过程中的安全
   - 敏感数据处理
   - 多方计算
